# 三、ARP 协议
**地址解析协议 Address Resolution Protocol**

* 通过 IP 地址获取 MAC 地址
* 同一网络中的主机或路由器接口之间才能使用 ARP 获取 MAC 地址
* 主机或路由器拿到目的主机的 MAC 地址后，会记录在自己的 ARP 缓存中
* 每次通信前，都会查看自己的ARP表，没有找到，才会再次发送ARP请求

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715689367000q9dfff.png)

> ARP 表中对应数据不是一直的存在，根本也是一种缓存表，是会更新数据的。

# 四、ICMP 协议
**网际控制报文协议Internet Control Message Protoco**

* 主机或路由器使用ICMP 来发送询问报文和差错报告报文
* 常用的询问报文有两种:回送请求和回答、时间戳请求和回答
* ICMP 是网络层协议
* **ICMP 报文被封装在IP 数据报中发送**
* **ICMP 报文作为IP 数据报的数据载荷**
![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/17156898000003jhqdi.png)

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715689879000vcwzwl.png)

> IP 协议报文中的协议就是IP协议中包含的协议是什么

* 协议的数据载荷并不一定都是从上一层来的，有的是没有上一层的


# 4.1 ICMP 询问报文-回送请求和回答
* 由主机或路由器向一个特定的目的主机或路由器发出
* 收到此报文的主机或路由器**必须发送** ICMP 回送回答报文
* 这种询问报文用来测试目的站是否可达以及了解其有关状态
* ping 命令使用的就是 ICMP 询问报文
* 用来向主机或路由器报告差错情况
* 常见的差错有:终点不可达、超时等
* tracert 命令用于探测 IP 数据报从源主机到达目的主机要经过哪些路由器(最多可以追踪30)
* tracert 使用的 ICMP 报文类型有询问报文以及差错报告报文
