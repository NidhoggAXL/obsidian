# 三、划分子网
产生的原因，有时候我们其实不需要很多的主机，但是IP地址中C类地址的可分配主机数最少也有200多台，会造成地址的浪费。

* 从主机号中借用一些比特位作为子网号来划分出不同的子网

* 可以使用子网掩码标识出子网的网络号和主机号分别有多少位

* 子网掩码:左起连续的1对应IP中的网络号和子网号，之后的0对应主机号

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715677488000fk5a1p.png)


**默认子网掩码**

默认子网掩码是指在未划分子网的情况下使用的子网掩码

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715677582000oqvs0i.png)

## 3.1 通过IP地址和子网掩码确定网络地址
将 IP 地址与相应的子网掩码按位与，就得到该IP 地址所在的网络地址

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/17156776850000ao2so.png)

## 3.2 CIDR
* 仅从IP 地址无法知道网络号和主机号，还需要子网掩码的配合
* 可以使用 CIDR 简化IP 和子网掩码的表达
* CIDR:无分类域间路由选择Classless Inter-Domain Routing

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715677902000qat3fp.png)


## 3.3 同一网络的判断
**使用交叉线直连的这两台主机可以正常通信吗?**
 ![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715678105000fwo27l.png)
> 通过实验是不可以正常通信的。

**判断过程：**

* 通信时，发送方主机用自己的IP地址和自己的子网掩码按位与
* 发送方主机再用目的主机的IP 地址和**自己的**子网掩码按位与
* 判断目的主机是否与自己在同一网络/段
	* 如果目的主机与自己在同一网络，可以直接向目的主机发送消息
	* 如果目的主机与自己不在同一网络，需要将消息发送到网关(路由器)

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715678563000jbzjco.png)

> A 向 B 发送消息是直接不能发送到 B
> B 向 A 发送数据的时候是到达了 A ，但是 A 认为不是同一个网络，把数据丢弃
> 所以说 A 和 B 不在同一个网段


# 四、公网IP和私网IP
**为了缓解 IP 地址不够用的问题**


* 公网IP:主要用于广域网，一般各不相同
* 私网IP:主要用于局域网，不同网络中可以相同
	* A类:10.0.0.0/8~10.255.255.255/8，1个A类网络
	* B类:172.16.0.0/16~172.31.0.0/16，16个B类网络
	* C类:192.168.0.0/24~192.168.255.0/24，256个C类网络

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715678968000mu5zc6.png)

那不同的网络的主机之间如何连接呢？

## 4.1 NAT
**网络地址转换 Network Address Translation**


* 在私有网络连接到互联网的路由器上安装 NAT 软件，称为 NAT 路由器
* NAT 路由器至少有一个有效的公网IP
	* 一般路由器中有公网IP池，当有私网来连接的时候会从IP池里面拿一个出来使用，当使用完的时候又会还回去。
* NAT 路由器会建立公网IP 和私网 IP 的对应关系表
* 所有私有网络的主机在和外部通信时，会在 NAT路由器上转换成公网IP

**NAT表**

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/17156794350001cjrsf.png)

每一个私网 IP 对应一个公网 IP
* 发送数据的时候通过把私网 IP 转换位公网 IP 发送出去
* 接收数据的时候，通过公网 IP 找到一一对应的私网 IP

> 但是这样了还是需要很多公网 IP

## 4.2 NAPT
**网络地址与端口号转换 Network Address and Port Translation**

* 将 NAT 转换和运输层端口号结合使用
* 使用私有网络的大量主机，可以共用 NAPT路由器上的1个全球 IP 地址
* NAPT 路由器会建立公网 IP 、端口号和私网 IP、 端口号的对应关系表
* 所有私有网络的主机在和外部通信时，会在 NAPT路由器上转换成公网IP

**NAPT表**

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715679762000j8t661.png)

* 每一个私网的 IP 对应不同的端口，相同的公网 IP 
* 接收数据的时候相同的公网 IP 通过不同的端口号来找回私网 IP
