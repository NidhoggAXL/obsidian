# 一、FTP 协议
## 1.1 FTP 定义
文件传送协议 File Transfer Protocol

常见用途是从 FTP 服务器批量下载文件

另一常见用途是批量上传网站资源(HTML/CSS/JS 等)到 Web 服务器

FTP 协议基于 TCP

## 1.2 工作原理
![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715869206000iyurso.png)


**FTP 客户端和服务器之间会建立两个 TCP 连接**

* 一个是控制连接，用于传输相关控制命令，整个会话期间保持打开，使用的时 21 端口

* 一个是数据连接，用于文件传输，每次文件传输时才建立，结束就关闭，使用的是 20 端口

**传输控制命令的 TCP 连接一定是客户端主动发起建立的**

**FTP 模式：**

* 传输数据的连接如果是服务器主动发起建立的，是 FTP主动模式
* 传输数据的连接如果是客户端主动发起建立的，是 FTP被动模式

# 二、邮件协议

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715869516000eamagy.png)

邮件发送协议:SMTP 简单邮件传送协议` Simple Mail Transfer Protocol`

邮件读取协议:POP3 邮局协议 `Post Office Protocol`，功能有限

邮件读取协议:IMAP `Internet Message Access Protocol`，功能强大

> 有些邮件是通过网页发送的，所以也可以是 HTTP 协议
> 但是邮件服务器之间一定是 SMTP 协议

**POP3 协议：**

* 功能有限的邮件读取协议

* 用户只能从邮件服务器下载邮件到计算机

* 不允许用户在邮件服务器上管理自己的邮件

**IMAP 协议：**

* 功能比 POP3 强大的邮件读取协议

* 用户在自己的计算机上就可以操控邮件服务器中的邮箱，像本地一样

# 三、DHCP 协议
**动态主机配置协议 Dynamic Host Configuration Protocol**

**IP 地址可以手动配置，也可以动态配置，动态配置使用 DHCP 协议**

* 就是电脑或者手机自动会添加 IP 地址，不同我们自己配置

**DHCP 服务器从可用的IP 地址中，挑一个租给主机一段时间R租期到之前，主机可以提前续约，否则就会被收回**

**DHCP 协议基于 UDP**

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715926506000wnr9gk.png)


## 3.1 DHCP 协议工作原理

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715926730000l9gfri.png)

**DISCOVER 发现 DHCP 服务器**
* 需要 IP 地址的客户端会向网络中发送一个广播帧请求一个 IP 地址的租约

**OFFER 服务器给主机提供租约**
* 收到的 DHCP 服务器都会向请求的客户端发送可以配置的 IP 地址的信息
* 请求 IP 的客户端发送时，也发送了 MAC 地址，所以路由器就不用发送广播帧

**REQUEST 接受一个服务器提供的IP**
* 最先到达的租就使用了这个 IP 的租约
* 并发送一个广播帧告诉全网络，接受了这个 IP 地址的租约

**ACK 服务器确认 IP 被接受**
* 被接受租约的服务器会告诉请求方知道了接受租约

# 四、DNS 协议
**域名系统 Domain Name System**

**IP 地址不方便记忆和使用，所以有了域名(比如 `www.imooc.com`)**

**但是为了访问到目的主机，还是需要知道其IP地址，而不是域名**

**所以需要将域名解析成IP 地址，这就是 DNS 协议做的事情**

**DNS 协议一般基于 UDP**

## 4.1 域名
**……三级域名.二级域名.顶级域名**

**上级域名管理对应的下级域名**

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715927557000skcdo4.png)

## 4.2 域名服务器
* 域名和 IP 的对应关系保存在域名服务器中，供查询
* DNS 使用分布在世界各地的域名服务器来实现域名到P 的转换
* 域名服务器分为:根、顶级、权限和本地域名服务器
* 所有域名服务器都记录了根域名服务器的 IP 地址
* 上一级域名服务器记录了下一级域名服务器的 IP 地址
