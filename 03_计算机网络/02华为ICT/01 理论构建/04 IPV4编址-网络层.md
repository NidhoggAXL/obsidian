一个IP数据报文由**首部**和**数据**两部分组成。首部的前一部分是固定长度，共20字节。**首部的大小在20个字节到60个字节**。是所有IP数据报文必须具有的。在首部的固定部分的后面是一些**可选字段**，其长度可变。

# 二、首部个字段的意义
## 2.1 Version（版本）
版本占4位，指IP协议的版本。通信双方使用的IP协议的**版本必须一致**。目前广泛使用的IP协议版本号为4(即IPV4)。版本号为6(即IPv6)

## 2.2 Header Length（首部长度）
首部长度占4位，可表示的最大十进制数值是15。首部长度字段所表示的单位是32位(4字节，与TCP首部中长度字端单位一致)。因为IP首部的固定长度是20字节，因此首部长度字段的最小值为5(0101)。**当首部长度为15(1111)时，表示的长度为（15`*`4=60)字节当IP分组的首部长度不是4的整数倍时，必须利用最后的填充字段加以填充达到4的整数倍。**

## 2.3 DS Field（区分服务）
区分服务占1字节（8位），用来获得更好的服务。这个字段在旧标准中叫做服务类型，但实际上一直没有被使用过。只有在区分服务时，这个字段才起作用。在一般情况下都不使用这个字段。

## 2.4 Total Length（总长度）
总长度占2字节（16位），指首部和数据之和的长度，单位为字节。能表示的最大长度为65535字节。在IP层下面的链路层协议规定了一个数据帧的**数据字段的最大长度**，这称为**最大传输单元MTU(maximum transfer unit)**。当一个 IP 数据报封装成链路层的帧时，此数据报的总长度(即首部加上数据部分)一定不能超过下面的链路层所规定的的 MTU 值。

> **以太网**规定 MTU 为1500字节。 MTU 是**数据链路层**所规定的，用于限制链路层数据部分的长度。按照分层组包思路MTU是作用于网络层(网络层此时是链路层的数据部分)。若所传送的数据报长度超过链路层的MTU值，就必须把过长的数据进行分片处理(后面会详细讲述)。

虽然使用尽可能长的 IP 数据报会使传输效率得到提高(载荷利用率高)。但数据报短些也有好处。每个 IP 数据报越短，路由器转发的速度就越快。

## 2.5 identification（标识）
标识(identification)占2字节。网络层软件在存储器中维持一个计数器，每产生一个数据报，计数器就加1，并将此值赋给标识字段。但这个“标识”并不同于TCP首部中的序号，因为IP是无连接的服务，数据报不存在按序接收的问题。当数据报长度超过网络的MTU而必须分片时，这个标识字段的值就被复制到所有被分片报文片的标设字段中，相同的标识字段的值使分片后的各数据报片最后能正确地重装层原来的数据。

## 2.6 Flags（标志）
	标志占3位，目前只有两位有意义。

> 标志字段中间的一位记为DF(dongt fragment)，意思是“不能分片”。当DF=0时才允许分片。
> 标志字段最低位MF(more fragment)。MF=1即表示后面“还有分片"的数据报。MF=0表示这已是若干数据报片中的最后一个。


## 2.7 Fragment Offset（片偏移）
片偏移占13位。片偏移指出:较长的IP报文在分片后，某片在原分组中的相对位置。也就是说，相对于用户数据字段的起点，该片从何处开始。片偏移以8个字节为偏移单位。每片的长度一定是8字节的数倍。

> 例如:一数据报的总长度为3820字节，数据部分为3800字节(IP首部为固定20字节)，需要分片传输。假设每片IP报文长度不超过1420字节。去掉固定首部长度20字节，每片报文数据部分长度不超过1400。于是分成3个数据报片，其数据部分长度分别为1400.1400、1000字节。原始数据报首部被复制为各数据报片的首部，只需要改变有关字段的值。

## 2.8 Time To Live
生存时间占8位，英文缩写TTL(Time To Live)，表明数据报在网络中的寿命。由发出数据报的源点设置这个字段。目的是防止无法交付的数据无限制地在互联网中兜圈子。路由器在每次转发数据报之前就把TTL值减1。若TTL值减小到零，就丢弃此报文，不在转发

## 2.9 Protocol(协议)
协议占8位，协议字段指出此数据报携带的数据是使用何种协议(上面数据包中协议字段为6表示使用TCP协议)，以便使用的目的主机的IP层知道应将数据部分上交给哪个协议进行处理。

## 2.10 Header Checksum
首部检验和占16位，也常成为校验和。这个字段只检验数据报的首部，但不包括数据部分(与UDP、TCP中的检验和不同)。IP数据报每经过一个路由器，路由器都需要重新计算一下首部检验和(IP首部中的TTL、标志、片偏移等都可能发生变化)。检验和的计算方式这里就不总结了。

## 5.1 子网掩码的作用
**子网掩码**是一个 32 位的二进制数，用于将 IP 地址划分为网络地址和主机地址。它确定了 IP 地址中哪些位表示网络，哪些位表示主机。

子网掩码的作用如下：

- **划分网络和主机地址**：子网掩码将 IP 地址划分为网络地址和主机地址。网络地址标识网络，而主机地址标识网络上的特定设备。
- **确定网络大小**：子网掩码的位数决定了网络的大小。例如，一个 24 位的子网掩码表示一个具有 256 个主机地址的网络。
- **路由数据包**：路由器使用子网掩码来确定数据包应该发送到哪个网络。路由器将数据包的 IP 地址与子网掩码进行按位 AND 运算，以确定数据包属于哪个网络。
- **广播和多播**：子网掩码用于确定广播和多播地址的范围。广播地址是所有主机地址的按位 OR 运算，而多播地址是保留的 IP 地址范围，用于向网络上的特定组设备发送数据包。

**示例**

考虑以下 IP 地址和子网掩码：

- IP 地址：192.168.1.100
- 子网掩码：255.255.255.0

按位 AND 运算：
```
IP 地址:  192.168.1.100 (11000000.10101000.00000001.01100100)
子网掩码: 255.255.255.0 (11111111.11111111.11111111.00000000)
按位 AND: 192.168.1.0  （11000000.10101000.00000001.00000000)
```

按位 AND 运算的结果是网络地址 192.168.1.0。这意味着 192.168.1.100 属于网络 192.168.1.0/24。

CIDR（无类别域间路由，Classless Inter-Domain Routing）是一种用于表示和分配 IP 地址的记法。它允许将 IP 地址划分为可变长度的子网，从而更有效地利用 IP 地址空间。

CIDR 表示法包括一个 IP 地址和一个子网掩码，用斜杠（/）分隔。例如：

```
192.168.1.0/24
```

# 七、网关
![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715410659000r2u5bx.png)

类似域路由器的那个接口


网关是一个网络设备，它允许不同网络之间的通信。网关将数据包从一个网络转发到另一个网络，即使这两个网络使用不同的协议或技术。

网关通常有以下功能：

- **协议转换：** 网关可以将一种协议的数据包转换为另一种协议。例如，一个网关可以将以太网数据包转换为串行数据包。
- **地址转换：** 网关可以将一个网络中的 IP 地址转换为另一个网络中的 IP 地址。例如，一个网关可以将私有 IP 地址转换为公有 IP 地址。
- **路由：** 网关可以根据路由表将数据包转发到正确的目的地。路由表包含有关如何到达不同网络的信息。

最常见的网关类型是：

- **默认网关：** 这是连接设备到互联网的网关。它通常是路由器或调制解调器。
- **内部网关：** 这是连接不同子网的网关。它通常是交换机或路由器。
- **应用层网关：** 这是在应用层操作的网关。它可以将一种应用协议的数据包转换为另一种应用协议。例如，一个应用层网关可以将 HTTP 数据包转换为 HTTPS 数据包。

网关在网络中扮演着重要的角色，因为它允许不同网络之间的通信。没有网关，设备将无法与其他网络上的设备通信。
