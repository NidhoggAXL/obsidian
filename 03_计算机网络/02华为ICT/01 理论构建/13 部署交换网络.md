# 二、STP原理与配置
为了提高网络可靠性，交换网络中通常会使用冗余链路。然而，几余链路会给交换网络带来环路风险，并导致广播风暴以及MAC地址表不稳定等问题，进而会影响到用户的通信质量。生成树协议STP(Spanning TreeProtocol)可以在提高可靠性的同时又能避免环路带来的各种问题。

**STP操作**

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715663458000v46hpw.png)

BPDU 是 STP 协议中用来实现生成树算法的关键数据包，它包含了交换机的信息，如交换机的优先级、MAC 地址、生成树的状态等。交换机会周期性地发送 BPDU，以便与其他交换机协商生成树的结构，从而避免网络中的环路和广播风暴。

1. 选举一个根桥(Bridge lD)。
	谁为根桥是根据谁的BID=priority（优先级）- 2个字节+MAC-4个字节
	priority为2个字节就是16个bit位，默认位0，可以调试的就只有前面的4位
	priority是进行调试的，但是必须是4096的倍数，最小0，最大61440
	![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/171566516500007l3kf.png)

	![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715664924000022lgf.png)

默认为32768

优先级数值越大 优先级越小

2. 每个非根交换机选举一个根端口（root port 简写 RP）。
	根端口是离根桥最近的
3. 每个网段(链路)选举一个指定端口（designate port 简写DP）。
	PID（root bridge id）根桥ID
	cost 开销值
	bridge id 桥ID
	port id 端口ID
	非根交换机在选举指定端口时会发送BPDU分别依据根路径开销、BID、PID
	未被选举为根端口或指定端口的端口为预备端口，将会被阻塞。
4. 阻塞非根、非指定端口（alternate port）简写AP。

# 三、RSTP
STP协议虽然能够解决环路问题，但是收敛速度慢，影响了用户通信质量。如果STP网络的拓扑结构频繁变化，网络也会频繁失去连通性，从而导致用户通信频繁中断。IEEE于2001年发布的802.1w标准定义了快速生RSTP在STP基础上成树协议RSTP(Rapid Spanning-Tree Protocol)，进行了改进，实现了网络拓扑快速收敛。

**RSTP 端口角色：**

* BP 对 DP 做一个备份
* Ap 对 RP 做一个备份

RSTP中Alternate端口和Backup端口均无法转发用户流量，但可以接收、处理、发送BPDU
