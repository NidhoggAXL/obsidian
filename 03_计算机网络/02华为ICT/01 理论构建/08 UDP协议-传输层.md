# 一、UDP定义
![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1715499584000d7ns1f.png)

UDP 是一种无连接、面向报文的传输层协议。它与 TCP（传输控制协议）不同，因为它不提供可靠的数据传输或流量控制。

# 三、UDP传输过程
UDP 传输过程涉及以下步骤：

1. **应用程序**使用套接字编程接口创建 UDP 套接字并绑定到本地端口。
2. 应用程序使用 `sendto()` 函数向特定目标地址和端口发送数据报。
3. UDP 将数据报封装在一个 IP 数据报中，并将其发送到网络接口。
4. 网络接口将 IP 数据报发送到目标主机。
5. 目标主机的 UDP 模块从 IP 数据报中提取 UDP 数据报。
6. UDP 模块将 UDP 数据报传递给应用程序。
7. 应用程序使用 `recvfrom()` 函数接收数据报，该函数返回数据报的源地址和端口号。

**注意：**

- UDP 不维护连接状态，因此应用程序负责管理数据报的发送和接收。
- UDP 不提供可靠性保证，因此数据报可能丢失、乱序或损坏。
- UDP 使用端口号来标识应用程序，而不是像 TCP 那样使用套接字。

# 四、UDP特征
**特点：**

- **无连接：**UDP 不建立连接，而是直接向目的地发送数据报。
- **面向报文：**UDP 将数据分割成称为数据报的独立单元，每个数据报都包含自己的目标地址和端口号。
- **不可靠：**UDP 不保证数据报的交付、顺序或完整性。
- **低开销：**UDP 的开销很低，因为它不维护连接状态或进行流量控制。

**工作原理：**

UDP 使用套接字编程接口，应用程序可以使用该接口向特定目标地址和端口发送数据报。UDP 不维护连接状态，因此应用程序负责管理数据报的发送和接收。

**应用：**

UDP 通常用于以下应用：

- **实时应用：**例如视频流和语音通话，其中延迟比可靠性更重要。
- **广播和多播：**UDP 可用于向多个接收方发送数据，例如网络广播和多播视频流。
- **DNS 查询：**UDP 用于 DNS 查询，其中客户端向 DNS 服务器发送查询并接收响应。
- **游戏：**UDP 用于在线游戏中，其中快速响应时间至关重要。

**优点：**

- 低开销和高性能
- 适用于实时应用
- 支持广播和多播

**缺点：**

- 不可靠，不保证数据报的交付
- 没有流量控制，可能导致网络拥塞
- 需要应用程序负责数据报的管理

# 五、TCP 和 UDP 的区别：

**流量控制：**

- **TCP：**具有流量控制机制，使用滑动窗口和拥塞控制算法来防止网络拥塞。
- **UDP：**没有流量控制机制，因此不适合在拥塞的网络中使用。

**重传机制：**

- **TCP：**具有重传机制，如果数据包丢失或损坏，它会重新发送数据包。
- **UDP：**没有重传机制，如果数据包丢失或损坏，它不会重新发送数据包。

**报文封装：**

- **TCP：**将数据封装在 TCP 段中，其中包含源端口号、目标端口号、序列号、确认号和数据。
- **UDP：**将数据封装在 UDP 数据报中，其中包含源端口号、目标端口号和数据。UDP 数据报比 TCP 段更小，开销更低。

**其他区别：**

- **连接导向 vs. 无连接：**TCP 是连接导向的协议，在数据传输前需要建立连接；UDP 是无连接的协议，不需要建立连接。
- **可靠性：**TCP 是可靠的协议，保证数据包的顺序和完整性；UDP 是不可靠的协议，不保证数据包的顺序和完整性。
- **开销：**TCP 的开销比 UDP 高，因为它需要维护连接状态和执行流量控制和重传机制。
- **适用场景：**TCP 适用于需要可靠、有序数据传输的应用，例如文件传输和 Web 浏览；UDP 适用于需要低延迟、高吞吐量和容忍数据丢失的应用，例如视频流和在线游戏。

**总结：**

TCP 和 UDP 是两种不同的传输层协议，具有不同的特性和适用场景。TCP 适用于需要可靠、有序数据传输的应用，而 UDP 适用于需要低延迟、高吞吐量和容忍数据丢失的应用。
