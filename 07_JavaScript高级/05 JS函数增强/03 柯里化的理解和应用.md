# 一、柯里化概念的理解

柯里化也是属于函数式编程里面一个非常重要的概念。

* 是一种关于函数的高阶技术;
* 它不仅被用于JavaScript，还被用于其他编程语言;

维基百科的解释:

* 在计算机科学中，**柯里化**(英语:Currying)，又译**为卡瑞化或加里化**
* 是把接收 **多个参数的函数**，变成 **接受一个单一参数**(最初函数的第一个参数)的函数，并且 **返回接受余下的参数**，而且**返回结果的新函数的技术;**
* 柯里化声称“**如果你固定某些参数，你将得到接受余下参数的一个函数**”

总结：

* 只传递给函数一部分参数来调用它，让它返回一个函数去处理剩余的参数;
* 这个过程就称之为柯里化;

柯里化是一种函数的转换，将一个函数从可调用的 f(a,b,c)转换为可调用的 f(a)(b)(c)。

* 柯里化**不会调用函数**。它只是**对函数进行转换。**

# 二、柯里化代码的转换

**柯里化代码转换方式一**：每一次返回一个函数，因为JS中的函数时 [[06 JS头等函数|头等函数]]

```js
    // function foo(x, y, z) {
    //   console.log(x + y + z)
    // }

    //上面的foo进行柯里化过程变为下面的柯里化函数foo
    function foo(x) {
      return function(y) {
        return function(z) {
          //x y 使用闭包的概念，从上层作用域查找
          console.log(x + y +z)
        }
      }
    }
  
    foo(10)(20)(30)//60
```


**柯里化代码转换方式二**：使用[[04 箭头函数|箭头函数]]

```js
    // function foo(x, y, z) {
    //   console.log(x + y +z)
    // }
  
    //柯里化为柯里化函数
    function foo(x) {
      return function(y) {
        return function(z) {
          console.log(x + y +z)
        }
      }
    }

    //转为箭头函数1
    function foo(x) {
      return function(y) {
        return z => {console.log(x + y +z)}
      }
    }
  
    //转为箭头函数2
    function foo(x) {
      return y => z => {console.log(x + y +z)}
    }
  
    //转为箭头函数3
    var foo = function(x) {
      return y => z => {console.log(x + y +z)}
    }
  
    //转为箭头函数4
    var foo = x => y => z => {console.log(x + y +z)}
  
    foo(10)(20)(30)//60
```


# 三、柯里化案例练习
## 3.1 模拟日志

要求：打印日志

1. 日志的事件
2. 日志的类型：debugger、feature
3. 日志的信息

```js
  var foo = data => type => message => {
    console.log(`日期：${data} 类型：${type} 消息：${message}`)
  }
  
  //先得到确定data，参数为type的函数名
  var data = foo("2024-11-08")
  //在得到确定type，参数为message的函数民
  var typeDebugger = data("debugger")
  var typeFeature = data("feature")
  
  //debugger信息处理
  typeDebugger("界面按钮的点击")
  //日期：2024-11-08 类型：debugger 消息：界面按钮的点击
  typeDebugger()
  //日期：2024-11-08 类型：debugger 消息：undefined

  
  //feature(特征)信息处理
  typeFeature("添加音乐组件")
  //日期：2024-11-08 类型：feature 消息：添加音乐组件
  typeFeature()
  //日期：2024-11-08 类型：feature 消息：undefined
```

# 四、柯里化的优势

## 4.1 函数的职责单一

那么为什么需要有柯里化呢?

* 在函数式编程中，我们其实往往希望**一个函数处理的问题尽可能的单一**，而不是将一大堆的处理过程交给一个函数来处理,
* 那么我们是否就可以将每次传入的参数在单一的函数中进行处理，处理完后在下一个函数中再使用处理后的结果:

练习：对传入的三个参数的相加计算

* 第一个参数：`+2`
* 第二个参数：`*2`
* 第三个参数：`**2`(幂)

```js
  <script>
    function foo(x) {
      x += 2//3
      return function(y) {
        y *= 2//4
        return function(z) {
          z **= 2//9
          console.log(x + y +z)
        }
      }
    }
  
    foo(1)(2)(3)//16
  </script>
```
