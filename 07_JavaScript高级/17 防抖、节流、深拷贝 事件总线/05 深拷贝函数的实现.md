# 一、js自带的深拷贝

前面我们已经学习了对象相互赋值的一些关系，分别包括： 

* 引入的赋值：指向同一个对象，相互之间会影响； 
* 对象的浅拷贝：只是浅层的拷贝，内部引入对象时，依然会相互影响； 
* 对象的深拷贝：两个对象不再有任何关系，不会相互影响；

前面我们已经可以通过一种方法来实现深拷贝了：JSON.parse 

* 这种深拷贝的方式其实对于**函数**、**Symbol**等是无法处理的； 

```js
<script>
  const info = {
    name: "axl",
    friend: {
      age: 18
    },
    running: function() {
      console.log("running")
    },
    [Symbol()]: "haha"
  }
  
  //JSON深拷贝
  const obj1 = JSON.parse(JSON.stringify(info))
  console.log("obj1:", obj1)
</script>
```

# 二、自定义深拷贝函数

## 2.1 自定义深拷贝的基本功能；
首先要知道我们再深拷贝的时候需要判断属性值是否为一个对象：

> 因为在 JS 自带深拷贝中，如果属性值是一个对象、函数的时候就不会拷贝

但是在我们的自定义深拷贝中需要拷贝的有：函数、对象。

先来看看一些特殊值的类型：

```js
<script>
  function foo() {}
  const arr = []
  
  console.log(typeof null)//object
  console.log(typeof foo)//function
  console.log(typeof arr)//object
</script>
```

通过上面看到 null 居然也是一个 object 对象，而函数是一个 function。和我们自定义需要拷贝的要求存在差异。

> [!tip] **自定义需要拷贝应该的属性**
> null 不为 object
> function 为 object

这个时候就需要编写一个工具来改变：

* 让值为 null 的时候不进行深拷贝（false）
* 类型为 function 的时候要深拷贝 （true）

```js
<script>
  //判断对象的工具函数
  function isObject(value) {
    const valueType = typeof value
    //放回的是一个表达式
    //满足表达式放回 true， 不满足放回 false
    return (value !== null) && (valueType === "object" || valueType === "function")
  }
  
  function foo() {}
  const obj = {}
  const bar = 20
  
  console.log(isObject(null))//false
  console.log(isObject(bar))//false
  console.log(isObject(foo))//true
  console.log(isObject(obj))//true
</script>
```

自定义深拷贝基本实现函数：

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1733479106000lt7iyg.png)

```js
<script>
  //判断对象的工具函数
  function isObject(value) {
    const valueType = typeof value
    //放回的是一个表达式
    //满足表达式放回 true， 不满足放回 false
    return (value !== null) && (valueType === "object" || valueType === "function")
  }
</script>
  
<script>
  const info = {
    name: "axl",
    age: 18,
    friend: {
      name: "kobe",
      address: {
        hocla: "曲靖"
      }
    }
  }
  
  function deepObject(originValue) {
    //如果是原始类型，之间返回
    if (isObject(originValue)) {
      return originValue
    }
  
    //如果不是原始类型，则进入
    const newObj = {}
    for (const key in originValue) {
      //如果是对象的话，都是相同的操作，
      //所以使用递归
      newObj[key] = deepObject(originValue[key])
    }
    return newObj
  }

  const newObj = deepObject(info)
  console.log("newObj:", newObj)
</script>
```
