# 了解指针

> [!tip] 指针的定义
> * 指针是个变量，用来存储地址的。
> * 地址是32个或者64个二进制位（bit）组成的，（8个bit等于1个字节），所以换算过来指针变量的大小就是4个字节或者8个字节。
**总结：指针就是变量，用来存放地址的变量。(存放在指针中的值都被当成地址处理)**

**指针的大小：**
* 在32位的机器上，地址是32个或者1组成二进制序列，那地址就得用4个字节的空间来存储，所以一个指针变量的大小就应该是4个字节。
* 那如果在64位机器上，如果有64个地址线，那一个指针变量的大小是8个字节，才能存放一个地址。

**总结:**
* 指针是用来存放地址的，地址是唯一标示一块地址空间的
* 指针的大小在32位平台是4个字节，在64位平台是8个字节

**指针的关系运算(比较大小)**
```c
#include <stdio.h>
int main()
{
	char arr[] = {1,2,3,4,5};
	char* p = arr;
	int len = sizeof(arr) / sizeof(arr[0]);
	//标准编写
	for (p = &arr[5]; p > &arr[0];)
	{
		*--p = 0;
	}
	//普通编写
	for (p = &arr[4]; p >= &arr[0];p--)
	{
		*p = 0;
	}
	int i = 0;
	for (i = 0; i < len; i++)
	{
		printf("%d ", arr[i]);
	}
	return 0;
}
```

实际在绝大部分的编译器上是可以顺利完成任务的，然而我们还是应该避免这样写，因为标准并不保证它可行。

> [!tip] 标准规定
> 允许指向数组元素的指针与指向数组最后一个元素后面的那个内存位置的指针比较但是不允许与指向第一个元素之前的那个内存位置的指针进行比较。

# 野指针

> [!tip] 概念：
野指针就是指针指向的位置是不可知的(随机的、不正确的、没有明确限制的）

**野指针成因**
1. 指针未初始化
```c
#include <stdio.h>
int main()
{
	//int a;//局部变量不初始化，默认为随机值
	int *p;//指针变量未初始化，默认为随机值
	*p = 20;
	return 0;
}
```

2. 指针越界访问
```c
int main()
{
	  int arr[10] = {0};
	  int* p = arr;
	  int i = 0;
	  for(i = 0;i < 12; i++)
	  {
		  //当指针指向的范围超出数组arr的范围时，p就是野指针
		  *(p++) = i;
	  }
	  return 0;
}
```

3. 指针指向空间释放
这里放在动态的内存开辟的时候讲解，这里可以简单提示一下
```c
int* text()
{
	int a = 10;
	return &a;
}
int main()
{
	int *p = texr();
	*p = 20;
	return 0;
}
这种就是错误的，int a 是一个局部变量，当返回值&a 后这个变量就销毁了，返回给了系统，不属于这个代码系统了。

*p 中也确实是存储了 a 的地址，但是当 text 结束后这一块存储的空间就销毁了，给了系统，而后面*p = 20 也是存储进去的，但是这一快空间已经是别人的，

你进行访问是就会存在错误。
```

*如何规避野指针*
1. 指针初始化
```C
int main()
{
	int a = 10;
	int* p = &a;//初始化
	int* p = NULL;//初始化为 NULL-空指针
}
```
3. 小心指针越界
4. 指针指向空间释放 即使 置NULL(空指针)
5. 指针使用以前检查有效性

# 多级指针
```c
int main()
{
	int a = 0;
	int* pa = &a;一级指针
	int* * ppa = &pa;二级指针
	int** * pppa = &ppa;三级指针
	.....
	一次类推，就是多级指针了
	最后面的一颗* 表示是一个指针变量，而前面的侧表示指针的指向对象的类型
}
```
