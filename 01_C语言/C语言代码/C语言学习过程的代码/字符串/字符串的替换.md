```c
#include <stdio.h>
void my_strcpy(char* dest,char* src)
{
	while (*src != '\0')
	{
		*dest = *src;
		dest++;
		src++;
	}
	*dest = *src;//当\0进行替换时的情况
}
int main()
{
	char arr1[] = "###############";
	char arr2[] = "bit";
	my_strcpy(arr1, arr2);
	printf("arr1 = %s\n", arr1);
	return 0;
}
```

这里可以看到，这个函数的声明这里比我们自己创建的函数有不同的地方，这里用``char*来接受，并使进行要替换的元素指针接受前面使用了const来定义``,这里我们就要明白这个 const 在这里的作用。

const的作用是对 const 作用的元素不能进行改变。

const修饰的位置不同也存在不同的修斯情况。
1. const 放在指针变量的``*``左边时，修饰的时``*p``,也就是说；不能通过指针p来改变``*p``（num）的值。
![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1705214567000xhufiu.png)


2. const放在指针变量的``*``右边时，修饰的是指针变量p本身，p不能被改变。
![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1705214580000vo8616.png)


3. const如果在``*``两边都进行修饰的时候，这个时候``*p 和 p ``都不能被改变。

4. 当const不是修饰``*``指针变量的两边时，const修饰的可以被改变。
![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/17052145910001ur97z.png)

但是在观察strcpy的定义的时候，还是会发现和我们自己创建的函数存在不同，我们缺少放回值。

这里我们就要知道这个返回值的作用。返回值是一个指针，那返回一个指针或者说是地址有什么用呢？printf 这个函数就可以通过地址来打印替换后的内容。

所以，在优化。
```c
#include <stdio.h>
#include <assert.h>//断言的头文件
char* my_strcpy(char* dest, char* src)
{
	char* ret = dest;
	assert(dest != NULL);//断言-如果为真什么不发生，如果为假则会报错
	assert(src != NULL);//断言
	while (*dest++ = *src++)
	{//把src指向的字符拷贝到dest指向的空间，包含'\0'字符
		;
	}
	return ret;
}
int main()
{
	char arr1[] = "###############";
	char arr2[] = "bit";
	printf("arr1 = %s\n", my_strcpy(arr1, arr2));
	return 0;
}
```
