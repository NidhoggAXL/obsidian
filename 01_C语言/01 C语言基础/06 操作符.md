**&& （逻辑与-并且）**
```c

**||  （逻辑或-或者）**
```c

//打印的结果位 1 
```

**逻辑操作符的套路：
* 在且(&&)中，从左往右，只要左边有假，侧后面的都不进行运算了。
* 在或(||)中，从左往右，只要左边有真，侧后面的都不进行运算了。

**练习：**
1.360笔试题-程序输入的结果是什么？
```c
include <stdio.h>
int main()
{
	int i = 0,a = 0, b = 2,c = 3,d = 4;
	i = a++ && ++b && d++;
	//i = a++||++b||d++
	printf("a = %d\n b = %d\n c = %d\n d=%d\n",a,b,c,d);
	return 0;
}
```
结果为 1 2 3 4 ，思路：**a++是先使用后++，但是后面是&&(且)所以后面都是假，就不进行运算了。**

**变题：**
1.第一种变题。
```c
include <stdio.h>
int main()
{
	int i = 0,a = 1, b = 2,c = 3,d = 4;
	i = a++ && ++b && d++;
	//i = a++||++b||d++
	printf("a = %d\n b = %d\n c = %d\n d=%d\n",a,b,c,d);
	return 0;
}
```
结果2 3 3 5.思路：a++先使用后++即a=2，++b先++后使用即b=3，所以a++(1)&&++b(3)为真、d++先使用后++即d=5，所以a++(1)&&++b(3)&&d++(4)也为真，所以打印出来是 2 3 3 5

2.第二种变题。
```c
include <stdio.h>
int main()
{
	int i = 0,a = 1, b = 2,c = 3,d = 4;
	i = a++||++b||d++;
	printf("a = %d\n b = %d\n c = %d\n d=%d\n",a,b,c,d);
	return 0;
}
```
结果：2 2 3 4 思路：第一个||就可以判断为真了，所以后面的++b和d++都不进行运算，只运算a++。

## 表达式求值：

表达式求值的顺序一部分是由操作符的优先级和结合性决定

同样，有些表达式的操作数在求值的过程中可能需要转换为其他类型

## 隐式类型转换：

C的整型算术运算*总是至少以缺省整型类型的精度*来进行的。

为了获得这个精度，表达式中的字符和短整型操作数在使用之前被转换为普通整型，这种转换称为**整型提升**。(小于int型的都要进行转换或者提升)

### 整型提升
*整型提升的意义：*
```
表达式的整型运算要在CPU的相应运算器件内执行，CPU内整运算器(ALU)操作数的字节长度股就是int的字节长度，同时也是CPUI的通用寄存器的长度

因此，即使两个char类型的相加，在CPU执行时实际上也要先转换为CPU内整型操作数的标准长度

通用CPU(generalpurposeCPU)是难以直接实现两个8比特字节直接相加运算(虽然机器指令中可能有这种字节相加指令)。所以，表达式中各种长度可能小于int长度的整型值，都必须先转换为int或unsignedint，然后才能送入CPU去执行运算
```

```c
int main()
{
	char a = 1;
	//00000000000000000000000000000011-整型内存中的存储方式
	//00000011-a char中8个bit位-截断-调最小的8位截断
	char b = 127;
	//00000000000000000000000001111111
	//00000011-b

	//a和b如何相加
	//00000011-最高位是0
	//00000000000000000000000000000011-按最高位提升其他提升全部位0

	//00000011-最高位是0
	//00000000000000000000000001111111

	//00000000000000000000000010000010-进行相加
	//10000010-c
	//11111111111111111111111110000010-进行整型提升-补码
	//11111111111111111111111110000001-反码
	//10000000000000000000000001111110-原码
	//原码取反+1得到补码。
	char c = a + b;
	printf("%d\n",c)
}
```

b和c的值被提升为普通整型，然后在执行加法运算。

加法运算完成之后，结果将被截断，然后在存储与a中。

```
整型提升式按照变量的数据类型的符号位来提升的
如果是无符号数直接补零
```

## 操作符的属性
复杂表达式的求值有三个影响的因素。

	1.操作符的优先级
	2.操作符的结合性
	3是否控制求值顺序

两个相邻的操作符先执行哪个?取决于他们的优先级。如果两者的优先级相同，取决于他们的结合性.

操作符优先级-由高到低
```
结合性：
N/A - 无结合性
L-R - 从左到右结合
R-L - 从右到左结合

是否控制求值顺序-从左到右的执行
为真则继续执行，为假则停止执行后面的代码
```


```
