# 库函数
**为什么会有库函数？**
> * 似我们知道在我们学习C语言编程的时候，总是在一个代码编写完成之后迫不及待的想知道结果，想把这个结果打印到我们的屏幕上看看。这个时候我们会频繁的使用一个功能:将信息按照一定的格式打印到屏幕上(printf)。
> * 在编程的过程中我们会频繁的做一些字符串的拷贝工作(strcpy )
> * 在编程是我们也计算，总是会计算n的k次方这样的运算( pow)。


>[!tip] 注解
>像上面我们描述的基础功能，它们不是业务性的代码。我们在开发的过程中每个程序员都可能用的到为了支持可移植性和提高程序的效率，所以C语言的基础库中提供了一系列类似的库函数，方便程序员进行软件开发。

> [!tip] 简单的总结，C语言常用的库函数都有:
> * l0函数
> * 字符串操作函数
> * 字符操作函数
> * 内存操作函数
> * 时间/日期函数
> * 数学函数
> * 其他库函数
>> **注**:但是库函数必须知道的这里对照文档来学习上面几个库函数，目的是掌握库函数的使用方法如何学会使用库函数?

##  两个数值的交换
> [!tip] 错误代码
> ![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1702972318000ibhmgb.png)
*原因：当a和b传进swp时和里面的pa、pb的存储地址不一样，所以单pa、pb改变时无法传回到a和b来（形参无法改变实参-利用指针的话可以），所以数值没有交换。*




> [!tip] 正确代码
> （用指针来确定地址的相同，这样的话就可以返回的到a和b中去。[[函数的调用]]
> ![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1702972465000utelyw.png)


```c
#include <stdio.h>
void swp(int* pa,int* pb)
{
	int tmp = 0;
	tmp = *pa;
	*pa = *pb;
	*pb = tmp;
}
int main()
{
	int a = 10;
	int b = 20;
	printf("交换前：a=%d b=%d\n",a,b);	
	swp(&a,&b);
	printf("交换后：a=%d b=%d\n",a,b);
	return 0;
}
```




# 函数参数
> [!TIP] 实际参数（实参）：
> 真实传给函数的参数，叫实参。实参可以是:常量、变量、表达式、函数等。无论实参是何种类型的量，在进行函数调用时，它们都必须有确定的值，以便把这些值传送给形参。

> [!tip] 形式参数(形参):
> 形式参数是指函数名后括号中的变量，因为形式参数只有在函数被调用的过程中才实例化(分配内存单元)，所以叫形式参数。形式参数当函数调用完成之后就自动销毁了。因此形式参数只在函数中有效。
>>注意：
>实参传给形参的时候两个存储的地址是不相同的。当实参传给形参的时候，形参其实是实参的一份临时拷贝，形参的修改是不会改变实参的


> [!tip] 形式参数(形参):
> 形式参数是指函数名后括号中的变量，因为形式参数只有在函数被调用的过程中才实例化(分配内存单元)，所以叫形式参数。形式参数当函数调用完成之后就自动销毁了。因此形式参数只在函数中有效。
>> 注意：
> 实参传给形参的时候两个存储的地址是不相同的。当实参传给形参的时候，形参其实是实参的一份临时拷贝，形参的修改是不会改变实参的

# 函数调用
**传值调用：**
> 函数的形参和实参分别占有不同内存块，对形参的修改不会影响实参。

**传址调用：**
> 传址调用是把函数外部创建变量的内存地址传递给函数参数的一种调用函数的方式。
> 这种传参方式可以让函数和函数外边的变量建立起正真的联系，也就是函数内部可以直接操作函数外部的变量。（通过指针与地址之间的关系）[[01_C语言/函数/函数的定义和使用/自定义函数#两个数值的交换|自定义函数]]。


# 函数的嵌套调用和链式访问
## 嵌套调用
```c
#include <stdio.h>
void new_line()
{
	printf("hahha");
}
void three_line()
{
	int i = 0;
	for(i = 0;i<3;i++)
	{
		new_line;
	}
}
int main()
{
	three_line();//main函数中对three_line调用，在three_line中对new_line调用。
	return 0;
}
```

# 函数的声明和定义
> [tip] 函数的声明：
> 1. 告诉编译器有一个函数叫什么，参数是什么，返回类型是什么。但是具体是不是存在，无关紧要。
> 2. 函数的声明一般出现在函数的使用之前。要满足先声明后使用
> 3. 函数的声明一般要放在头文件中的。

```c
#include <stdio.h>
int add(int x,int y);//函数的声明
int add(int x,int y)//函数的定义
{
	int z = x + y;
}
int main()
{
	int a = 2;
	int b = 3;
	int sum = 0;
	sum = add(a,b);//函数的调用
	printf("%d\n",sum);
	return 0;
}
```

# 函数递归
**递归的主要思考方式在于:把大事化小**
> [!tip] 什么是函数的递归？
> 程序调用自身的编程技巧称为递归( recursion )。递归做为一种算法在程序设计语言中广泛应用。一个过程或函数在其定义或说明中有直接或间接调用自身的一种方法，它通常把一个大型复杂的问题层层转化为一个与原问题相似的规模较小的问题来求解，递归策略只需少量的程序就可描述出解题过程所需要的多次重复计算，大大地减少了程序的代码量。

> [!tip] 递归的两个必要条件(注意不是充要条件)
>存在限制条件，当满足这个限制条件的时候，递归便不再继续。
>每次递归调用之后越来越接近这个限制条件。

```c
例如：
int fib(int n)
{
	if (n < 100000)
	{
		fib(n + 1);
	}
}
int main()
{
	fib(1);
	return 0;
}
虽然满足了条件，当时也会栈溢出。
```
