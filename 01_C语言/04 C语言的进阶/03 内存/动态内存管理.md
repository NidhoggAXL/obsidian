# malloc和free

> [!tip] malloc 定义
> ![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1708936439000tegeuj.png)


这个函数向内存申请一块连续可用的空间，并返回指向这块空间的指针。
* 如果开辟成功，则返回一个指向开辟好空间的指针。
* 如果开辟成功，则返回一个指向开辟好空间的指针。
* 返回值的类型是void*，所以malloc函数并不知道开辟空间的类型，具体在使用的时候使用者自己来决定。
* 如果参数size为0 ,malloc的行为是标准是未定义的，取决于编译器。

> [!tip] free 定义
> ![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1708936567000d9n54z.png)

free函数用来**释放动态开辟**的内存。
* 如果参数ptr指向的空间不是动态开辟的，那free函数的行为是未定义的。
* 如果参数ptr是NULL指针，则函数什么事都不做。

malloc和free的使用：
```C
#define _CRT_SECURE_NO_WARNINGS 1
#include <stdio.h>
#include <stdlib.h>
#include <errno.h>

int main()
{
	int* p = (int*)malloc(10 * sizeof(int));
	//malloc返回类型是void* 这里需要的是一个int* 类型的空间
	//所以强制类型转换为int*类型
	int i;
	if (p == NULL)
	{
		//打印错误原因的一个方式
		printf("%s\n", strerror(errno));
	}
	else
	{
		//正常使用空间
		for (i = 0; i < 10; i++)
		{
			*(p + i) = i;
			printf("%d ", *(p + i));
		}
	}
	free(p);//释放掉p指针所指向的空间
	p = NULL;
	//空间释放了，但是指针还存在。
	//如果不滞空的话，就会产生野指针
	return 0;
}
```
#strerror #errno [[野指针]]

# calloc
> [!tip] 定义
> ![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1708938647000zaj2go.png)
> * 函数的功能是为num个大小为size的元素开辟一块空间，并且把空间的每个字节初始化为0。
> * 与函数malloc的区别只在于 ca1loc 会在返回地址之前把申请的空间的每个字节初始化为全0.

```C
int main()
{
	int* p = (int*)calloc(10, sizeof(int));
	if (p == NULL)
	{
		printf("%s\n", strerror(errno));
	}
	int i;
	for (i = 0; i < 10; i++)
	{
		printf("%d ", *(p + i));
	}
	free(p);
	p = NULL;
	return 0;
}
```

# realloc
> [!tip] 定义
> ![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1708939390000z7sz04.png)
>  * 调整动态内存空间的大小
> 	* realloc函数的出现让动态内存管理更加灵活。
> 	* 有时会我们发现过去申请的空间太小了，有时候我们又会觉得申请的空间过大了，那为了合理的时候内存，我们一定会对内存的大小做灵活的调整。那realloc函数就可以做到对动态开辟内存大小的调整。函数原型如下:
> *  ptr是要调整的内存地址
> * size调整之后新大小
> * 返回值为调整之后的内存起始位置。
> * 这个函数调整原内存空间大小的基础上，还会将原来内存中的数据移动到新的空间。

realloc使用的注意事项：
1. 如果指针p指向malloc开辟的空间之后有足够追加的空间，则直接追加，返回指针p。
2. 如果指针p指向的空间之后没有足够的内存空间可以追加,则realloc函数会重新找一个新的内存区域开辟一块满足需求的空间，并且把原来内存中的数据拷贝回来，释放旧的内存空间，最后返回新开辟的内存空间地址。
3. 得用一个新的变量来接受realloc函数的返回值。因为如果追加失败返回的是一个空指针，这样原本的空间也就找不到了。

realloc和malloc相同的功能
```C
int* p = realloc(NULL,40);
也可以创建动态内存
```

#  常见错误
## 对非动态内存开辟内存的free
```C
int main()
{
	int a = 10;
	int* p = &a;
	*p = 20;//err
	free(p);
	p = NULL;
	return 0;
}
```
a是局部变量，是在栈上面开辟的内存空间。
free只能释放开辟在堆上面的动态内存。

## 使用free释放动态内存的一部分
```C
int main()
{
	int* p = (int*)malloc(40);
	if(p == NULL)
	{
		return 0;
	}
	int i;
	for(i = 0; i < 10 ; i++ )
	{
		*p++=i;
	}
	//回收1空间
	free(p);
	p = NULL;
	return 0;
}
```
p指针的位置已经发生了改变，这时候的p指针不在是指向整个动态内存，
free(p)释放的不在是全部的动态内存。

# 经典动态内存题目
## 动态内存的传参
**运行下面代码会有什么结果？**
```C
void GetMemory(char* p)
{
	p = (char*)malloc(100);
}

void Text(void)
{
	char* str = NULL;
	GetMemory(str);
	strcpy(str, "hello world");
	printf(str);
}

int main()
{
	Text();
	return 0;
}
```
上面代码运行后，会奔溃。
又malloc那就应该又free释放内存，这样会存在内存泄漏的问题。

1. 改正1：
```C
void GetMemory(char** p)
{
	*p = (char*)malloc(100);
}

void Text(void)
{
	char* str = NULL;
	GetMemory(&str);
	strcpy(str, "hello world");
	printf(str);
}

int main()
{
	Text();
	return 0;
}
```
要改变指针str，那么就要传&str一级指针的地址，那么形参就要用二级指针来接收，这样一级指针的内容才会改变。

2. 改正2：
```C
char* GetMemory(char p)
{
	p = (char*)malloc(100);
	return p;
}

void Text(void)
{
	char* str = NULL;
	str = GetMemory(str);
	strcpy(str, "hello world");
	printf(str);
	free(str);
	str = NULL;
}

int main()
{
	Text();
	return 0;
}
```
这种是直接把形参的地址返回过来，在赋值给指针，这样指针的内容就是形参指向的内容。

**运行下面代码会有什么结果？**
```C
char* GetMemory(void)
{
	char p[] = "hello world";
	return p;
}
void Text(void)
{
	char* str = NULL;
	str = GetMemory();
	printf(str);
}

int main()
{
	Text();
	return 0;
}
```
会打印出来随机值。
因为p内容是（hello worlf）是一个栈空间的地址，是一个临时变量。当GetMemory函数运行结束的时候，p内容也就销毁了。后面str在拿到p的地址，这时候也不知道p的内容是什么了。
这是一个经典的返回**栈空间地址的问题**

**运行下面代码会有什么结果？**
```C
void Text()
{
	char* p = (char*)malloc(100);
	strcpy(p, "hello");
	free(p);
	if (p != NULL)
	{
		strcpy(p, "world");
		printf(p);
	}
}

int main()
{
	Text();
	return 0;
}
```
代码也是会奔溃，这里提前释放了p指向的空间，p变成了野指针。但是p的地址还在，在后面使用的时候会进入if语句里面，这时候在strcpy的话，空间已经被释放了，在使用的话就会非法访问内存空间。

# C/C++程序内存开辟
![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1709102000000o8h14u.png)

C/C++程序内存分配的几个区域：
> 1. 栈区（stack );在执行函数时，函数内局部变量的存储单元都可以在栈上创建，函数执行结束时这些存储单元自动被释放。栈内存分配运算内置于处理器的指令集中，效率很高，但是分配的内存容量有限。栈区主要存放运行函数而分配的局部变量、函数参数、返回数据、返回地址等。
> 2. 堆区 ( heap):一般由程序员分配释放，若程序员不释放，程序结束时可能由OS回收，分配方式类似于链表。
> 3. 数据段(静态区）( static)存放全局变量、静态数据。程序结束后由系统释放。
> 4. 代码段:存放函数体（类成员函数和全局函数)的二进制代码。
> 

# 柔性数组
> [!tip] 定义
> C99中，结构中的最后一个允许是未知大小的数组，这就叫做**柔性数组成员**

**例如：**
```C
struct S
{
	int n;
	int arr[];
	//或者 int arr[0];
	//未知大小的柔性数组成员，数组的大小可以调整
};

int main()
{
	struct S* ps = (int*)malloc(sizeof(struct S) + 5 * sizeof(int));
	//对柔性数组进行扩展。
	if (ps != NULL)
	{
		ps->n = 100;
	}
	int i;
	for (i = 0; i < 5; i++)
	{
		ps->arr[i] = i;
		printf("%d ", ps->arr[i]);
	}
	struct S* str = realloc(ps, 44);//再次扩展
	if (str != NULL)
	{
		ps = str;
	}
	for (i = 5; i < 10; i++)
	{
		ps->arr[i] = i;
		printf("%d ", ps->arr[i]);
	}
	return 0;
}
```

可以看到柔性数组在没有扩展前不建立大小。

**柔性数组的特点：**
> * 结构中的柔性数组成员前面必须至少一个其他成员。
> * sizeof返回的这种结构大小不包括柔性数组的内存。
> * 包含柔性数组成员的结构用malloc ()函数进行内存的动态分配，并且分配的内存应该大于结构的大小，以适应柔性数组的预期大小。
