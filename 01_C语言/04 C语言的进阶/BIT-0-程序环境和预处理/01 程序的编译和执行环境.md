在ANSIC的任何一种实现中，存在两个不同的环境。
> 第1种是翻译环境，在这个环境中源代码被转换为可执行(.exe)的机器指令。放入的是二进制信息（二进制文件）
> > .c(C代码-文本文件)-.exe
> 第2种是执行环境，它用于实际执行代码。
> .obj-object(目标)

#  编译+链接
## 1 编译环境
![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1709623188000zmo63y.png)

> 每个源文件(.c)都会生成一个目标文件(.obj)
> 组成一个程序的每个源文件通过编译过程分别转换成目标代码(（object code )。·
> 每个目标文件由链接器（linker ）捆绑在一起，形成一个单一而完整的可执行程序。
> 链接器同时也会引入标准C函数库中任何被该程序所用到的函数，而且它可以搜索程序员个人的程序库，将其需要的函数也链接到程序中。

**把目标文件(.obj或者.0)链接成可执行程序(.exe)：**
1. 合并段表：其中.boj会被分成几段的elf格式，当不同的.obj文件在链接的过程中，会把不同的.obj文件里面的段进行链接，合并成一个elf格式的文件。
2. 符号表的合并和重定位：不同的符号表进行合并，相同的符号以有意义的为准。(声明函数的符号表是没有意义的)

# 运行环境
1. 程序必须载入内存中。在有操作系统的环境中:一般这个由操作系统完成。在独立的环境中，程序的载入必须由手工安排，也可能是通过可执行代码置入只读内存来完成。
2. 程序的执行便开始。接着便调用main函数。
3. 开始执行程序代码。这个时候程序将使用一个运行时堆栈(stack)，存储函数的局部变量和返回地址。程序同时也可以使用静态(static)内存，存储于静态内存中的变量在程序的整个执行过程一直保留他们的值。
4. 终止程序。正常终止main函数;也有可能是意外终止。
