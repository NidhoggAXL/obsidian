# 直接插入排序
![[插入排序结构流程示意图]]
end移动了4次，比数据个数少一。while循环结束的情况时找到了要插入的地方break跳出while循环，或者时当end=-1的时候，这个时候里面的数都比要插入的数据大，插入的数据只能插入到 a[0]的位子-end+1。
for实在控制end结束的位置。

```c
升序
void InsertSort(int* a ,int n)//n代表的是元素个数，不是下标
{
	for (int i = 0; i < n - 1; i++)
	{
		int end=i;
		int tmp = a[end + 1];
		while (end >= 0)	
		{
			if (a[end] >tmp)
			{
				a[end + 1] = a[end];
				--end;
			}
			else
			{
				break;
			}
		}
		a[end + 1] = tmp;
	}
}
```

>[!tip] 时间复杂度为O(n^2)
>什么情况下最坏？ 逆序 1+2+...+n-1  等差数列
>什么情况下最好?  顺序

# 希尔排序(缩小增量排序)
直接插入排序的基础上面优化：
1. 先进行预排序，让数组接近有序
2. 直接插入排序
3. 间隔为gap是一组

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/17032354830008g5gkk.png)

```C
void ShellSort(int* a, int n)//希尔排序
{
	int gap = n;
	while (gap > 1)
	{
		gap /= 2;
		//把间隔为gap的多组数据同时排
		for (int i = 0; i < n - gap; i++)
		{
			int end = i;
			int tmp = a[end + gap];
			while (end >= 0)
			{
				if (a[end] > tmp)
				{
					a[end+gap]=a[end];
					end -= gap;
				}
				else
				{
					break;
				}
			}
			a[end + gap] = tmp;
		}
	}
}
```

>[!tip] gap
>1. gap大于1都是预排序，接近有序
>2. gap等于1就是直接插入排序
>3. gap没有专门的固定是多少，直接确定一个值不准确。
>4. gap常见的都是使用gap/2或者gap/3+1,因为要确定最后的gap一定要是1

>[!tip] 时间复杂度
>1. whil的时间复杂度是O(log以2为底n的对数)
>2. foe的时间复杂度和gap有关
>	1. gap很大时，下面排序的时间复杂度时O(n)
>	2. gap很小时，数组已经接近有序，这时差不多也是O(n)
>3. 总的时间复杂度为while*for的时间复杂度相乘
>4. 平均时间复杂度是O(n^1.3)
