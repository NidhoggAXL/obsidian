# 一、代码的页面跳转

有时候希望通过代码来完成页面的跳转，比如点击的是一个按钮：

在 opsition API 里面使用的话：

```js
methods: {
	WebSwitch() {
		this.$router.push('/home')
	}
}
```

## ## useRouter

**作用**：获取路由器实例，用于编程式导航

**返回值**：返回 `Router` 实例，提供导航方法

**常用方法**：

- `push()` - 导航到新路由，添加历史记录
    
- `replace()` - 替换当前路由，不添加历史记录
    
- `go()` - 在历史记录中前进或后退
    
- `back()` - 返回上一页
    
- `forward()` - 前进一页

![gh|500](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/174774577200015h92i.png)

当然，我们也可以传入一个对象：

```js
const router = useRouter()
function WebSwitch() {
  router.push({
    path: '/home',
  })
}
```

## useRoute和useRouter区别

#useRoute和useRouter

| 特性           | useRoute      | useRouter      |
| ------------ | ------------- | -------------- |
| **作用**       | 获取当前路由信息      | 进行编程式导航        |
| **返回值**      | 响应式 Route 对象  | Router 实例      |
| **主要用途**     | 读取路由参数、查询参数等  | 跳转路由、操作历史记录    |
| **数据特性**     | 只读、响应式        | 可操作、包含方法       |
| **类比 Vue 2** | `this.$route` | `this.$router` |

**简单记忆**：

- `useRoute` - 用于**读取**当前路由信息
    
- `useRouter` - 用于**操作**路由导航

# 二、query方式的参数

也可以通过query的方式来传递参数：

```js
function WebSwitch() {
  router.push({
    path: '/home',
    //使用query传递参数
    query: {
      id: 123,
      name: 'zhangsan'
    }
  })
}
```

---

```url
http://localhost:8080/#/home/product?id=123&name=zhangsan
```

> [!abstract]
> 
> 也可以不使用 path 路径来路由，可以使用 name 来进行路由`name: 'home'`,但是这样的话就不会有上面这种 URL 来显示了，<mark class="hltr-orange">开发中一般使用 path 来进行路由</mark>

在跳转目标的界面中通过 $route.query 来获取参数：

```html
<h2>query: {{ $route.query.name }} - {{ $route.query.id }}</h2>
```

# 三、替换当前的位置

使用push的特点是压入一个新的页面，那么在用户点击返回时，上一个页面还可以回退，但是如果希望当前页面是一个替换操作，那么可以使用replace：

| 声明式                               | 编程式                   |
| --------------------------------- | --------------------- |
| `<router-link :to="..." replace>` | `router.replace(...)` |
