# 一、为什么要动态管理路由

在一个网站里面（大概率是一个后台管理系统的网站），如果存在不同身份（销售、管理、超级用户），这个时候希望给不同身份展示不同的路由：

 **方式一**：就是把全部的路由都在 createRoute 的时候进行注册，在根据不同身份显示不同路由

* 缺点就是一些人可以通过这个推断出不同身份的网页，这样就可以显示出来一些组件，这是一件非常危险的事情。
* 给一些技术人员就可以黑进后台

**方式二**：先让不同身份的进行一个登录进入后台，在更具身份来注册不同的路由，这就是<mark class="hltr-cyan">动态路由</mark>

> [!tip] 区别
> 
> **方式一**：是先全部路由注册，在根据身份显示路由
> **方式二**：是先登录身份，在更具身份进行路由的注册，在显示路由

# 二、动态添加路由

某些情况下可能需要动态的来添加路由： 

* 比如根据用户不同的权限，注册不同的路由； 
* 这个时候我们可以使用一个方法 **addRoute**；

如果我们是为router添加一个动态路由，并且模拟管理员进行验证登录

* 如果是管理员就添加 Admin 路由
* 如果不是则反之

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1747750684000unq6ks.png)

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1747750708000coaij9.png)

如果现在是管理员是没有登录:`const isAdmin = false`,就没有添加 Admin 路由了。

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1747750770000dc45ae.png)

当然路由的添加也可以使用父级的 name 来进行进行路由的嵌套：

* 在 home 的路由中添加 moment 路由
* URL格式：`http://localhost:8080/#/home/moment`

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/17477508990007fil0e.png)
