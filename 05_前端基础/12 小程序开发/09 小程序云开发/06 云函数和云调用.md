# 一、什么是云函数
**云函数即在云端（服务器端）运行的函数**： 

- 在物理设计上，一个云函数可由多个文件组成，占用一定量的CPU 内存等计算资源； 
- <mark class="hltr-orange">各云函数完全独立，可分别部署在不同的地区</mark>； 
- 开发者无需购买、搭建服务器，只需编写函数代码并部署到云端即可在小程序端调用； 
- 同时<mark class="hltr-orange">云函数之间也可互相调用</mark>；

**云函数的编写方式：** 

- 一个云函数的写法与一个在本地定义的 JavaScript 方法无异，代码**运行在云端 Node.js 中**
- 当云函数被小程序端调用时，定义的代码会被放在Node.js 运行环境中执行； 
- 我们可以如在 Node.js 环境中使用 JavaScript 一样<mark class="hltr-orange">在云函数中进行网络请求</mark>等操作，而且我们还可以通过云函数后端 SDK 搭配使用多种服务，比如使用云函数 SDK 中提供的<mark class="hltr-orange">数据库和存储 API 进行数据库和存储的操作</mark>

**云开发的云函数的独特优势在于与微信登录鉴权的无缝整合：**

- 当小程序端调用云函数时，云函数的传入参数中会被注入小程序端用户的 openid，开发者无需校验 openid 的正确性因为微信已经完成了这部分鉴权，开发者可以直接使用该 openid。

# 三、云函数的调试

云函数的调试有：本地调试、云端调试

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1754620767000cfk5kp.png)

云函数代码：

```js
// 云函数入口文件
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV }) // 使用当前云环境

// 云函数入口函数
exports.main = async (event, context) => {
  console.log(event.sum1, event.sum2);
  return event.sum1 + event.sum2
}
```

调用云函数：

```js
// pages/cloud-function/cloud-function.js
Page({
  cloud01() {
    wx.cloud.callFunction({
      name: 'sum',
      data: {
        sum1: 10,
        sum2: 20
      }
    }).then(res => {
      console.log(res.result);
    })
  }
})
```

# 六、发送http请求

云函数中支持对其他服务器进行http请求，也支持使用axios库发生网络请求：

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/175463398100064v9zd.png)

> [!info] 注意事项：
> - 因为每一个云函数是想好独立的，所以比如在使用第三方库的时候要进行下载
> 	- 需要**进入到云函数当前自己的文件夹里面**
> - 使用第三方库，会在自身的云函数创建 node modlue 文件夹
> 	![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1754634222000nsc3mf.png)
> - 所以对于**小程序某些域名的限制**无法配置时，我们可以通过云函数作为代理来请求数据，再返回给小程序端；
> 	- 就是域名必须备案的，在微信本地进行开发的时候就不可以校验地址合法性
