useTransition到底是干嘛的呢？它其实在告诉**react对于某部分任务的更新优先级较低，可以稍后进行更新。**

- 放回值一：判断后面第二个返回值里面的回调函数是否执行完成
- 放回值二：稍后跟学的函数，里面放入 calback

当fakerNames里面有大量的数据的时候，filter 过滤需要花费大量的事件，才会重新渲染页面。

当用户输入或者删除一个字符的时候，进行过滤，过滤完才会重新渲染页面，但是在过滤的这个事件段，用户输入的东西就会看不到，因为要等过滤完才会渲染，这样是给用户一个不好的体验的。

这个时候就使用 useTransition ，让页面先渲染，并且通过 pending ，如果为 true 表示还没有把 use Transition 里面代码执行完，false 则表示执行完啦。通过这样一个机制，可以给用户添加一些好的提示，比如上面 `{isPending && <h3>loading</h3>`
