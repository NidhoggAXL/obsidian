# 一、解析命令

```shell
xlcli --version
```

这个命令只会把js执行一遍，并不会有对应的版本，那么就需要去解析这个后面的命令，
可以通过 process.argv 来获取，但是如果要自己进行解析这是一个麻烦的过程。

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1773646572000f90gzu.png)

这个时候就可以使用一个解析的工具 commander 来解析：

```shell
npm install commander
```

```js title = "./lib/index.js"
#!/usr/bin/env node
const { program } = require("commander")

// 处理 --version -v 的操作
const version = require("../package.json").version
program.version(version, "-v --version")

// 让commonder解析process.argv参数
program.parse(process.argv)
```

# 二、解析命令参数

```js
#!/usr/bin/env node
const { program } = require("commander")

// 处理 --version -v 的操作
const version = require("../package.json").version
program.version(version, "-v --version")

// 增加其他 Options 选项
program.option("-a --axl", "a axl cli program~")
program.option("-d --dest <dest>", "a destination folder, 例如: -d src/components")


// 让commonder解析process.argv参数
program.parse(process.argv)

// 获取额外传递的参数
console.log(program.opts().dest)
```
