使用 npm 创建项目;

```shell
npm init
```

在目录下面创建一个js文件：

```js title = "./lib/index.js"
#!/usr/bin/env node

const message = "axl"
console.log(message)
```

> [!tip] `#!/usr/bin/env node` 的作用：
> 在使用这个入口文件的时候，会读取这个注释，知道是使用 node 来执行的。

在 package.json 的文件里面配置，让命令行可以使用：

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/17736458710002n1h5c.png)

但是在命令行使用，却发现会出现确实，这样因为没有配置环境，命令行还并不知道这个命令，命令行还没有和这个命令建立连接。

建立连接：**这样本地电脑就会建立环境变量，在其他文件中也可以使用。**

```shell
npm link
```

命令行使用命令：

```shell
xlcli
axl
```

> [!tip] 
> 如果是发送到 npm 仓库的话，并不需要手动建立连接，当别人`npm install xlcli`的时候就会自动和命令行建立连接。
