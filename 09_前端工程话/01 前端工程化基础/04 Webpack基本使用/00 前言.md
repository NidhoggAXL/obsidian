# 二、path常见的API

从路径中获取信息 

* dirname：获取文件的父文件夹； 
* basename：获取文件名； 
* extname：获取文件扩展名

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/17442013950009u7ti1.png)


**路径的拼接：path.join** 

* 如果我们希望将多个路径进行拼接，但是不同的操作系统可能使用的是不同的分隔符； 
* 这个时候我们可以使用path.join函数；
* 拼接的是一个相对路径

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1744201531000flttox.png)


**拼接绝对路径：path.resolve**  ^b867a8

* path.resolve() 方法会把一个路径或路径片段的序列解析为一个绝对路径； 
* 给定的路径的序列是从右往左被处理的，后面每个 path 被依次解析，**直到构造完成一个绝对路径**； 
* 如果在处理完所有给定path的段之后，还没有生成绝对路径，则使用当前工作目录； 

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1744201608000diotg8.png)

* 生成的路径被规范化并删除尾部斜杠，零长度path段被忽略； 

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1744201664000pe9hpy.png)

* 如果没有path传递段，`path.resolve()`将返回当前工作目录的绝对路径；

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1744201741000isz2bl.png)

# 三、webpack中使用

在webpack中获取路径或者起别名的地方也可以使用：

[[06 Node的全局对象#二、特殊的全局对象|__dirname]] 是 Node.js 中的一个**全局变量**，表示**当前模块所在目录的绝对路径**（与 `path.dirname` 效果一致）。

在你提供的 `craco.config.js` 中，
`const resolve = dir => path.resolve(__dirname, dir);` 

这行代码的作用是：通过 `__dirname` 获取当前配置文件所在目录的绝对路径，再与传入的 `dir`（如 `"src"`）拼接，生成一个可靠的绝对路径（例如若配置文件在 `/project/config`，则 `resolve("src")` 会得到 `/project/config/src`）。

这种写法常用于 Node.js 中处理文件路径，确保无论脚本在何处执行，都能正确定位到目标目录。
