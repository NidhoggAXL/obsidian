# 二、webpack-dev-server

上面的方式可以监听到文件的变化，但是事实上它本身是没有自动刷新浏览器的功能的： 

* 当然，目前我们可以在VSCode中使用live-server来完成这样的功能； 
* 但是，我们希望在不适用live-server的情况下，可以具备live reloading（实时重新加载）的功能；

安装webpack-dev-server

```bash
npm install webpack-dev-server -D
```

修改 **package.json** 配置文件，启动时加上serve参数：

![gh|500](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1744723220000ew0ka9.png)

运行就会开始搭建本地服务：

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1744723292000akkr39.png)

但是当我们点击链接的时候，页面会发出警告：

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1744723370000lk4axe.png)

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1744723392000k7zzbc.png)

这个时候我们需要设置一下 [[05 mode模式配置|mode]]

![gh|600](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/17447235390005qhxy1.png)

这个时候在进行 `npm run serve` 就可以在页面中正常显示，并可以随时刷新页面了

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/174472364900073qcai.png)

> [!tip] 重点
> webpack-dev-server 在编译之后不会写入到任何输出文件，而是**将 bundle 文件保留在内存中**： 
>事实上webpack-dev-server使用了一个库叫memfs（memory-fs webpack自己写的）
