# 一、FC
全称：formatting context - 格式化上下文

**官网文档中对FC的解释如下：**

> Boxes in the normal flow belong to a formatting context, which may be block or inline, but not bothsimultaneously, Block-level boxes participate in a block formatting context, inline-level boxes participate in aninline formatting context.

**翻译**：在标准流中，所有盒子属于格式化上下文，可以是块级或内联，但不能同时是两者。块级盒子参与块格式化上下文。内联级盒子参与内联格式化上下文。

块级元素的布局属于Block Formatting Context(BFC)

* 也就是**block level box**都是在**BFC**中布局的;

行内级元素的布局属于Inline Formatting Context(IFC)

* 而**inline level box**都是在**IFC**中布局的;

# 二、BFC

全称：block level box - 块格式化上下文

那么 BFC 到底是在那里呢？

**MDN上有整理出在哪些具体的情况下会创建BFC:**

* 根元素`(<html>)`
	* html里面有body，body里面有一个默认div
	* 这个时候 body 和 div 都在 html 的BFC里面，但是 div 不在 body 的 BFC 里面，因为 body 就不是一个 BFC
* 浮动元系(元系的 foat 不是 none)
* 绝对定位元系(元系的position为absolute 或 fixed)
* 行内块元素(元素的 display为inline-block)
* 表格单元格(元系的 display 为 table-ce，HTML表格单元格默认为该值)，表格标题(元索的 display为 table-caption，HTML表格标题默认为该值)
* 匿名表格单元格元素(元素的 display为 table、table-row、table-row-group、table-header-group、table-footer-group(分别是HTML table.row、tbody、thead、tfoot的默认属性)或inline-table)
* overflow 计算值(Computed)不为 visible 的块元素
* 弹性元系(display为fex 或 inline-flex 元系的直接子元系)
* 网格元素(display 为 grid 或inline-grid 元系的直接子元素)
* display 值为 flow-root 的元系
* 等等

## 2.1 BFC的作用

**官方文档的概述：**

> In a block formating context, boxes are laid out one after the other, vertically,beginning at the top of a containing block. The vertical distancebetween two sibling boxes is determined by the 'margin properties. Vertical margins between adjacent block-level boxes in a block formattingcontext collapse.
> 
> In a block formatting context, each box's left outer edge touches the left edge of the containing block (for right-to-left formating, right edgestouch). This is true even in the presence of floats (although a box's line boxes may shrink due to the floats), unless the box establishes a new blockformatting context (in which case the box itself maybecome narrower due to the floats).

**简单概述如下：**

* 在BFC中，box会在垂直方向上一个挨着一个的排布:
* 垂直方向的间距由margin属性决定:
* 在同一个BFC中，相邻两个box之间的margin会折叠(collapse)
	* 这就是为什么[[12 CSS的盒子模型（Box Model）#6.4 margin上下的折叠（collapse）|margin上下的折叠（collapse）]]的根本原因
* 在BFC中，每个元素的**左边缘是紧挨着包含块的左边缘的;**

> [!tip] 注意：
> 我们在页面里面布局起作用时，可以说时浏览器进行布局，但是根本应该说时浏览器里面的 BFC 来进行布局的控制。

## 2.3 BFC 解决浮动高度塌陷

**网上有很多说法，BFC可以解决浮动高度塌陷，可以实现清除浮动的效果。**

* 但是从来没有给出过BFC可以解决高度塌陷的原理或者权威的文档说明，
* 他们也压根没有办法解释，为什么可以解决浮动高度的場陷问题，但是不能解决绝对定位元素的高度塌陷问题呢?

**事实上，BFC解决高度塌陷需要满足两个条件:**

* 浮动元素的父元素触发BFC，形成独立的块级格式化上下文(BlockFormatting Context);
* 浮动元素的父元素的高度是auto的;

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1726382278000rtxse3.png)

**BFC的高度是auto的情况下，是如下方法计算高度的**

1. 如果只有inline-level，是行高的顶部和底部的距离:口 2.如果有block-level，是由最底层的块上边缘和最底层块盒子的下边缘之间的距离
2. 如果有绝对定位元素，将被忽略;
3. 如果有浮动元素，**那么会增加高度以包括这些浮动元素的下边缘**
