当我们没有给父级元素设置高度，是给子集元素设置高度把父级元素撑起来的时候，**其本质还是父级元素没有高度的**。

有关于父级元素高度的一些属性设置的时候就会出现错误。

```css
<style>
	.box {
		display: flex;
		width: 800px;
		margin: 50px auto;
		background-color: darkcyan;
	}
	.a {
		height: 400px;
		width: 300px;
		background-color: gray;
	}
	.b {
		position: relative;
		top: 50%;
		transform: translate(0, -50%);
		height: 100px;
		width: 200px;
		background-color: red;
	}
</style>

<div class="box">
	<div class="a"></div>
	<div class="b"></div>
</div>
```

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1725590293000qz8rne.png)

明明我们对 b 设置了 `top:50%; transform:translate(0, -50%);` 按照正常应该是会 b 先相对与父级元素 box 盒子里面向下移动 50% 在向上移动自生的 50% 但是这里我可以看到，**只执行了第二步，并没有执行第一步**。

> [!tip] 原因
> box 盒子并没有设置过高度，它的高度其实是里面的 a 撑起来啦。这样的高度在后面对子元素 b 设置 `top: 50%;` 并没有生效， box 本身是没有高度的，所以无效。
> 上面例子中对 box 设置高度这个问题就解决啦。
