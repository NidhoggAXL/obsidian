块级元素里面包含一张 img 会存在间隙

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1724246696000njmiuh.png)

p 与 x 中 x 的底部就是基线，里面的 img 是一个行内级元素，默认情况下就是基线对齐的方式。

解决的方法，对 img 进行如下设置：

```css
img {
	 /* 任意选下面其中一个就可以 */
	vertical-align: bottom;
	vertical-align: top;
	display: block;
}
```

当然就算设置了，也还是会存在一小点的间隙的，网易云是通过背景图片是添加图片本身的高斯模糊的图片来让哪一点小间隙看不到。

**如果是非要去除哪一点间隙：**

* 如果父级元素有固定高度，那么就对 img 本身的高度设置一个和父级元素同样的高度就好啦。
* 如果父级元素没有高度，那么就对父级元素设置一个和图片高度一样的高度就好啦。
* **虽然这样可以解决间隙，但是如果是图片和文字在一行的话会感觉到文字会超出包含块（其实是基线对齐）**

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1725448112000evbw15.png)


> [!tip] 自我总结（有待考证）
> 包含快里面只要包含了行内级元素、行内替换元素、line-block 元素就会在底部存在间隙。
> 
> **为什么产生这种间隙？**
> 
> 行内级非替换元素、行内替换元素、line-block 元素最早的使用都是作用与文字或者说字母的，都会存在基线，所以会基线对齐，这样就会在底部存在一定的间隙。
> 
> **img 元素就可以看作一个 line-block 元素。**
> 
> **那么这个间隙会计算在包含块的高度上吗**？
> 
> * 在没有对包含块设置高度的时候，包含块的高度为：包含内容的高度+间隙的高度
> * 在对包含块设置高度的时候，包含块的高度为：包含内容的高度
