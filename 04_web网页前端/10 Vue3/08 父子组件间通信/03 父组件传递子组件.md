# 二、props的字符串数组用法

在 App.vue 向 ShowInfo.vue 里面传入数据:

```html
<template>
    <!-- 传入数据 -->
 <show-info name="axl" age="18"></show-info>
 <show-info name="koube" age="30"></show-info>
</template>

<script>
import ShowInfo from './ShowInfo'
export default {
 components: {
  ShowInfo
 }
}
</script>

<style scoped>

</style>
```

在 ShowInfo.vue 接收数据:

```html
<template>
 <div class="infos">
  <h2>姓名: {{ name }}</h2>
  <h2>年龄: {{ age }}</h2>
 </div>
</template>

<script>
export default {
 //通过props自定义attribute来接收父组件传入的参数
  props:["name", "age"]
 }
</script>

<style scoped>

</style>
```


上面的传递也可以是传递一个数组，再在子组件里面使用数组来接收：

```html
<template>
 <div class="app">
  <!-- 传递一个数组 -->
  <ConSolg :titles="['nba', 'cba', 'mba']" />
 </div>
</template>

<script>
import ConSolg from "./consolg.vue";
export default {
 //注册局部组件
 components: {
  ConSolg,
 },
}
</script>

<style scoped>

</style>
```

```html
<template>
 <div class="consolg">
  <template v-for="(item, index) in titles" :key="index">
   <div>{{ item }}</div>
  </template>
 </div>
</template>

<script>
export default {
```

# 三、props的对象用法

数组用法中我们**只能说明传入的attribute的名称**，并不能对其进行**任何形式的限制**，接下来我们来看一下对象的写法是如何让 我们的props变得更加完善的。 

当使用对象语法的时候，我们可以对传入的内容限制更多： 

* 比如指定传入的<mark class="hltr-orange">attribute的类型</mark>； 
* 比如指定传入的<mark class="hltr-orange">attribute是否是必传的</mark>； 
* 比如指定没有传入时，<mark class="hltr-orange">attribute的默认值</mark>；

```html
<template>
 <div class="infos">
  <h2>姓名: {{ name }}</h2>
  <h2>年龄: {{ age }}</h2>
 </div>
</template>

<script>
export default {
 //通过props自定义attribute来接收父组件传入的参数
 props: {
  name: {
   type: String,
   require: true,
   default: '我是默认的名字'
  }
  age: {
   type: Number,
   default: '我是默认年龄'
  }
 }
}
</script>

<style scoped>

</style>
```

# 六、细节三：Prop 的大小写命名
Prop 的大小写命名(camelCase vs kebab-case) 

* HTML 中的 attribute 名是大小写不敏感的，所以浏览器会把所有大写字符解释为小写字符； 
* 这意味着当你使用 DOM 中的模板时，camelCase (驼峰命名法) 的 prop 名<mark class="hltr-orange">可以使用其等价的 kebab-case (短横线分隔命名) 命名；</mark>

```html
props: {
	messageInfo
}

<div>
	<show-message messagInfo="哈哈哈"></show-message>
	<show-message message-info="哈哈哈"></show-message>
</div>
```
