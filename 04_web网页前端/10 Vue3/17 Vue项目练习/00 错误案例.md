# 一、拼接字符串

[[03 常见的数据类型#二、string类型|拼接字符串]]的使用中，尽量在函数中使用时，作为参数传入的时候，一定要进行一个保存。

```js
const relativeUrl = `../assets/imgs/${imgUrl}`
return new URL(relativeUrl, import.meta.url).href;
```
这一段代码，对路径进行了一个保存，就可以正常的放回URL,但是使用下面的代码就不可以正常放回URL，这是为什么

```js
return new URL(`../assets/imgs/${imgUrl}`, import.meta.url).href;
```

来自deepseek解答：

> [!tip] 核心原因:
> 在于**模板字符串中的换行符和空格会被保留**，当直接内联在 `new URL()` 中时，如果代码格式化工具（如 Prettier）将模板字符串分割成多行，就会引入不可见的空白字符（换行符 `\n` 或空格），导致最终路径错误。

### 原因分析：

1. **模板字符串保留空白字符**：

```js
// 直接内联时（格式化后可能变成多行）
return new URL(`../assets/imgs/${
     imgUrl  // <- 这里可能插入换行符和空格
  }`, import.meta.url).href;
```

如果模板字符串被格式化成多行，`${imgUrl}` 前后的换行符和缩进会被保留，生成类似 `"../assets/imgs/\n image.png"` 的字符串，破坏路径结构。

2. **变量赋值规避了该问题**：

```js
const relativeUrl = `../assets/imgs/${imgUrl}`; // 通常写在一行内
return new URL(relativeUrl, import.meta.url).href; // 传入的是干净的字符串
```

赋值给变量时，开发者通常会保持单行写法，避免引入额外空白。

### 解决方案：

#### 方法 1：保持模板字符串为单行

确保模板字符串**不换行**：

```js
return new URL(`../assets/imgs/${imgUrl}`, import.meta.url).href; // 整个模板写在一行
```

#### 方法 2：使用 `.trim()` 清除空白（更健壮）

```js
return new URL(
  `../assets/imgs/${imgUrl}`.replace(/\s+/g, ""), // 移除所有空白
  import.meta.url
).href;
```

或直接调用 `trim()`（需确保路径中间无空格）：

```js
return new URL(`../assets/imgs/${imgUrl}`.trim(), import.meta.url).href;
```

#### 方法 3：显式拼接字符串（传统写法）

```js
return new URL("../assets/imgs/" + imgUrl, import.meta.url).href;
```

### 为什么变量赋值能避免问题？

- **代码格式化规则差异**：  
    当模板字符串单独赋值给变量时（`const relativeUrl = ...`），Prettier 等工具通常不会将其拆分成多行。而直接内联在函数参数中时，格式化工具更容易添加换行。
    
- **路径规范要求**：  
    URL 路径对特殊字符（如换行符 `\n`）敏感，`../assets/imgs/\nimage.png` 是非法路径，而变量赋值通常生成合法的 `../assets/imgs/image.png`

# 二、ref和computed区别

```js
const startDateStr = computed(() => formatMouthDate(startDate.value))

const startDateStr = ref(formatMouthDate(startDate.value))
```

## **1. `ref` 的依赖追踪**

### **设计目标**

`ref` 是一个通用的响应式容器，用于包装任意值（基本类型或对象）。它的核心作用是**存储单一值**，并在值变化时触发更新。

### **工作原理**

- `ref` 通过 `.value` 属性访问内部值，其响应式是基于 **`getter/setter`** 实现的。
- **依赖追踪**：当你访问 `ref.value` 时，Vue 会记录当前的副作用（如组件渲染函数或计算属性），但**仅在直接访问 `ref.value` 时生效**。

### **示例问题**

在你的代码中：

```js
const startDateStr = ref(formatMouthDate(startDate.value))
```

- `startDateStr` 的初始值是 `formatMouthDate(startDate.value)` 的结果，但后续 `startDate.value` 变化时：
    - `startDateStr` 不会重新计算，因为它只是存储了一个静态值（初始化时的计算结果）。
    - `ref` 本身的 `.value` 没有被重新赋值，因此不会触发依赖更新。

## **2. `computed` 的依赖追踪**

### **设计目标**

`computed` 是一个**派生值**，用于根据其他响应式数据自动计算新值。它的核心是**自动依赖收集和缓存**。

### **工作原理**

- `computed` 通过传入的 getter 函数动态计算值，并**自动追踪所有依赖**（即 getter 函数中访问的所有响应式数据）。
- **依赖追踪**：
    1. 当 `computed` 被访问时，Vue 会执行 getter 函数，并记录其中访问的所有响应式数据（如 `startDate.value`）。
    2. 当这些依赖中的任何一个变化时，Vue 会标记 `computed` 为脏值，并在下次访问时重新计算。

### **示例分析**

在你的代码中：

```js
const startDateStr = computed(() => formatMouthDate(startDate.value))
```

- `computed` 的 getter 函数中访问了 `startDate.value`，因此 `startDate` 成为其依赖。
- 当 `startDate.value` 变化时，`computed` 会自动重新计算，并触发使用它的 DOM 更新。

## **3. 核心差异对比**

| 特性       | `ref`                   | `computed`            |
| -------- | ----------------------- | --------------------- |
| **设计目标** | 存储可变值，手动更新              | 基于依赖自动计算值，只读（默认）      |
| **依赖追踪** | 仅追踪对 `ref.value` 本身的访问  | 追踪 getter 中访问的所有响应式数据 |
| **更新机制** | 手动赋值（`ref.value = ...`） | 依赖变化时自动重新计算           |
| **缓存**   | 无缓存                     | 有缓存（仅在依赖变化时重新计算）      |

## **4. 何时用 `ref` 或 `computed`？**

- **使用 `ref`**：
    
    - 当你需要存储一个可变值，并手动控制其更新时。
    - 当你需要监听变化并执行副作用时（配合 `watch`）。
- **使用 `computed`**：
    
    - 当值完全由其他响应式数据派生时。
    - 当你希望依赖变化时自动更新，且避免冗余计算（缓存）。

## **总结**

你的问题中，`ref` 版本不响应的原因是：


- `ref` 只追踪对自身 `.value` 的直接修改，而不会自动追踪其初始值的计算逻辑中依赖的其他响应式数据。
- 相比之下，`computed` 会主动分析 getter 函数中的依赖关系，并建立响应式连接。

如果需要 `ref` 版本实现响应式，必须手动监听依赖变化并更新其值（如使用 `watch`）。但在这种场景下，`computed` 是更简洁且符合语义的选择。
