# 一、拼接字符串

[[03 常见的数据类型#二、string类型|拼接字符串]]的使用中，尽量在函数中使用时，作为参数传入的时候，一定要进行一个保存。

```js
const relativeUrl = `../assets/imgs/${imgUrl}`
return new URL(relativeUrl, import.meta.url).href;
```
这一段代码，对路径进行了一个保存，就可以正常的放回URL,但是使用下面的代码就不可以正常放回URL，这是为什么

```js
return new URL(`../assets/imgs/${imgUrl}`, import.meta.url).href;
```

来自deepseek解答：

> [!tip] 核心原因:
> 在于**模板字符串中的换行符和空格会被保留**，当直接内联在 `new URL()` 中时，如果代码格式化工具（如 Prettier）将模板字符串分割成多行，就会引入不可见的空白字符（换行符 `\n` 或空格），导致最终路径错误。

### 原因分析：

1. **模板字符串保留空白字符**：

```js
// 直接内联时（格式化后可能变成多行）
return new URL(`../assets/imgs/${
     imgUrl  // <- 这里可能插入换行符和空格
  }`, import.meta.url).href;
```

如果模板字符串被格式化成多行，`${imgUrl}` 前后的换行符和缩进会被保留，生成类似 `"../assets/imgs/\n image.png"` 的字符串，破坏路径结构。

2. **变量赋值规避了该问题**：

```js
const relativeUrl = `../assets/imgs/${imgUrl}`; // 通常写在一行内
return new URL(relativeUrl, import.meta.url).href; // 传入的是干净的字符串
```

赋值给变量时，开发者通常会保持单行写法，避免引入额外空白。

### 解决方案：

#### 方法 1：保持模板字符串为单行

确保模板字符串**不换行**：

```js
return new URL(`../assets/imgs/${imgUrl}`, import.meta.url).href; // 整个模板写在一行
```

#### 方法 2：使用 `.trim()` 清除空白（更健壮）

```js
return new URL(
  `../assets/imgs/${imgUrl}`.replace(/\s+/g, ""), // 移除所有空白
  import.meta.url
).href;
```

或直接调用 `trim()`（需确保路径中间无空格）：

```js
return new URL(`../assets/imgs/${imgUrl}`.trim(), import.meta.url).href;
```

#### 方法 3：显式拼接字符串（传统写法）

```js
return new URL("../assets/imgs/" + imgUrl, import.meta.url).href;
```

### 为什么变量赋值能避免问题？

- **代码格式化规则差异**：  
    当模板字符串单独赋值给变量时（`const relativeUrl = ...`），Prettier 等工具通常不会将其拆分成多行。而直接内联在函数参数中时，格式化工具更容易添加换行。
    
- **路径规范要求**：  
    URL 路径对特殊字符（如换行符 `\n`）敏感，`../assets/imgs/\nimage.png` 是非法路径，而变量赋值通常生成合法的 `../assets/imgs/image.png`
