# 一、常见的鼠标事件
常见的鼠标事件(不仅仅是鼠标设备，也包括模拟鼠标的设备，比如手机、平板电脑)

事件的调用时在 on 上面的

```js
divEl.onclick = function() {
	console.log("div发生了点击")
}
```
## 1.1 mouseover和mouseenter区别
mouseenter和mouseleave

* 不支持冒泡
* 进入子元素依然属于在该元素内，没有任何反应

mouseover和mouseout

* 支持冒泡
* 进入元素的子元素时
	* 先调用父元素的mouseout
	* 再调用子元素的mouseover
	* 因为支持冒泡，所以会将mouseover传递到父元素中;

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/17286303670003tfjot.png)

## 1.2 区别练习

**要求：一个div里面存在2个按钮，当我们移动鼠标到按钮上的时候打印按钮的内容，移动到div上面的时候不进行打印**

mouseenter：

```js
<body>
  <div class="box">
    <button>提交</button>
    <button>保存</button>
  </div>
  <script>
    var boxEl = document.querySelector(".box")
    boxEl.onmouseenter = function(event) {
      console.log(event.target.textContent)
    }
  </script>
</body>
```

> **mouseenter 不支持冒泡，所以就不能通过 div 对 button 进行监听**
> 会发现鼠标已进入到div就会打印出来 **提交 保存**
> 这并不是我们想要的结果
> 当然我们可以监听 button ，但是这样要对 button 进行遍历，代码的效率较低


mouseover：

```js
<body>
  <div class="box">
    <button>提交</button>
    <button>保存</button>
  </div>
  <script>
    var boxEl = document.querySelector(".box")
    boxEl.onmouseover = function(event) {
      if (event.target !== boxEl) {
        console.log(event.target.textContent)
      }
    }
  </script>
</body>
```

> moseover支持冒泡，就可以通过 div 对 button 进行监听
> 这样就不会进入到 div 就进行一个打印，如果不添加那条 if 条件的话，已经入 div 就会打印（答应 保存），但是通过判断的话就可以避免这个问题。

**如果把这里 mouseover 换成 mouseenter ：**

```js
<body>
  <div class="box">
    <button>提交</button>
    <button>保存</button>
  </div>
  <script>
    var boxEl = document.querySelector(".box")
    boxEl.onmouseenter = function(event) {
      if (event.target !== boxEl) {
        console.log(event.target.textContent)
      }
    }
  </script>
</body>
```

> 鼠标不管移动到 div 还是 button 上面都不会进行打印

# 二、常见的键盘事件

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1728633838000gvph2t.png)

事件的执行顺序是onkeydown、onkeypress、onkeyup

* down事件先发生;
* press发生在文本被输入;
* up发生在文本输入完成;

例如：当我们按住 3 秒键盘 f 键进行输入文本

* 当我们按下去的时候时 down
* 在输入宽输入一个字符 f 的时候时 press
* 3 秒内输入几个字符 f 就有几个 down 和 press
* 最后 3 秒结束的时候我们抬起键盘的字符 f 就是文本输入完成 up

我们可以通过key和code来区分按下的键:

* code:“按键代码”("KeyA"，“ArrowLeft"等)，特定于键盘上按键的物理位置。（**开发中建议使用code来进行事件监听的判断**）
	* 使用code的好处是，不区分大小写，是键盘的一种抽象
* key:字符("A"，“a"等)，对于非字符(non-character)的按键，通常具有与 code 相同的值。)
	* 例如回车键



# 2.1 键盘事件练习

事项按下回车键继续搜索功能：

# 三、常见的表单事件
![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1728636052000zgvtge.png)

onchange和oninput是在元素（input）上面进行监听

onreset和onsubmit是在表单（form）上面进行监听
## 3.1 onchange和oninput的区别

oninput是在文本输入的时候进行的监听，onchange是在文本输入完成的时候（失去焦距）或者 回车

* 不过回车的时候焦距还会再

```js
<body>
  <input type="text">
  <script>
    var inputEl = document.querySelector("input")
    inputEl.oninput = function() {
      console.log("input文本输入",inputEl.value)
    }
    inputEl.onchange = function() {
      console.log("input发生改变",inputEl.value)
    }
  </script>
</body>
```

当我们依次输入 a b c 的时候打印

```
input文本输入 a
input文本输入 ab
input文本输入 abc
```

当我们回车的时候打印

```
input发生改变 abc
```

# 四、文档加载事件
**DOMContentLoaded**:浏览器已完全加载 HTML，并构建了 DOM 树，但像 `<img>`和样式表之类的外部资源可能尚未加载完成。

**load**:浏览器不仅加载完成了 HTML，还加载完成了所有外部资源:图片，样式等。
