# 一、函数的默认参数
在ES6之前，我们编写的函数参数是没有默认值的，所以我们在编写函数时，如果有下面的需求： 

* 传入了参数，那么使用传入的参数； 
* 没有传入参数，那么使用一个默认值； 

而在ES6中，我们**允许给函数一个默认值：**

**ES6的代码:**

```js
<script>
  function foo(x = 20, y = 30) {
    console.log(x + y)
  }
  
  foo()//50
  foo(10, 10)//20
</script>
```

> **会发现源码里面并没有对 null 进行判断，说明是可以传入 null 的**

## 1.1 注意事项

**注意一**：有默认参数的形参尽量写到后面

```js
<script>
  function foo(name, age, height = 18.9) {
    console.log(name, age, height)
  }
  foo()//undefined undefined 18.9
</script>
```

**注意二**：有默认参数的形参，是不会计算在length之内(并且后面所有的参数都不会计算在length之内)

```js
<script>
  function foo1(name = "axl", age, height) {
    console.log(name, age, height)
  }
  console.log(foo1.length)//0
  
  function foo2(name, age = 18, height) {
    console.log(name, age, height)
  }
  console.log(foo2.length)//1
  
  function foo3(name, age, height = "19.8") {
    console.log(name, age, height)
  }
  console.log(foo3.length)//2
</script>
```

**注意三**：剩余参数和默认参数都存在时，剩余参数放到最后

```js
<script>
  function foo(name, age, height = 19.8, ...otherAry){
    console.log(name, age, height, otherAry)
  }
  foo("name", 18, 17.5, "eatting", "running")
  //name 18 17.5 Array(2)
  //name 18 17.5 (2) ['eatting', 'running']
</script
```

# 二、函数默认值和解构一起使用
## 2.1 解构
什么是解构？

```js
<script>
  var obj = {name: "axl", age : 19}
  console.log(obj.name)//axl
  var {name} = obj
  console.log(name)//axl
</script>
```

> 通过上面的结果可以知道，name 在一个代码块里面，我们把 obj 对象赋值给 这个空对象，这个时候定义的 name 就等于 obj 里面 name 属性的值，并不会把 obj 里面 age 的值也赋值过来。

**那么如果两个对象里面的属性名不一样还可以吗？**

```js
  <script>
    var obj = {name: "axl", age : 19}
    console.log(obj)//{name: 'axl', age: 19}
    var {age} = obj
    console.log(age)//19
    var {a} = obj
    console.log(a)//undefined
  </script>
```

> 这里可以看出来，只能赋值相同属性名的值，不同属性名不可以。


**新对象里面的属性值会被赋值，那么是否新对象里面不进行初始化呢？**

```js
<script>
  var obj = {name: "axl", age : 19}
  console.log(obj)//name: 'axl', age: 19}
  var {name, age} = obj
  console.log(name,age)//axl 19
</script>
```

> 新对象里面可以不进行初始化


## 2.2 结合使用

```js
<script>
  //推导过程
  //过程一：函数的默认值是一个对象
  function foo1(obj = { name: "axl", age: 18}) {
    console.log(obj.name,obj.age)
  }
  foo1()//axl 18
  
  //过程二：解构加入
  function foo2({ name, age} = { name: "axl", age: 18}) {
    console.log(name,age)
  }
  foo2()//axl 18
  
  //最终代码:函数的默认值
  function foo3({ name = "axl", age = 18} = {}) {
    console.log(name, age)
  }
  foo3()//axl 18
</script>
```

# 三、函数的剩余参数
ES6中引用了rest parameter，可以将不定数量的参数放入到一个数组中:

* 如果最后一个参数是… 为前缀的，那么它会将剩余的参数放到该参数中，并且作为一个数组;

```js
<script>
  function foo(m, n, ...other) {
    console.log(m, n)//1 2
    console.log(other)//[3, 4, 5, 6, 7, 8]
  }
  foo(1, 2, 3, 4, 5, 6, 7, 8)
</script>
```

那么剩余参数和[[01 函数属性和arguments#二、认识arguments|arguments]]有什么区别呢?

* 剩余参数只包含那些**没有对应形参的实参**，而 arguments 对象包含了**传给函数的所有实参**
* **arguments对象不是一个真正的数组**，而**rest剩余参数是一个真正的数组**，可以进行数组的所有操作;
* arguments是**早期的ECMAScript**中为了方便去获取所有的参数提供的一个数据结构，而rest参数是**ES6中提供**并且希望以此来替代arguments的;

# 四、箭头函数的补充

* 箭头函数是**没有显式原型prototype**的，所以不能作为构造函数，使用new来创建对象； 

* 箭头函数也不绑定this、arguments、super参数；

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/17320199240003dqu9e.png)

```js
<script>
  var foo = () => {}
  
  console.log(foo.prototype)//undefined

  var a = new foo()//foo is not a constructor
</script>
```
