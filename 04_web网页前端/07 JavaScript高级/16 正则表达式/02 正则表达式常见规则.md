# 一、字符类(Character classes)
**字符类**（Character classes） 是一个特殊的符号，匹配特定集中的任何符号。

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1733045436000un2ylj.png)

反向类（Inverse classes） 

* ``\D`` 非数字：除 ``\d`` 以外的任何字符，例如字母。 
* `\S` 非空格符号：除 ``\s`` 以外的任何字符，例如字母。 
* ``\W``  非单字字符：除 ``\w`` 以外的任何字符，例如非拉丁字母或空格。

# 二、锚点(Anchors)
符号 ^ 和符号 $ 在正则表达式中具有特殊的意义，它们被称为“锚点”。 

* 符号 `^` 匹配文本开头； 
* 符号 `$` 匹配文本末尾；

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1733127945000b789sm.png)


词边界（Word boundary） 

* 词边界 `\b` 是一种检查，就像 `^` 和 `$` 一样，它会检查字符串中的位置是否是词边界。 
* 词边界测试 `\b` 检查位置的一侧是否匹配 `\w`，而另一侧则不匹配 `"\w"`

在字符串 Hello, Java! 中，以下位置对应于 `\b`：

匹配下面字符串中的时间：

```js
<script>
  const str = "num is 12:23, aoao is 44:49, miao 123:123"
  
  const timePattern1 = /\d\d:\d\d/g
  console.log(str.match(timePattern1))
  //['12:23', '44:49', '23:12']
  
  const timePattern2 = /\b\d\d:\d\d\b/g
  console.log(str.match(timePattern2))
  //['12:23', '44:49']
</script>
```

# 三、转义字符串
如果要把特殊字符作为常规字符来使用，需要对其进行转义： 

* 只需要在它前面加个反斜杠； 

常见的需要转义的字符：

``[] \ ^ $ . | ? * + ( )``

斜杠符号 ‘`/’` 并不是一个特殊符号，但是在字面量正则表达式中也需要转义；

练习：匹配所有以.js或者jsx结尾的文件名

```js
<script>
  const fileName = ["abc.js", "cba.java", "nba.html", "mba.js"]
  const fil = /\.js$/
  const newName = fileName.filter(item => fil.test(item))
  console.log(newName)
  //['abc.js', 'mba.js']
</script>
```


在webpack当中，匹配文件名时就是以这样的方式。

# 四、集合(Sets)和范围(Range)
有时候我们只要选择多个匹配字符的其中之一就可以： 

* 在方括号 `[…]` 中的几个字符或者字符类意味着“搜索给定的字符中的任意一个”； 

集合（Sets） 

* 比如说，`[eao]` 意味着查找在 3 个字符 `‘a’`、`‘e’` 或者 `‘o’` 中的任意一个；

范围（Ranges） 

* 方括号也可以包含字符范围； 
* 比如说，`[a-z]` 会匹配从 a 到 z 范围内的字母，`[0-5]` 表示从 0 到 5 的数字； 
* `[0-9A-F]` 表示两个范围：它搜索一个字符，满足数字 0 到 9 或字母 A 到 F； 
* `\d` —— 和 `[0-9]` 相同； 
* `\w` —— 和 `[a-zA-Z0-9_]` 相同；

# 五、量词(Quantifilers)
 假设我们有一个字符串 +7(903)-123-45-67，并且想要找到它包含的所有数字。 
 
  * 因为它们的数量是不同的，所以我们需要给与数量一个范围； 
  * 用来形容我们所需要的数量的词被称为量词（ Quantifiers ）。 
  
  **数量 `{n}`** 
  
* 确切的位数：`{5}` 
* 某个范围的位数：`{3,5}` 

**缩写：** 

* `+`：代表“一个或多个”，相当于 `{1,}` 
* `?`：代表“零个或一个”，相当于 `{0,1}`。换句话说，它使得符号变得可选； 
* `*`：代表着“零个或多个”，相当于 `{0,}`。也就是说，这个字符可以多次出现或不出现；

# 六、贪婪(Greedy)和惰性(Iazy)模式
如果我们有这样一个需求：匹配下面字符串中所有使用`《》`包裹的内容

```js
<script>
  const message = "我喜欢的两本书: 《javascript高级》和《龙族》"
  const fil = /《.+》/ig
  console.log(message.match(fil))
  //['《javascript高级》和《龙族》']
</script>
```

默认情况下的匹配规则是查找到匹配的内容后，会继续向后查找，一直找到最后一个匹配的内容 

* 这种匹配的方式，我们称之为**贪婪模式（Greedy）**

**懒惰模式**中的**量词**与贪婪模式中的是相反的。 

* 只要获取到对应的内容后，就不再继续向后匹配； 
* 我们可以在量词后面再加一个问号 `'?'` 来启用它； 
* 所以匹配模式变为 `*?` 或 `+?`，甚至将 `'?'` 变为 `??`

```js
<script>
  const message = "我喜欢的两本书: 《javascript高级》和《龙族》"
  const fil = /《.+?》/ig
  console.log(message.match(fil))
  //['《javascript高级》', '《龙族》']
</script>
```

# 七、捕获组（capturing group）
模式的一部分可以用括号括起来 (...)，这称为“捕获组（capturing group）”。 

这有两个作用： 

* 它允许将匹配的一部分作为结果数组中的单独项； 
* 它将括号视为一个整体；

```js
<script>
  const bar = "abcabc123 dfakjsabcabcflj 44abcabc"
  const fil = /(abc){2,}/ig
  
  const newbar = bar.match(fil)
  console.log(newbar)
  //['abcabc', 'abcabc', 'abcabc']
</script>
```

方法 str.match(regexp)，如果 regexp 没有 g 标志，将查找第一个匹配并将它作为一个数组返回： 

* 在索引 0 处：完全匹配。 
* 在索引 1 处：第一个括号的内容。 
* 在索引 2 处：第二个括号的内容。 
* …等等…
