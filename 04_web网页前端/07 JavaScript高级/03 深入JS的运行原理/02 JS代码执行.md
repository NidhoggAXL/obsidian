# 一、初始化全局对象

**js引擎会在执行代码之前**，会在堆内存中创建一个**全局对象**：Global Object（GO） - Window

* 该对象 所有的作用域（scope）都可以访问； 
* 里面会包含Date、Array、String、Number、setTimeout、setInterval等等； 
* 其中还有一个window属性指向自己；

# 二、执行上下文（Execution Contexts）
js引擎内部有一个**执行上下文栈**（Execution Context Stack，简称ECS），它是用于执行代码的调用栈。 

那么现在它要执行谁呢？执行的是**全局的代码块**： 

* 全局的代码块为了执行会构建一个 **Global Execution Context**（GEC）- 全局上下文； 
* GEC会 **被放入到ECS中** 执行； 
* 每一个代码块都是**不同的执行上下文**

GEC被放入到ECS中里面包含两部分内容： 

* 第一部分：在代码执行前，在**parser转成AST的过程**中，会将全局定义的变量、函数等加入到GlobalObject中，但是**并不会赋值**； 
	* 这个过程也称之为变量的**作用域提升（hoisting）** 
* 第二部分：在代码执行中，对变量赋值，或者执行其他的函数；

# 三、认识VO对象（Variable Object）

==每一个执行上下文会关联一个VO==**（Variable Object，变量对象），变量和函数声明会被添加到这个VO对象中。**

* 包括函数的参数也会被添加到VO对象中
* 同时函数的**声明体**会被放到**FO（函数对象中）**
* **VO 中保存函数名及对应的函数声明体地址。**

# 五、函数如何的执行
在执行的过程中**执行到一个函数时**，就会根据函数体创建一个**函数执行上下文**（Functional Execution Context，简称FEC）， 并且压入到**EC Stack**中。 

因为每个执行上下文都会关联一个VO，那么函数执行上下文关联的VO是什么呢？ 

* 当进入一个函数执行上下文时，会创建一个**AO对象**（Activation Object）； 
* 这个AO对象会**使用arguments作为初始化**，并且**初始值是传入的参数**； 
* 这个**AO对象会作为执行上下文的VO来存放变量的初始化**；

# 六、执行过程总结

**代码确实实在栈空间执行的，只是会在堆空间创建对应的对象**

* 在代码进行前，对应对象里面创建对应的属性进行初始化
	* 属性中的函数和对象会直接初始化为指定函数对象和指定对象的地址
	* 其他属性初始为 undefined
* 在代码执行过程中，对应从对象会被进行赋值
	* 先运行全局上下文，当运行不是全局上下文的时候（新的代码块）会在栈空间创建新的执行上下文
	* 在新的上下文中依次反复操作，知道上下文里面没有新的执行上下文
	* 运行完一个执行上下文的时候，就会把这个上下文进行弹栈操作
	* 知道全局上下文执行完毕，并弹栈。
* 在执行完后，栈空间里面就没有执行上下文啦


```js
  <script>
    console.log(message)//underfined
    var message = "aoao"
    console.log(message)//aoao
  </script>
```

# 七、作用域和作用域链（Scope Chain）

当进入到一个执行上下文时，执行上下文也会关联一个作用域链（Scope Chain）

* 作用域链是一个对象列表，用于变量标识符的求值； 
* 当进入一个执行上下文时，这个作用域链被创建，并且根据代码类型，添加一系列的对象；

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/173089356200010ahus.png)

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1730893631000ymuh03.png)


```js
  <script>
    var message = "Global Message"
  
    function foo() {
      console.log(message)
    }

    var obj = {
      name: "obj",
      bar: function() {
        var message = "bar message"
        foo()
      }
    }
  
    obj.bar()//Global Message
  </script>
```

> [!tip] 为什么打印的是 Global Message 而不是 bar message 呢？
> 函数在**预处理的时候就已经确定好了作用域链（可以理解为声明的时候）**，这个时候的作用域链就已经确定好啦。
> 上面题目给出的作用域链是 Local -> Global(Window)
> **所以并不会在函数调用的时候才进行作用域链的确定而是在变量定义的时候就已经确定了作用域链了**

# 八、作用域提升面试题
## 8.1 题目一
```js
  <script>
    var n = 100
    function foo() {
      n = 200
    }
    foo()
    console.log(n)//200
  </script>
```

函数里面并没有定义变量，而是进行赋值操作，赋值的是在 GO 中的。
后面`console.log(n)`打印为200

## 8.2 题目二
```js
  <script>
    function foo() {
      console.log(n)//undefined
      var n = 200
      console.log(n)//200
    }
    var n = 100
    foo()
  </script>
```

两个打印操作都是在函数内部进行的，并且函数内部已经进行了n变量的定义`var n = 200`这样就是产生作用域的提升（解析的时候就会创建变量 `n = undefined`）。

在函数内部使用 n 的时候

* 没有进行 n 赋值前使用，是使用作用域提升的 `n = undefined`
* 进行 n 赋值后使用，是使用赋值后`n = 200`

## 8.3 题目三
```js
  <script>
    var n = 100
    function foo1() {
      console.log(n)
    }
    function foo2() {
      var n = 200
      console.log(n)//200
      foo1()//100
    }

    foo2()
    console.log(n)//100
  </script>
```

`foo2()`调用时，函数内部已经定义了`var n =200`这个时候`console.log(n)`打印的为AO对象里面的 n ，所以打印为 200.

当运行`foo2()`内部的`foo1()`的时候运行 foo1 里面的函数代码`console.log(n)`这个时候在 foo1 的AO中并没有定义 n 这个时候就是在解析的时候就确定了 `scope` 是 GO
，所以在 GO 里面找到定义的 `n = 100` 打印结果为 100

最后全局代码的`console.log(n)`在GO里面查到到 `n = 100` ，所以为 100

## 8.4 题目四
```js
  <script>
    var a =100
    function foo() {
      console.log(a)
      return
      var a =100
    }
    foo()//undefined
  </script>
```

在这里首先要明白一个代码解析和运行的前后关系：

* 如果定义了变量，那么在解析代码的时候就会先定义这个变量了，并对这个变量进行初始化为`undefined`
* `return` 是在代码运行的时候才会运行的

按照上面的关系，当运行`foo()`的时候 a 已经在函数内部解析并初始化了（`var a = undefined)`)，这个时候在 return 前面 `console.log(a)` 打印的结果为 undefined。

## 8.5 题目五
```js
  <script>
    function foo() {
      var a = b = 100
    }
    foo()

    console.log(b)//100
    console.log(a)//a is not defined(报错)
  </script>
```

这段代码可以解析为

```js
  <script>
    function foo() {
      b = 100
      var a = b
    }
    foo()
    console.log(b)
    console.log(a)
  </script>
```

这里要明白一点，如果JS中在进行变量的定义的时候，如果没有关键字的话，默认是添加到全局变量的，不管是在那里通过这种办法定义的。

```js
  <script>
    function foo() {
      b = 100
    }
    foo()
    console.log(window.b)//100
    //window.b全局里面的b是存在的，
    //并且这里的 b 是在函数内部定义的
  </script>
```

> 通过这样就可以知道为什么在函数外部的时候可以`console.log(b)`打印为100
> `console.log(a)`的时候，a 是在函数内部定义，存在函数的AO，`console.log(a)`是打印 GO 中的 a ，而 GO 中并没有 a ，所以会报错误。
