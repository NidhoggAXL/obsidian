# 一、什么是生成器
生成器是ES6中新增的一种**函数控制**、**使用**的方案，它可以让我们**更加灵活的控制函数什么时候继续执行、暂停执行**等。 

* 平时我们会编写很多的函数，这些函数终止的条件通常是返回值或者发生了异常。


生成器函数也是一个函数，但是和普通的函数有一些区别： 

* 首先，生成器函数需要在**function**的后面加一个符号`*`：

```js
//1.第一种编写法
function* foo(){}

//2.第二种编写法
function *foo(){}
```

* 其次，生成器函数可以通过**yield关键字**来控制函数的**执行流程**： 
* 最后，生成器函数的返回值是一个Generator（生成器）： 
	* 生成器事实上是一种特殊的迭代器； 
	* MDN：Instead, they return a special type of iterator, called a Generator.

```js
<script>
  function* foo() {
    console.log("111")
    yield
    console.log("222")
    yield
    console.log("333")
  }
</script>
```

> 不先函数代码块开始就编写 yield

# 二、生成器函数执行
我们发现下面的生成器函数foo的执行体压根没有执行，它只是返回了一个生成器对象。 

* 那么我们如何可以让它执行函数中的东西呢？**调用next即可**； 
* 我们之前学习迭代器时，知道迭代器的next是会有返回值的； 
* 但是我们很多时候不希望next返回的是一个undefined，这个时候我们**可以通过yield来返回结果**；

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/17327798540003wxy2j.png)

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1732779858000zzy884.png)

**那么生成器函数在执行到yield是执行到yield前面还是后面呢？** 具体看下面这段代码

```js
<script>
  function* foo() {
    console.log("111")
    yield console.log("next1")
    console.log("222")
    yield "bb"
    console.log("333")
  }
  
  const generator = foo()
  generator.next()
  //111
  //next1
</script>
```

> 通过上面代码可以知道，是**执行到 yield 后面同行的代码**

# 三、生成器传递参数-next函数
函数既然可以暂停来分段执行，那么函数应该是可以**传递参数**的，我们是否可以给每个分段来传递参数呢？ 

* 答案是**可以的**； 
* 我们在调用next函数的时候，可以给它传递参数，那么这个参数会作为上一个yield语句的返回值； 
* 注意：**也就是说我们是为本次的函数代码块执行提供了一个值**；
* **只能传入一个参数，不可以是多个参数**

```js
<script>
  function* foo(value1) {
    console.log("执行内部代码:111", value1)
    const value2 = yield "aaa"
    console.log("执行内部代码:222", value2)
    const value3 = yield "bbb"
    console.log("执行内部代码:333",value3)
  }
  
  const generator = foo("aaa")
  
  console.log(generator.next())

  console.log(generator.next("bbb"))

  console.log(generator.next("ccc"))
</script>
```

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1732781386000sjulaz.png)

> [!tip] 细节
> * 第一个 next 的参数在 **调用 foo** 的时候就传入
> * 第一次指向 next 的时候就不需要传入参数
> * 后面调用的 next 的时候就可以正常传入形参，形参会被保存到 yield 的关键字前的变量上。


# 四、生成器提前结束 – return函数
还有一个可以给生成器函数传递参数的方法是通过return函数： 

* return传值后这个生成器函数就会**结束**，之后调用next**不会继续生成值了**；

```js
<script>
  function* foo() {
    console.log("代码开始执行next1")
    yield "aa"
    console.log("next2")
    return "000"
    yield "bbb"
    console.log("next3")
  }

  const generator = foo()
  console.log(generator.next())
  console.log(generator.next())
  console.log(generator.next())
  console.log(generator.next())
</script>
```

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1732782341000wtha68.png)

> 这样会**执行** next2 的里面 **console.log("next2")** 这段代码

也可以编写为：

```js
<script>
  function* foo() {
    console.log("代码开始执行next1")
    yield "aa"
    console.log("next2")
    yield "bbb"
    console.log("next3")
  }
  
  const generator = foo()
  console.log(generator.next())
  console.log(generator.return("000"))
  console.log(generator.next())
  console.log(generator.next())
</script>
```

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1732782446000aio7tj.png)

> 这样会**不执行** next2 的里面 **console.log("next2")** 这段代码


# 五、生成器抛出异常 – throw函数（了解）
除了给生成器函数内部传递参数之外，也可以给生成器函数内部抛出异常： 

* 抛出异常后我们可以在生成器函数中**捕获异常**； 
* 但是在catch语句中不能继续yield新的值了，但是可以在catch语句外使用yield继续中断函数的执行；

# 六、生成器替代迭代器
我们发现生成器是一种特殊的迭代器，那么在某些情况下我们可以使用生成器来替代迭代器：

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1732784703000o03dsj.png)

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1732785281000bxsv2i.png)


> 每次调用 next 都会把 yield 后面的放回值放回回来，这样就可以实现迭代啦。
> 调用生成器函数得到的**生成器本身就是一个特殊的可迭代器**，所以可以调用 next。


事实上我们还可以使用`yield*`来生产一个可迭代对象： 

* 这个时候相当于是一种yield的语法糖，只不过会依次迭代这个可迭代对象，每次迭代其中的一个值；
