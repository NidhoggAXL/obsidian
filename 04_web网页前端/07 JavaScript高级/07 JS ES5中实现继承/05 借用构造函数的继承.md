# 一、组合借用继承（属性继承）
为了解决原型链继承中存在的问题（**属性的继承**），开发人员提供了一种新的技术: constructor stealing(有很多名称: 借用构造函数或者称之 为经典继承或者称之为伪造对象)： 

* steal是偷窃、剽窃的意思，但是这里可以翻译成借用；

借用继承的做法非常简单：**在子类型构造函数的内部调用父类型构造函数.** 

* 因为函数可以在任意的时刻被调用； 
* 因此通过[[01 this的绑定规则#1.3.1 call、apply|call()和apply()方法]]也可以在新创建的对象上执行构造函数；


```js
<script>
  function Person(name, age) {
    this.name = name
    this.age = age
  }
  
  Person.prototype.running = function() {
    console.log("running~")
  }

  function Student(name, age, sno, score) {
    //借用Person构造函数
    Person.call(this, name, age)
    this.sno = sno
    this.score = score
  }
  
  //01 把函数对象Person的显式原型赋值给 p 对象的隐式原型
  var p = new Person()
  //02 把p对象隐式原型赋值给 Student 函数的函数对象中的显式原型
  Student.prototype = p
  
  Student.prototype.eatting = function() {
    console.log("eatting~")
  }

  var stu1 = new Student("axl", 19, 100, 90)
  //会把 Student 函数的显式原型赋值给 stu1 的隐式原型
  //根据上面的 02 ，stu1 的隐式原型就等于 p 对象的隐式原型
  //根据上面的 01，stu1 的隐式原型就等于 Person 的显式原型
  //这样在 Person.prototype 上面定义的方法就可以在 stu1 中使用
  console.log(stu1)
  stu1.eatting()
  stu1.running()
</script>
```

借用构造函数的精髓就是：**子类构造函数调用父类的构造函数，并且让父类中的this指向子类。**

```js
Person.call(this, name, age)
```

在 Student 中调用，这里的 this 是指向调用 Studetn 构造函数创建的对象。

通过 call 显式绑定的方法，把 Student 中的 this 绑定定到 Person 构造函数上

这样通过 call 显式绑定调用的时候，就是 Person 构造函数内部执行的是 Student 的 this，而不是 Person 的this。

这样调用 Student 构造函数的时候，Student 内部的 this 还是指向自己

而通过 call 显式绑定调用的 Person 构造函数内部的 this 也还是指向 Student

并且这样的话，单独调用 Person 构造函数的时候，对本身的 this 没有影响，还是指向 Persong 自己。


# 二、组合借用继承的问题
组合继承是JavaScript最常用的继承模式之一： 

* 如果你理解到这里, 点到为止, 那么组合来实现继承只能说问题不大； 
* 但是它依然不是很完美，但是基本已经没有问题了；

组合继承存在什么问题呢? 

* 组合继承最大的问题就是无论在什么情况下，都会**调用两次父类构造函数。** 
	* 一次在创建子类原型的时候； 
	* 另一次在子类构造函数内部(也就是每次创建子类实例的时候)； 
* 另外，如果你仔细按照我的流程走了上面的每一个步骤，你会发现：**所有的子类实例事实上会拥有两份父类的属性** 
	* 一份在当前的实例自己里面(也就是person本身的)，另一份在子类对应的原型对象中(也就是`person.__proto__`里面)； 
	* 当然，这两份属性我们无需担心访问出现问题，因为默认一定是访问实例本身这一部分的；
