# 一、认识严格(strict)模式
JavaScript历史的局限性:

* 长久以来，**JavaScript 不断向前发展且并未带来任何兼容性**问题;
* 新的特性被加入，旧的功能也没有改变，这么做**有利于兼容旧代码**
* 但缺点是 JavaScript 创造者的**任何错误或不完善的决定也将永远被保留在JavaScript 语言中;**

在ECMAScript5标准中，JavaScript提出了**严格模式的概念(Strict Mode)**

* 严格模式很好理解，是一种**具有限制性的JavaScript模式**，从而**使代码隐式的脱离了”懒散(sloppy)模式**
* **支持严格模式的浏览器在**检测到代码中有严格模式时，会**以更加严格的方式对代码进行检测和执行;**

严格模式对正常的JavaScript语义进行了一些限制:

* 严格模式通过 **抛出错误** 来消除一些原有的 **静默(silent)** 错误;
* 严格模式让**JS引擎在执行代码时可以进行更多的优化**(不需要对一些特殊的语法进行处理);
* 严格模式禁用了**在ECMAScript未来版本中可能会定义的一些语法，**

# 二、开启strict模式
那么如何开启严格模式呢?严格模式支持粒度话的迁移:

* 可以支持在js文件中开启严格模式
* 也支持对某一个函数开启严格模式;

严格模式通过在文件或者函数开头使用 use strict 来开启。

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1731221015000js6gf2.png)

**没有类似于"no use strict"这样的指令可以使程序返回默认模式。**

* 现代JavaScript支持“class"“module”，它们会自动启用 use strict;

# 三、strict模式的限制
1. 无法意外的创建全局变量

```js
  <script>
    "use strict"
    function foo() {
      message = "hello"
    }
    foo()
    console.log(message)
    //message is not defined
  </script>
```

> 不在严格模式下：
> * 函数内部的 message 会被定义的全局变量中
> * 全局中`console.log(message)`可以打印出来`hello`

2. 严格模式会使引起静默失败(silently fail,注:不报错也没有任何效果)的赋值操作抛出异常

```js
<script>
  "use strict"
  var obj = {
    name: "axl"
  }
  
  Object.defineProperty(obj, "name", {
    writable:false
    //不可写
  })
  
  obj.name = "coder"
  console.log(obj.name)
  //Cannot assign to read only property 'name' of object '#<Object>'
  //无法分配只读属性
</script>
```

> [!tip] 不在严格模式下：
> * `obj.name = "coder"`是错误的，但是这个静默失败是不会报错的
> * `console.log(obj.name)`打印结果为`axl`

3. 严格模式下试图删除不可删除的属性

```js
<script>
  "use strict"
  var obj = {
    name: "axl"
  }
  
  Object.defineProperty(obj, "name", {
    configurable: false
    //可配置的
  })
  
  delete obj.name
  console.log(obj.name)
  //Cannot delete property 'name' of #<Object>
  //无法删除属性
</script>
```

> [!tip] 不在严格模式下：
> * `delete obj.name`是错误的，但是这个静默失败不会报错
> * `console.log(obj.name)`打印结果为`axl`

4. 严格模式不允许函数参数有相同的名称

```js
<script>
  "use strict"
  function foo(num, num) {}
  //Duplicate parameter name not allowed in this context
  //此上下文不允许相同的参数名
</script>
```

> [!tip] 不在严格模式下：
> 不会进行报错

5. 不允许0的八进制语法

```js
<script>
  "use strict"
  console.log(0123)
  //Octal literals are not allowed in strict mode
  //严格模式不允许使用八进制文本
</script>
```

> [!tip] 不在严格模式下：
> 不会出现报错打印为 `0123八进制转换为的二进制 83`

6. 在严格模式下，不允许使用with

```js
<script>
  "use strict"
  function foo() {
    var message = "hello"
  }
  foo()
  
  with(obj) {
    console.log(message)
  }
  // Strict mode code may not include a with statement
</script>
```


7. 在严格模式下，eval不再为上层引用变量

```js
<script>
  "use strict"
  eval(`var message = "hello";`)
  console.log(message)
  //message is not defined
</script>
```

> [!tip] 不在严格模式下：
> `console.log(message)`结果为hello

8. 严格模式下，this绑定不会默认转成对象

```js
<script>
  "use strict"
  function foo() {
    console.log(this)
  }
  foo.apply("abc")//abc
</script>

<script>
  function foo() {
    console.log(this)
  }
  
  //显式绑定this
  var bar = foo.apply("abc")
  //String {'abc'}
</script>
```

> [!tip] 不在严格模式下：
> `String {'abc'}`把 abc 转换为 String 对象。

9. 严格模式下，默认this绑定undefined

```js
<script>
  "use strict"
  function foo() {
    console.log(this)
  }
  foo()//undefined
</script>
```

> [!tip] 不在严格模式下：
> 默认绑定 Window

10. 严格模式下，this可以绑定undefined或者null

```js
<script>
  "use strict"
  function foo() {
    console.log(this)
  }
  foo.apply(null)//null
  foo.apply(undefined)//undefined
</script>
```

> [!tip] 不在严格模式下：
> 不可以绑定 `null undefined`
