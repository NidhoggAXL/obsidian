**首先要知道什么时候执行：**

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1733383639000cpbhxk.png)

```js
<script>
  const inputEl = document.querySelector("input")
  
  function hythrottle(fn, interval) {
    //记录开始时间，并进行初始化
    let startTime = 0
  
    const _throttle = function() {
      //获取当前时间
      const nowTime = new Date().getTime()
  
      //计算发送请求需要等待的时间
      const waitTime = interval - (nowTime - startTime)
      if (waitTime <= 0) {
        fn()
        //当发送网络请求后，修改开始时间为现在发送请求的时间
        startTime = nowTime
      }
    }
    return _throttle
  }
  
  let count = 1
  inputEl.oninput = hythrottle(function() {
    console.log(`发送网络请求${count++}`)
  }, 2000)
</script>
```

> 当输入第一个字符 a 的时候，立刻发送请求，后面每过两秒发送网络请求

# 四、最后一次是否执行（了解）
**有两种思路：**

1. 在每一次输入字符的时候，对当前输入的开启一个定时器
	* 如果后面还监听到输入，那就清除上一次的定时器
	* 如果没有那么到定时器规定的时间就发生请求

2. 在节流函数的规定的非节点，设置定时器
	* 如果在节点到节点之间一直有输入字符，那么到下一个节点的时候，就取消上一次非节点的定时器，再在本次非节点开启定时器
	* 如果节点之间存在没有一直输入字符，那么定时器到等待时间（waitTime）就会发送请求，完成请求操作

> [!tip] 第二种方案的好处：
> * 不存在大量的开启和取消定时器-节省资源
> * 只需要对节点的定时器进行处理-方便控制

**下面的所有优化和操作都是按照方案二：**
