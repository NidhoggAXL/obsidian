# 一、什么是箭头函数
箭头函数是ES6之后增加的一种编写函数的方法，并且它比函数表达式要更加简洁:

* 箭头函数**不会绑定this、arguments属性**;
	* arguments 里面包含传入的实参
* 箭头函数**不能作为构造函数来使用**(不能和new一起来使用，会抛出错误)

箭头函数如何编写呢?

* ():函数的参数
* {}:函数的执行体

```js
  <script>
    var names = ["a", "b", "c"]
    //不使用箭头函数
    names.forEach(function (item, index, arr) {
      console.log(item, index, arr)
    })
    //a 0 (3) ['a', 'b', 'c']
    //b 1 (3) ['a', 'b', 'c']
    //c 2 (3) ['a', 'b', 'c']

    //使用箭头函数
    names.forEach((item, index, arr) => {
      console.log(item, index, arr)
    })
    //a 0 (3) ['a', 'b', 'c']
    //b 1 (3) ['a', 'b', 'c']
    //c 2 (3) ['a', 'b', 'c']
  </script>
```

# 二、箭头函数的编写优化
**优化一**: 如果只有一个参数()可以省略

```js
  <script>
    var names = ["a", "b", "c"]
    
    //没有优化
    names.forEach((item) => {
      console.log(item)//a b c
    })
    //优化一
    names.forEach(item => {
      console.log(item)//a b c
    })
  </script>
```


**优化二**: 如果函数执行体中只有一行代码，那么可以省略大括号
* 并且这行代码的**返回值会作为整个函数的返回值**
* 但是不可以在返回值那里手动编写return

```js
<script>
  var names = ["a", "b", "c"]
  //没有优化
  names.forEach((item) => {
    console.log(item)//a b c
  })
  //优化二
  names.forEach(item => console.log(item))//a b c


  var box = [2, 9, 1, 5, 4]
  //没有优化
  var baz = box.filter(function(item) {
    return item % 2 === 0
  })
  console.log(baz)//[2, 4]
  //优化二
  //错误优化二，不能编写return
  // var baz = box.filter(item => return item % 2 === 0)
  //正确优化二：
  var baz = box.filter(item => item % 2 === 0)
  console.log(baz)//[2, 4]
</script>
```

```js
<script>
  var names = ["a", "b", "c"]
  var box = [2, 9, 1, 5, 4]
  
  names.forEach(item => console.log(item))//a b c
  console.log( typeof(console.log()) )//undefine
  
  var baz = box.filter(item => item % 2 === 0)
  //item % 2 === 0 返回一个布尔值
  console.log(baz)//[2, 4]
</script>
```


**优化三**: 如果函数执行体**只有返回一个对象**，那么需要给这个对象加上()

```js
<script>
  //不优化
  var bar = () => {
    return { name: "axl" }
  }
  console.log(bar)

  //优化三：
  //错误编写
  // var bar = () => {name: "axl"}
  var bar = () => ({name: "axl"})
  console.log(bar)

  //var bar = () => {} 注意这里的{}是一个执行体
</script>
```

# 三、this规则之外-ES6箭头函数
箭头函数**不使用this的四种标准规则**(也就是不绑定this，不会受四种标准规则改变)，而是根据**外层作用域来决定this.**

```js
<script>
  var bar = () => {
    console.log(this)
    //箭头函数里面没有this
    //没有向外侧作用域查找
    //查找到全局this，全局this绑定全局Window
  }
  
  bar()//Window
  //这是全局的Window
  
  bar.apply("aaa")//Window
  //箭头函数里面放的this不适用this的四种规则
  //所以箭头函数里面的this不可以改变
  //只根据外层作用域来决定
</script>
```

## 3.1 this的作用域查找

**案例一：**

```js
<script>
  var obj = {
    name: "axl",
    demo: function() {
      var bar = () => {
        console.log(this)
      }
      //定义一个箭头函数，返回这个函数
      return bar
    }
  }

  //这里注意是obj调用了this
  var baz = obj.demo()//等价与把bar赋值给了baz
  baz()//调用函数使用()，所以要baz()
  //结果obj对象{name: 'axl', demo: ƒ}
</script>
```

this的作用域查找顺序：

* 注意：obj对象里面不是一个作用域，**代码块**或者**函数里面**才有作用域

![[JavaScript高级箭头函数this指向01]]

> 在箭头函数里面没有找到this的指向 => 普通函数里面找到，但是这里存在隐式装换，找到了this的指向，所以this的指向就是obj对象`{name: 'axl', demo: ƒ}`


**案例二：**

```js
<script>
  var obj = {
    name: "axl",
    demo: () => {
      var bar = () => {
        console.log(this)
      }
      //定义一个箭头函数，返回这个函数
      return bar
    }
  }
  
  var bar = obj.demo()
  bar()//Window
</script>
```
