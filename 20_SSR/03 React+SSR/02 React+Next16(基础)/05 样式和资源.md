# 一、全局和局部样式

Next.js 允许在 JavaScript 文件中直接通过 import 关键字来导入CSS 文件（**不是：@import** ）

全局样式：

 - 在 assets 目录或 styles 目录下编写，然后在 `app/layout.css` 入口组件中导入
 - 也支持导入node_modules中样式，导入文件后缀名不能省略

局部样式：

 - Next.js 默认是支持 CSS Module 的，如：`[name].module.css`
 - CSS Module 中的选择器会自动创建一个唯一的类名。
 - 唯一类名保证在不同的文件中使用相同CSS类名，也不用担心冲突

```scss
// styles/home.module.sass
.title {
  color: red;
}

$primary: #0f0;
.primaryColor {
  color: $primary;
}
```

```tsx
import homeStyle from '../styles/home.module.scss'
const About = function () {
  return (
    <div>
      <h3 className={homeStyle.title}>About页面</h3>
      <h2 className={homeStyle.primaryColor}>使用primaryColor样式</h2>
    </div>
  )
}

export default About
```

# 二、静态资源

public目录

 - 常用于存放静态文件，例如：robots.txt、favicon.ico、img等，并直接对外提供访问。
 - 访问需以 / 作为开始路径，例如：添加了一张图片到 public/me.png中
	 - 可通过静态URL： /me.png 访问，如右图
	 - 静态URL也支持在背景中使用
 - **注意**： 确保静态文件中没有与 pages/ 下的文件重名，否则导致错误。

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1774521583000cnrm4p.png)


assets目录

 - 常用存放样式、字体 、图片或 SVG 等文件
 - 可用 import 导入 位于assets目录的文件，支持相对路径和绝对路径
	 - import Avatar from `"../assets/images/avatar.png"`
	 - import Avatar from `“@/assets/images/avatar.png”`
 - **背景图片和字体**： `url("~/assets/images/bym.png")`

# 三、字体图标

字体图标使用步骤

 - 将字体图标存放在 assets 目录下
 - 字体文件可以使用相对路径和绝对路径引用。
 - 在 layout.tsx 文件中导入全局样式
 - 在页面中就可以使用字体图标了
