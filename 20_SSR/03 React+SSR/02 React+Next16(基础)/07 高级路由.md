# 一、动态路由

ext.js也是支持动态路由，并且也是根据目录结构和文件的名称自动生成。

动态路由语法：

 - 页面组件目录支**持 `[ ]` 方括号语法（方括号前后不能有字符串）。**
 - 方括号里编写的字符串就是：动态路由的参数。

例如，动态路由 支持如下写法：

 - `pages/detail/[id]/page.tsx` -> /detail/:id
 - `pages/detail/[role]/[id]/page.tsx` -> /detail/:role/:id
 - `pages/detail-[role]/[id]/page.tsx` -> /detail-:role/:id


# 二、路由参数(useParams)

## 1. `useParams`

**作用:**

- 获取当前路由中的**动态路径参数**。
    
- 例如：对于路由 `app/about/[index]/page.tsx`，当访问 `/about/123` 时，`useParams()` 会返回 `{ index: "123" }`。
    

**使用方式:**

```tsx
"use client";
import { useParams } from "next/navigation";

export default function Page() {
  const params = useParams();
  // params 是一个对象，键为动态片段名，值为参数值（字符串或字符串数组）
  const { id, slug } = params; // 解构多个参数
  return <div>参数: {id}</div>;
}
```

- **返回值类型**：`Record<string, string | string[]>`
    
- **注意**：如果某个动态片段是 `[...catchAll]` 或 `[[...optional]]` 形式，对应的值可能是字符串数组或 `undefined`。

## 2. `useSearchParams`

**作用:**

- 获取当前 URL 中的**查询参数**（`?key=value` 部分）。
    
- 返回一个 `ReadonlyURLSearchParams` 对象，它继承自 `URLSearchParams`，提供了常用的方法来读取查询字符串。
    

**使用方式:**

```tsx
"use client";
import { useSearchParams } from "next/navigation";

export default function Page() {
  const searchParams = useSearchParams();
  const type = searchParams.get("type");     // 获取单个参数
  const allTags = searchParams.getAll("tag"); // 获取同名参数的所有值（如 ?tag=a&tag=b）
  const hasFilter = searchParams.has("filter"); // 判断参数是否存在
  const fullString = searchParams.toString(); // 得到 ?type=foo&tag=a&tag=b
  return <div>type: {type}</div>;
}
```

- **常用方法**：
    
    - `get(key)`：返回第一个匹配的值，若不存在返回 `null`。
        
    - `getAll(key)`：返回所有同名参数值的数组。
        
    - `has(key)`：返回布尔值。
        
    - `toString()`：返回查询字符串（不含 `?` 前缀，如 `"type=foo&tag=a"`）。
        
    - 迭代器方法：`keys()`, `values()`, `entries()`。

## 3. 使用场景与最佳实践

### 仅在客户端组件中使用

这两个钩子**必须**在 `"use client"` 指令下使用，因为它们依赖于浏览器的 `window.location` 对象。如果你需要在服务端组件中访问路由参数，应该使用页面组件的 `params` 和 `searchParams` props（例如在 `app/page.tsx` 中）：

url：http://localhost:3000/cate/11/222

```tsx
import { memo } from "react";
import type { ReactNode, FC } from "react";

interface IProps {
  children?: ReactNode;
  params: Promise<{ index: string }>;
  searchParams: Promise<{ type?: string }>;
}

const AboutIndex: FC<IProps> = async ({ params, searchParams }) => {
  const { index } = await params;
  const { type } = await searchParams;

  return (
    <div>
      AboutIndex的页面
      <div>动态路由参数:{index}</div>
      <div>路由查询参数:{type}</div>
    </div>
  );
};
export default memo(AboutIndex);
```

# 三、404 Page

| 文件                    | 作用                           | 触发场景                                                                                                                                                                                                                                                     |
| --------------------- | ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`not-found.js`**    | 处理 **404** (Not Found) 错误    | 1. 访问了一个完全不存在或没有匹配的路由。  <br>2. 在 Server/Client Component 中调用了 `notFound()` 函数。[](https://nextjscn.org/docs/app/getting-started/error-handling)[](https://nextjscn.org/docs/app/api-reference/file-conventions/not-found#global-not-foundjs-experimental) |
| **`error.js`**        | 处理其他**意外运行时错误** (如 500 错误)   | 在渲染组件或数据获取时抛出了未被捕获的异常。                                                                                                                                                                                                                                   |
| **`global-error.js`** | 处理根布局 (`app/layout.js`) 中的错误 | 当错误发生在根布局内部，且 `error.js` 无法捕获时使用。                                                                                                                                                                                                                        |

## 3.1 创建 404 页面 (`not-found.js`)

最简单的情况是在 `app` 文件夹的根目录下创建一个 `not-found.js` 文件。这将作为整个应用的全局 404 页面。

```tsx
// app/not-found.js
import Link from 'next/link';

export default function NotFound() {
  return (
    <div>
      <h2>404 - 页面走丢了</h2>
      <p>您访问的页面不存在。</p>
      <Link href="/">返回首页</Link>
    </div>
  );
}
```

你还可以在需要的地方**手动触发** 404 页面。例如，在获取数据时，如果找不到对应的数据，就调用 `notFound()` 函数。

```tsx
// app/blog/[slug]/page.js
import { notFound } from 'next/navigation';

async function getPost(slug) {
  const res = await fetch(`/api/posts/${slug}`);
  if (!res.ok) return undefined;
  return res.json();
}

export default async function BlogPost({ params }) {
  const post = await getPost(params.slug);
  // 如果文章不存在，则渲染 not-found.js 文件
  if (!post) {
    notFound();
  }
  // ... 正常渲染文章内容
}
```

> [!tip] **关键点**：
> `notFound()` 函数**必须**在 Server Component 中调用（例如在 `async` 的页面组件或数据获取函数内部），它不能在 `useEffect` 等客户端钩子中调用。

## 3.2 创建通用错误页面 (`error.js`)

`error.js` 用于捕获意料之外的运行时错误，并显示一个友好的回退 UI，防止整个应用崩溃。它利用了 React 的 Error Boundary。

在你想捕获错误的目录下创建 `error.js` 文件（例如在 `app/dashboard/error.js`，只会影响 `dashboard` 及其子路由）。

```tsx
// app/error.js
'use client'; // 重要：error 组件必须是客户端组件

import { useEffect } from 'react';

export default function Error({ error, reset }) {
  useEffect(() => {
    // 你可以将错误上报给错误追踪服务
    console.error(error);
  }, [error]);

  return (
    <div>
      <h2>出错了！</h2>
      <button
        onClick={
          // 尝试重新渲染该路由段，如果成功则恢复
          () => reset()
        }
      >
        重试
      </button>
    </div>
  );
}
```


- **Props**:
    
    - `error`: 抛出的错误对象实例。
        
    - `reset`: 一个函数，用于提示 React 尝试重新渲染错误边界的内容。如果成功，`error` 组件会被替换为正常渲染的内容

## 3.3 创建全局错误页面 (`global-error.js`)

`global-error.js` 处理的是 `error.js` 无法捕获的最外层错误，即根布局 (`app/layout.js`) 中的错误。它需要自己定义 `<html>` 和 `<body>` 标签。

```tsx
// app/global-error.js
'use client';

export default function GlobalError({ error, reset }) {
  return (
    <html>
      <body>
        <h2>全局错误，应用崩溃了！</h2>
        <button onClick={() => reset()}>重试</button>
      </body>
    </html>
  );
}
```

> [!tip] **注意**：
> `global-error.js` 会替换根布局，因此你需要在这里重新导入所有必要的样式和字体依赖
