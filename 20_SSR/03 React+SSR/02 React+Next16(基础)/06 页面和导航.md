# 一、新建页面

Next.js项目页面需在 app目录下新建（ .js, .jsx, .ts, or .tsx ）文件，该文件需导出的React组件。 

Next.js 会根据  app 目录结构和文件名，来自动生成路由，比如：

  - app/page.jsx → / （首页， 一级路由）
 -  app/about.jsx → /about （一级路由）
 -  app/blog/page.jsx →/blog （一级路由）
 -  app/blog/post.jsx →/blog/post (嵌套路由，一级路由)
 -  `app/blog/[slug].jsx` → /blog/:slug （动态路由， 一级路由）

新建页面步骤：

 - 1.新建一个命名为 app/about.jsx 组件文件，并导出（export）React 组件。
 - 2.接着通过 /about 路径，就可访问新创建的页面了。

> [!tip] 注意：
> Nuxt3 需要添加`<NuxtPage>`内置组件占位， Next.js则不需要


# 二、组件导航（Link）

页面之间的跳转需要用到`<Link>`组件，需从 next/link 包导入。
	
`Link`组件底层实现是一个`<a>`标签，所以使用 `a + href` 也支持页面切换（不推荐, 会默认刷新浏览器）

`<Link>`组件属性：

 - href 值类型（不支持 to）
	 - 字符类型：` / 、/home、/about`
	 - 对象类型：`{ pathname：’’ , query : { } }`
	 - URL：外部网址
 - as：在浏览器的 URL 栏中显示的路径的别名。
 - **replace**：**替换当前url页面**，而不是将新的 url 添加到堆栈中。默认为 false
 - target：和a标签的target一样，指定何种方式显示新页面

# 三、编程导航 (useRouter)

Next16除了可以通过<Link>组件来实现导航，同时也支持使用编程导航。

编程导航可以轻松的实现动态导航了，缺点就是不利于SEO。

我们可以从 next/router 中导入 useRouter 函数（或 class 中用 withRouer），调用该函数可以拿到 router对象进行编程导航。

router 对象的方法：

- `push( url [, as , opts] ) `：页面跳转
- `replace( url [, as , opts] )` ：页面跳转（会替换当前页面）
- `back()`：页面返回

> [!tip] 注意
> - 获取router的导入的方式是`import { useRouter } from "next/navigation";`
> - 在服务器端组件中不可以使用 usrRouter
> - 要使用的话，需要进行客户端组件和服务器端组件进行一个分离


服务端组件：

```tsx
import { Metadata } from "next";
import HomeContent from "@/components/home-content";

export const metadata: Metadata = {
  title: "页面",
};

export default function Home() {
  return <HomeContent />;
}

```

客户端组件：

```tsx
'use client';

import XLButton from './home-button/index'
import { useRouter } from "next/navigation";

export default function HomeContent() {
  const router = useRouter();

  return (
    <>
      <h1>home页面</h1>
      <button onClick={() => router.push("/detail")}>detail</button>
      <button>
        <XLButton />
      </button>
    </>
  );
}

```
