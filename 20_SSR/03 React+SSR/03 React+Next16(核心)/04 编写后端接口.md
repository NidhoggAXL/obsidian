Next.js 提供了编写后端接口的功能（即 API Routes），编写接口可以在 **pages/api** 目录下编写

```test
app/
├── api/
│   ├── users/
│   │   └── route.ts          → 处理 /api/users
│   └── products/
│       └── route.ts          → 处理 /api/products
├── api/
│   └── users/
│       └── [id]/
│           └── route.ts      → 处理 /api/users/:id
```

方法定义:

在 `route.ts` 中导出 HTTP 方法对应的函数：

- `GET`
- `POST`
- `PUT`
- `PATCH`
- `DELETE`
- 等等

完整类型定义：

```ts
import { NextRequest } from 'next/server';

type RouteHandler = (
  request: NextRequest,
  context: { params: Promise<{ [key: string]: string | string[] }> }
) => Promise<Response>;
```

使用示例：

1. 静态路由（无动态参数）

```ts
// app/api/hello/route.ts
import { NextRequest, NextResponse } from 'next/server';

export async function GET(request: NextRequest) {
  // 可以访问请求头、URL 等
  const url = request.nextUrl; // Next.js 增强的 URL 对象
  return NextResponse.json({ message: 'Hello, World!' });
}
```

2. 动态路由（带路径参数）

假设文件路径为 `app/api/users/[id]/route.ts`：

```ts
import { NextRequest, NextResponse } from 'next/server';

export async function GET(
  request: NextRequest,
  context: { params: Promise<{ id: string }> } // 类型可根据路由参数定义
) {
  const { id } = await context.params; // 等待 params 解析（Next.js 15+）
  return NextResponse.json({ userId: id });
}
```

**参数详解:**

| 参数        | 类型                                                        | 说明                                                    |
| --------- | --------------------------------------------------------- | ----------------------------------------------------- |
| `request` | `Request` 或 `NextRequest`                                 | 原生请求对象，`NextRequest` 添加了 `nextUrl`、`cookies`、`geo` 等。 |
| `context` | `{ params: Promise<Record<string, string \| string[]>> }` | 包含动态路由参数，如 `[id]`、`[...slug]`。参数值可能是字符串或数组。           |

**返回类型:**

函数必须返回一个 `Response` 对象。可以使用原生 `new Response()`，也可以使用 `NextResponse`（扩展了原生 Response，提供了更多便捷方法如 `json()`、`redirect()` 等）。

> [!tip] 小提示
> - 在 Next.js 15 之前，`params` 是同步对象；从 15 开始，`params` 变为 Promise，需 `await` 获取。
> - 如果使用 `NextRequest`，可以通过 `request.nextUrl` 获取包含路径、查询参数的 URL 对象，并且可以通过 `request.cookies` 方便地操作 Cookie。


总结：

```ts
export async function GET(
  request: NextRequest,
  { params }: { params: Promise<{ id: string }> }
): Promise<Response> {
  const { id } = await params;
  // 处理逻辑...
  return new Response(JSON.stringify({ id }), { status: 200 });
}
```
