

在Nuxt中数据的获取主要是通过下面4个函数来实现（支持Server和Client ）：

# 一、useAsyncData

useAsyncData(key, func):专门解决异步获取数据的函数，**会阻止页面导航**。
	
 - 发起异步请求需用到 **$fetch** 全局函数（类似Fetch API）
 - $fetch(url, opts)是一个类原生fetch的**跨平台请求库**

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/177409848900094wc0v.png)

> [!tip]
> 在组件中直接使用 $fetch 而未使用 useAsyncData 包裹，**会导致数据被获取两次**：一次在服务端，一次在客户端 hydration 过程中。因为 $fetch 不会将服务端获取的数据状态传递到客户端，因此客户端必须再次请求数据。

# 二、useFetch

useFetch(url, opts)：用于获取任意的URL地址的数据，**会阻止页面导航**

- 本质是 useAsyncData(key, () => $fetch(url, opts)) 的语法糖。

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1774098654000zpnyln.png)

## 2.1 拦截器
 
```ts
const { data, status, error, refresh, clear } = await useFetch('/api/auth/login', {
  onRequest({ request, options }) {
    // 设置请求头
    // 注意：这依赖于 ofetch >= 1.4.0，你可能需要刷新你的锁文件
    options.headers.set('Authorization', '...')
  },
  onRequestError({ request, options, error }) {
    // 处理请求错误
  },
  onResponse({ request, response, options }) {
    // 处理响应数据
    localStorage.setItem('token', response._data.token)
  },
  onResponseError({ request, response, options }) {
    // 处理响应错误
  }
})

```

## 2.2 响应式和共享状态

可以使用计算属性 ref 或普通 ref 作为 URL，以实现动态数据获取，并在 URL 更改时自动更新：

```vue
// [id].vue
<script setup lang="ts">
const route = useRoute()
const id = computed(() => route.params.id)

// 当路由更改且 id 更新时，数据将自动重新获取
const { data: post } = await useFetch(() => `/api/posts/${id.value}`)
</script>

```

> [!abstract] 当在多个组件中使用相同的 URL 和选项调用 `useFetch` 时，它们将共享相同的 `data`、`error` 和 `status` ref。这确保了组件之间的一致性。

# 三、useLazyFetch

useLazyFetch(url, opts):用于获取任意URL数据，**不会阻止页面导航**

 - 本质和 useFetch 的 lazy 属性设置为 true 一样

```vue
<script setup lang="ts">
/* 导航将在获取完成之前发生。
 * 直接在组件模板中处理 'pending' 和 'error' 状态
 */
const { status, data: posts } = await useLazyFetch('/api/posts')
watch(posts, (newPosts) => {
  // 因为 posts 可能初始为 null，您无法立即访问其内容，
  // 但您可以监视它。
})
</script>

<template>
  <div v-if="status === 'pending'">
    加载中 ...
  </div>
  <div v-else>
    <div v-for="post in posts">
      <!-- 进行操作 -->
    </div>
  </div>
</template>

```

# 六、useFetch vs axios

获取数据Nuxt推荐使用 useFetch 函数，为什么不是 axios ?

 - useFetch 底层调用的是$fetch函数，该函数是基于unjs/ohmyfetch请求库，并与原生的Fetch API有者相同API
 - unjs/ohmyfetch 是一个跨端请求库： A better fetch API. Works on node, browser and workers
	 - 如果运行在服务器上，它可以智能的处理对 API接口的直接调用。
	 - 如果运行在客户端行，它可以对后台提供的 API接口 正常的调用（类似 axios），当然也支持第三方接口的调用
	 - 会自动解析响应和对数据进行字符串化
 - useFetch 支持智能的类型提示和智能的推断 API 响应类型。
 - 在setup中用useFetch获取数据，会减去客户端重复发起的请求。

useFetch（url, options）语法

 - url: 请求的路径
 - options：请求配置选项
	 - method、query ( 别名 params )、body、headers、baseURL
	 - onRequest、 onResponse、lazy....
 - 返回值： data, pending, error, refresh

# 七、useFetch封装

封装useFetch步骤

 - 1.定义XLRequest类，并导出
 - 2.在类中定义request、get、post方法
 - 3.在request中使用useFetch发起网络请求
 - 4.添加TypeScript类型声明

```ts
const BASE_URL = "http://www.baidu.com";

type Methods = "GET" | "POST" | "PUT" | "DELETE";
import type { AsyncData, UseFetchOptions } from "nuxt/app";

class XLRequest {
  request<T = any>(
    url: string,
    method: Methods,
    data?: any,
    options?: UseFetchOptions<T>,
  ): Promise<AsyncData<T, Error>> {
    let newOptions = {
      method: method || "GET",
      baseURL: BASE_URL,
      ...options,
    };
    if (method == 'GET') {
      newOptions.query = data || {}
    }
    if (method == "POST") {
      newOptions.body = data || {}
    }
    return new Promise((resolve, reject) => {
      useFetch<T>(url, newOptions as any)
        .then((res) => {
          resolve(res as AsyncData<T, Error>);
        })
        .catch((err) => {
          reject(err);
        });
    });
  }
}

export default new XLRequest();

```

## 7.1 基础类型定义

```ts
const BASE_URL = "http://www.baidu.com";
```

BASE_URL: 常量字符串类型，存储 API 请求的基础 URL

TypeScript 会自动推断为 const BASE_URL: "http://www.baidu.com" 字面量类型

```ts
type Methods = "GET" | "POST" | "PUT" | "DELETE";
```

Methods: 联合类型，限制 HTTP 请求方法只能是这四种之一

提供类型安全，防止传入无效的 HTTP 方法

使用场景：在 request 方法的 method 参数中使用

## 7.2 导入的 Nuxt 类型

```ts
import type { AsyncData, UseFetchOptions } from "nuxt/app";
```

`AsyncData<T, Error>`: Nuxt 的异步数据响应类型

 - T: 响应数据的类型泛型
 - Error: 错误类型
 - 包含 data, pending, error, refresh 等属性

`UseFetchOptions<T>`: Nuxt 的 useFetch 配置选项类型

 - T: 响应数据的类型泛型
 - 包含 method, query, body, headers 等配置选项

## 7.3 类方法类型

```ts
class XLRequest {
  request<T = any>(
    url: string,
    method: Methods,
    data?: any,
    options?: UseFetchOptions<T>,
  ): Promise<AsyncData<T, Error>>

```


**泛型参数 `<T = any>`:**

 - T: 类型参数，默认值为 any
 - 用于表示响应数据的类型
 - 调用时可以指定具体类型，如 `request<UserData>(...)`

**参数类型:**

 - `url: string`
	 - 请求 URL，必须是字符串类型
 - method: Methods
	 - 请求方法，只能是 "GET" | "POST" | "PUT" | "DELETE" 之一
	 - 使用之前定义的 Methods 联合类型
 - data?: any
	 - 可选参数（? 表示可选）
	 - 类型为 any，表示可以是任意类型
	 - 用于传递请求参数或请求体
 - `options?: UseFetchOptions<T>`
	 - 可选参数
	 - 类型为 Nuxt 的 UseFetchOptions，使用泛型 T 保持类型一致性
	 - 用于传递额外的 fetch 配置选项

**返回类型 `Promise<AsyncData<T, Error>>`**

 - 返回一个 Promise
 - Promise 解析后的值为 AsyncData<T, Error> 类型
 - T 与方法泛型参数保持一致
 - Error 表示可能的错误类型

## 7.4 方法内部类型

```ts
let newOptions = {
  method: method || "GET",
  baseURL: BASE_URL,
  ...options,
};

```

- newOptions: 对象类型，TypeScript 自动推断
- 包含 method (string), baseURL (string) 和 options 的所有属性

```ts
if (method == 'GET') {
  newOptions.query = data || {}
}

```

 - newOptions.query: 添加查询参数属性
 - 类型为 any（来自 data 参数）

```ts
if (method == "POST") {
  newOptions.body = data || {}
}

```

 - newOptions.body: 添加请求体属性
 - 类型为 any（来自 data 参数）


## 7.5 Promise 类型

```ts
return new Promise((resolve, reject) => {
  useFetch<T>(url, newOptions as any)
    .then((res) => {
      resolve(res as AsyncData<T, Error>);
    })
    .catch((err) => {
      reject(err);
    });
});

```

 - `new Promise(...)`: 创建新的 Promise 对象
 - `resolve`: 函数类型，接受 AsyncData<T, Error> 类型的参数
 - `reject`: 函数类型，接受 Error 类型的参数
 - `useFetch<T>(...)`: Nuxt 的 fetch 函数，使用泛型 T
 - `as any`: 类型断言，临时将类型视为 any
 - `as AsyncData<T, Error>`: 类型断言，明确指定返回类型
