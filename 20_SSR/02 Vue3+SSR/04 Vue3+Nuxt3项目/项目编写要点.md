# 一、vue组件获取参数

```vue
<script lang="ts" setup>
// js编写
const props = defineProps({
  title: {
    type: String,
    default: ""
  }
})

// ts编写
// 1.无法设置default默认值
interface IPros {
  title: string
}
const props = defineProps<IPros>()
// 2.可以设置default默认值
const props = withDefaults(defineProps<IPros>(), {
  title: ""
})

console.log(props.title)
</script>

<template>
  <div class="example">
    <h1>home</h1>
  </div>
</template>

<style lang="less" scoped></style>

```

## 3.2 PM2

PM2 常用命令和配置文件: https://pm2.keymetrics.io/docs/usage/application-declaration/

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1774341062000t2ryid.png)


正常如果只是启动一个进程，如果这一个进程挂了，那么整个网站就挂，所以可以启动一个 PM2 的负载均衡，启动多个进程来维护同一个项目：

1. 在当前项目目录下面生成 PM2 的配置文件

```shell
pm2 init simple
```

2. 在 PM2 配置文件里面进行配置

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1774341276000f7ept3.png)

- name 项目的名称
- script 项目的入口
- env 项目的环境配置
	- POST 项目启动的端口
- 等同于`POST=8080 pm2 start ./.putput/server/index.mjs --name oppo-nuxt-web`

3. 执行配置

```shell
pm2 start ecosystem.config.js
```

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1774341474000db3m88.png)


4. 启动 pm2 的负载均衡模式，需要对 pm2 的配置文件进行配置

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1774341614000xk9bt9.png)

这样启动项目的时候就会得到如下图，如果其中一个实例挂了后，其他实例只要还没有挂就可以正常访问网页：
