# 一、认识泛型

软件工程的主要目的是构建不仅仅明确和一致的API，还要让你的代码具有很强的可重用性： 

- 比如我们可以通过函数来封装一些API，通过传入不同的函数参数，让函数帮助我们完成不同的操作； 
- 但是对于参数的类型是否也可以参数化呢？ 

什么是类型的参数化？ 我们来提一个需求：**封装一个函数，传入一个参数，并且返回这个参数；**

```ts
function returnItem(item) {
  return item
}
```

如果我们是TypeScript的思维方式，要考虑这个参数和返回值的类型需要一致：

```ts
function returnItem(item: number): number {
  return item
}
```
上面的代码虽然实现了，但是不适用于其他类型，比如string、boolean、Person等类型：

```ts
function returnItem(item: any): any {
  return item
}
```

# 二、泛型实现类型参数化

**虽然any是可以的，但是定义为any的时候，我们其实已经丢失了类型信息：** 

- 比如我们传入的是一个number，那么我们希望返回的可不是any类型，而是number类型； 
- 所以，我们需要在函数中可以捕获到参数的类型是number，并且同时使用它来作为返回值的类型；

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1760514871000d7cnvo.png)


我们需要在这里使用一种特性的变量 - **类型变量（type variable）**，它作用于类型，而不是值：

```ts
function returnItem<Type>(item: Type): Type {
  return item;
}
```

这里我们可以使用两种方式来调用它： 

- 方式一：通过 `<类型>` 的方式将类型传递给函数
- 方式二：通过**类型推导(type argument inference)**，自动推导出传入变量的类型
	- const 得到的是更加具体的字面量类型
	- let 得到是不太具体的普通类型

```ts
function returnItem<Type>(item: Type): Type {
  return item;
}

//1.完整的语法编写(显示确定的类型，返回确定的类型)
const item1 = returnItem<number>(1)
//得到的类型是 const item1: number
let item2 = returnItem<string>('hello')
//得到的类型是 let item2: string

//2.类型推断(不显示确定的类型，返回推断的类型)
//  2.1 const 得到更加具体的字面量类型
const item3 = returnItem(1)
//得到的类型是 const item3: 1
const item4 = returnItem('hello')
//得到的类型是 const item4: 'hello'

//  2.2 let 得到不太具体的类型
let item5 = returnItem(1)
//得到的类型是 let item5: number
let item6 = returnItem('hello')
//得到的类型是 let item6: string
```

# 四、泛型的基本练习

实现 React 中 useState 函数，要求有类型约束：

```ts
function useState<Type>(initialState: Type): [Type, (newState: Type) => void] {
  let state = initialState
  function setState(newState: Type) {
    state = newState
  }
  return [state, setState]
}

const [count, setCount] = useState(100)
//count的类型 const count: number
//setCount的类型 const setCount: (newState: number) => void

const [name, setName] = useState('张三')
//name的类型 const name: string
//setName的类型 const setName: (newState: string) => void

const [names, setNames] = useState(['张三', '李四', 123])
//names的类型 const names: (string | number)[]
//setNames的类型 const setNames: (newState: (string | number)[]) => void

const [user, setUser] = useState({ name: '张三', age: 18 })
//user的类型 const user: { name: string; age: number; }
//setUser的类型 const setUser: (newState: { name: string; age: number; }) => void

//因为数组是为空的（没有值），所以类型是never，不确定的类型
const [friends, setFriends] = useState([])
//friends的类型 const friends: never[]
//setFriends的类型 const setFriends: (newState: never[]) => void

//下面显示的确定类型
const [length, setLength] = useState<any[]>([])
//length的类型 const length: any[]
//setLength的类型 const setLength: (newState: any[]) => void
```


# 三、多类型和常用名称

当然也可以同时传入多个类型：

```ts
function foo<Type1, Type2>(a1: Type1, a2: Type2) {
  console.log(typeof a1); //number
  console.log(typeof a2); //string
}

foo<number, string>(1, '2')
```

平时在开发中我们可能会看到一些常用的名称： 

- T：Type的缩写，类型 
- K、V：key和value的缩写，键值对 
- E：Element的缩写，元素 
- O：Object的缩写，对象
