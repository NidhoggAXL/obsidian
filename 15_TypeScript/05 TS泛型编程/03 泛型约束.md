# 一、泛型约束的基本使用

有时候我们希望**传入的类型有某些共性**，但是**这些共性可能不是在同一种类型中**： 

- 比如string和array都是有length的，或者某些对象也是会有length属性的； 
- 那么只要是拥有length的属性都可以作为我们的参数类型，那么应该如何操作呢？

```ts
interface ILength {
  length: number
}

//1.不使用泛型约束
function logLength(arg: ILength) {
  return arg
}

//失去了传入logLength参数中属性本身的类型
const item1 = logLength('123')
//item1的类型呢 const item1: ILength
const item2 = logLength([1, 2, 3])
//item2的类型呢 const item2: ILength

//2.使用泛型约束
//使用泛型约束后，传入getLength的参数类型会保留
//T extends ILength表示T类型必须满足ILength类型,其中有length属性
//这样就可以保证传入的参数一定有length属性
function getLength<T extends ILength>(arg: T) {
  return arg
}

const item3 = getLength('123')
//item3的类型呢 const item3: "123"
const item4 = getLength([1, 2, 3])
//item4的类型呢 const item4: number[]
let item5 = getLength("123")
//item5的类型呢 let item5: "123"


//类型“number”的参数不能赋给类型“ILength”的参数
//报错，因为number没有length属性
// const item6 = getLength(123)
```



> [!note]
> 这里表示是传入的类型必须 length 有这个属性，也可以有其他属性，但是必须至少有这个成员。


# 二、泛型约束中使用类型参数

在泛型约束中使用类型参数（Using Type Parameters in Generic Constraints） ：可以声明一个类型参数，**这个类型参数被其他类型参数约束**；

举个栗子：我们希望获取一个对象给定属性名的值

- 我们需要确保我们不会获取 obj 上不存在的属性；
- 所以我们在两个类型之间建立一个约束；

```ts
//没有泛型的类型的约束的条件下
function getValue(obj: any, key: string) {
  return obj[key];
}

const obj = {
  name: '张三',
  age: 18,
  height: 18.8
}

//下面两个都不会报错
const item1 = getValue(obj, 'name');
//就算 obj 里面没有 address属性也不会报错
const item2 = getValue(obj, 'address');
```

```ts
//为泛型的类型添加约束
//在泛型约束中使用类型约束
interface IPerson {
  name: string
  age: number
}

type Ikun = keyof IPerson
//推导出的Ikun类型是type Ikun = keyof IPerson
//实质是 type Ikun = "name" | "age"
//联合了 IPerson 接口中的所有键，所以Ikun类型是"name" | "age"

function getValue<O, K extends keyof O>(obj: O, key: K) {
  return obj[key];
}

const obj = {
  name: '张三',
  age: 18,
  height: 18.8
}

const item1 = getValue(obj, 'name');
//item1 的类型是 const item1: string
//会根据obj[key]的值推断出item1的类型

// const item2 = getValue(obj, 'address');//报错
//类型“"address"”的参数不能赋给类型“"name" | "age" | "height"”的参数。
```
