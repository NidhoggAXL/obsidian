# 一、映射类型

**有的时候，一个类型需要基于另外一个类型，但是你又不想拷贝一份，这个时候可以考虑使用映射类型。**

- 大部分内置的工具都是通过映射类型（Mapped Types）来实现的；
- 大多数类型体操的题目也是通过映射类型完成的；


映射类型建立在[[03 TS对象和类的类型#2.2 索引签名|索引签名]]的语法上：

- 映射类型，就是使用了 PropertyKeys 联合类型的泛型；
- 其中 PropertyKeys 多是通过 keyof 创建，然后循环遍历键名创建一个类型；

```ts
interface IPerson {
  name: string
  age: number
}

type ReadonlyPerson<T> = {
  [index in keyof T]: T[index]
}

type newPerson = ReadonlyPerson<IPerson>
//newPerson的类型为
/*
type newPerson = {
  name: string;
  age: number;
}
**/
```

# 二、映射类型修饰符

在使用映射类型时，有两个额外的修饰符可能会用到：

- 一个是 **readonly**，用于设置属性只读；
- 一个是 **`?`** ，用于设置属性可选；

你可以通过前缀 `-` 或者 `+` 删除或者添加这些修饰符，如果没有写前缀，**相当于使用了 `+` 前缀。**

默认添加修饰符：

```ts
interface IPerson {
  name: string;
  age: number;
  height: number;
}

type MapIperson<T> = {
  readonly [index in keyof T]?: T[index]
}
//等价于,显示添加 + 
// type MapIperson<T> = {
//   +readonly [index in keyof T]+?: T[index];
// };

type newIperson = MapIperson<IPerson>;
//type newIperson的类型添加了readonly和?修饰符
/*
type newIperson = {
    readonly name?: string | undefined;
    readonly age?: number | undefined;
    readonly height?: number | undefined;
}
**/
```

使用 - 修饰符：

```ts
interface IPerson {
  name: string;
  age?: number;
  readonly height: number;
}

//使用 - 修饰符
type MapIperson<T> = {
  -readonly [index in keyof T]-?: T[index]
}


type newIperson = MapIperson<IPerson>;
//type newIperson的类型删除了readonly和?修饰符
/*
type newIperson = {
  name: string;
  age: number;
  height: number;
}
**/
```
