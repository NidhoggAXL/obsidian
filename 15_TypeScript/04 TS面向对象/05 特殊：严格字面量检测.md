# 一、特殊现象

对于对象的字面量赋值，在TypeScript中有一个非常有意思的现象：

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1760439007000h5u973.png)

同样都是把对象赋值给 info，但是进行赋值的时候左边是通过变量赋值，左边是直接赋值。

# 二、为什么会出现这种情况？

引入TypeScript成员在GitHub的issue中的回答：

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/176043911200050o1ve.png)

简单对上面的英文进行翻译解释： 

- 每个对象字面量最初都被认为是“**新鲜的（fresh）**”。 
- 当一个新的对象字面量分配给一个变量或传递给一个**非空目标类型的参数**时，对象字面量指定目标类型中**不存在的属性是错误**的。 
- 当类型断言或对象字面量的**类型扩大时**，**新鲜度**会消失。

```ts
interface Iperson {
  name: string;
  age: number;
}

//最初的obj是有新鲜度的
const obj = {
  name: "李四",
  age: 18,
  height: 1.88,
};

//赋值给info的时候，obj失去了新鲜度
const info: Iperson = obj; // ✅ 不报错
```
> [!note]
> **原因**：当使用中间变量 `obj` 时，TypeScript 会进行**结构类型兼容性检查**。`obj` 的类型包含 `Iperson` 的所有必需属性（`name` 和 `age`），因此赋值是允许的。多余的属性 `height` 被忽略。

```ts
interface Iperson {
  name: string;
  age: number;
}

const info: Iperson = {
  name: "李四",
  age: 18,
  height: 1.88, // ❌ 报错
};
```

> [!note]
> **原因**：当直接使用对象字面量赋值时，TypeScript 会进行**更严格的检查**（freshness check），不允许出现目标类型中不存在的属性。

那么 TS 为什么要这样设计呢？TS 这样的设计哲学是什么呢？来看下面这段代码：

```ts
interface Iperson {
  name: string;
  age: number;
}

const obj = {
  name: "李四",
  age: 18,
  height: 1.88,
};

function person(person: Iperson) {
  console.log(person.name, person.age)
}
person(obj);//李四 18
```

这里的类型检测之所以不报错，就是因为 obj 通过 person(obj) 的类型是扩大的，所以在函数 Iperson 的类型检测的时候，是不报错的。

那么，如果报错的话，就比较麻烦啦，函数的参数类型，必须更具传入的对象具有不同的属性，这是不同的函数，大大增加了代码量，以及函数的通用性降低。

例如如果都要编写，下面的代码就会很麻烦：

```ts
//为了没有height创建一个属性和方法相同的接口
interface Iperson_1 {
  name: string;
  age: number;
}
const obj1 = {
  name: "李四",
  age: 18,

};
function person_1(person: Iperson_1) {
  console.log(person.name, person.age)
}
person_1(obj1);

//为了height创建一个属性和方法相同的接口
//即使没有用过 height 属性
interface Iperson_2 {
  name: string;
  age: number;
  height: number;
}
const obj2 = {
  name: "王五",
  age: 19,
  height: 1.88,
}
function person_2(person: Iperson_2) {
  console.log(person.name, person.age)
}
person_2(obj2);
```

如果利用好 TS 的这种特殊的规则，就可以编写下面的代码，大大简洁：

```ts
interface Iperson {
  name: string;
  age: number;
}

const obj = { name: "axl", age: 18, height: 1.88 };

const obj_1 = { name: 'axl', age: 18 }
const obj_2: Iperson = obj

function person(person: Iperson) {
  console.log(person.name, person.age)
}
person(obj_1)
person(obj_2) 
```

> [!note]
> 即使 obj_2 里面存在 height 属性，但是函数可以接收属性，并对 Iperson 定义的属性进行操作。
