# 一、认识抽象类

继承是多态使用的前提：

- 所以在定义很多通用的调用接口时, 通常会让调用者传入父类，通过多态来实现更加灵活的调用方式。 
- 但是，**父类本身可能并不需要对某些方法进行具体的实现**，所以父类中定义的方法,，我们可以定义为抽象方法。 

什么是 **抽象方法**? 在TypeScript中没有具体实现的方法(没有方法体)，就是抽象方法。 

- 抽象方法，必须存在于抽象类中； 
- 抽象类是使用abstract声明的类；

```ts
//abstract定义抽象类
abstract class getArea {
  //abstract定义抽象方法
  abstract getArea(): number
}
```

**抽象类有如下的特点**： 

- 抽象类是不能被实例的话（也就是不能通过new创建） 
- 抽象类可以包含抽象方法，也可以包含有实现体的方法； 
- 有抽象方法的类，必须是一个抽象类； 
- **抽象方法必须被子类实现，否则该类必须是一个抽象类；**

# 二、抽象类的使用

需求是设计一个通用函数，这个函数要可以计算三角形、圆形、矩形的方法：

- 第一种：可以使用联合类型来编写，更具不同的类型，使用不同的实现，但是如果后面需要添加其他需要进行计算的图形，那么其实是改变了这个通用函数。
- 第二种：设计一个抽象类，父类引用子类的方法，父类设计方法调用，具体的事先交给子类，这样如果需要添加新的图形需要，只需要继承父类，并在自己的类里面编写方法就好啦，不会修改那个通用函数。
	- 这种方式在 java 中非常常用。

```ts
//abstract定义抽象类
abstract class Shape {
  //abstract定义抽象方法
  abstract getArea(): number

  //抽象类包含实现体方法
  running() {
    console.log("running")
  }
}

//普通类继承抽象类（圆形实现）
class Circle extends Shape {
  constructor(private r: number) {
    //必须要有super，即便是继承抽象类
    super();
    this.r = r;
  }

  //子类必须实现抽象类中的抽象方法
  getArea(): number {
    return Math.PI * this.r ** 2;
  }

  //继承抽象类的类可以编写自己的类方法
  swiming() {
    console.log("swiming");
  }
}

//获取circle实例
const circle = new Circle(10)

function cacleArea(shape: Shape) {
  console.log(shape.getArea());
}

cacleArea(circle);//314.1592653589793

//circle调用了Circle类中继承Shape抽象类的running方法
circle.running();//running

//circle调用了Circle类中自己编写的swiming方法
circle.swiming();//swiming
```
