# 一、类的类型

类的作用:

- 可以创建类对应的实例对象
- 类本身可以作为这个实例的类型
- 类也可以当中有一个[[06 函数的类型和函数签名|构造签名]]的函数

```ts
class Person { }

//创建类对应的实例
const p = new Person()

//类作为实例类型
function printName(p: Person) { }
printName(p)

//类的构造签名函数
function foo(ctor: new () => void) {}
foo(Person)
```

# 二、对象类型

## 2.1 对象类型修饰符

对象类型中的每个属性可以说明它的**类型、属性是否可选、属性是否只读**等信息。 

可选属性（Optional Properties）：我们可以在**属性名后面加一个 `?` 标记表**示这个属性是可选的； 

只读属性（Readonly Properties）：

- 在 TypeScript 中，属性可以被**标记为 readonly**，这不会改变任何运行时的行为； 
- 但在类型检查的时候，一个标记为 readonly的属性是**不能被写入**的。

```ts
interface Person{
  name: string
  age ?: number
  readonly height: number//只读属性
}

const p: Person = {
  name: '张三',
  height: 1.88
}
// p.height = 1.88 报错
```

## 2.2 索引签名

### 2.2.1 基本使用

什么是索引签名呢？ 

- 有的时候，你不能提前知道一个类型里的所有属性的名字，但是你知道这些值的特征； 
- 这种情况，你就可以用一个**索引签名 (index signature) 来描述可能的值的类型**；

```ts
interface ICollection {
  [index: number]: string
  length: number
}

function printCollection(collection: ICollection) {
  for (let i = 0; i < collection.length; i++) {
    console.log(collection[i])
  }
}

const names = ['abc', 'cba', 'nba']
const tuple: [string, string] = ['321', '123']
printCollection(names)
printCollection(tupl)
```

#### 属性类型问题一

一个索引签名的属性类型必须是 string、number、symbol或者模板文本类型中的一个。

```ts
//string类型
interface ICollection {
  [index: string]: string
}
```

```ts
//number类型
interface ICollection {
  [index: number]: string
}
```

```ts
//symbol类型
interface ICollection {
  [index: symbol]: string
}
const names: ICollection = {
  [Symbol()]: "abc",
  [Symbol()]: "cba",
  [Symbol()]: "nba"
}
```

#### 属性类型问题二

**索引签名的返回值：隐藏的属性类型**

第一种类型编写方式：(正确)

```ts
interface ICollection {
  [index: number]: string;
}

const names: ICollection = ["abc", "cba", "nba"];
```

第二种编写方式：(正确)

```ts
interface ICollection {
  [index: number]: any
}

const names: ICollection = ["abc", "cba", "nba"];
```

第三种编写方式：(正确)

```ts
interface ICollection {
  [index: string]: any
}

const names: ICollection = ["abc", "cba", "nba"];
```

> [!tip] 前面的三种方式为什么编写正确呢？
> 
index的类型，一定要知道在 JS 和 TS 中，`names[0]` 调用的本质是装换为 `name['0']` 来调用的。所以 index 的类型是 number 或者 string 都是正确的。

**第四种编写方式：(错误)**

```ts
interface ICollection {
  [index: string]: string
}

const names: ICollection = ["abc", "cba", "nba"];
```

> [!tip] 会什么会报这种错误呢？
> 前面的 index 的类型为 string ，肯定是正确的，错误的是后面 **索引签名** 返回的 string 类型的错误。
> 
> ![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/17604441780003wqbms.png)
> 
> 这里的 names 本质是一个数组实例对象，他**本身还有其他的一些属性**，比如 forEach 、map等属性，当 `names['forEach']` 放回的是一个 **函数类型**，并不是 string 类型，所以就会报错。
> 
> 这也是为什么第三种编写方式 `[index: string]: any` 会是正确的本质原因，any 就不会对返回的类型进行检测。

### 2.2.2 双索引签名

#### 基本使用

同时编写两个 索引签名：不过需要编写不同的索引名称。

```ts
interface ICollection {
  [index: string]: string | number
  [key: number]: string
}
```

#### 返回类型问题一

虽然 TypeScript 可以同时支持 string 和 number 类型，但**数字索引的返回类型一定要是字符索引返回类型的子类型**；

```ts
//正确编写
interface ICollection {
  [index: string]: string | number
  [key: number]: string
}

//错误编写
interface ICollection {
  [index: string]: string;
  [index: number]: string | number;//错误
}
```

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/17604467250008twrw1.png)

> [!tip] 为什么要这样设计呢？
>例如是一个数组的实例对象，里面还存在着 forEach 的属性，当调用的时候，如果是 `['forEach']` 返回的是 **函数类型**，那么从整体的来看，number 类型的返回的类型是没有 string 类型返回的类型的多的，（可以理解为，number类型放回的类型其实是string类型放回类型的子类），所以**数字索引的返回类型一定要是字符索引返回类型的子类型**


#### 返回类型问题二

如果索引签名中有**定义其他属性**，其他属性返回的类型，必须将合索引签名中string类型返回的属性

```ts
interface ICollection {
  [index: string]: string | number;
  [key: number]: string;

  // 索引签名定义其他属性
  name: string | number;
  age: number;
}
```

如果不符合string类型返回的属性，就会报错：

```ts
//下面代码会报错的
interface ICollection {
  [index: string]: string | number;
  [key: number]: string;

  // 索引签名定义其他属性
  name: boolean
}
```

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1760447374000ofsz9g.png)


> [!tip] 为什么会出现这种情况呢？
> 这是因为，当满足了 name: boolean 的属性的同时，也满足了索引前面的 string 属性，所以需要两个都满足，所以 name 的类型，也必须是缩影前面类型的子类型。
