# 一、自定义声明

什么情况下需要自己来定义声明文件呢？

- 情况一：我们使用的**第三方库是一个纯的JavaScript库**，没有对应的声明文件；比如lodash
- 情况二：我们**给自己的代码中声明一些类型**，方便在其他地方直接进行使用；

首先我们自己使用 JS 开发了一个库，里面包含变量、函数、类：

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/17606865870006rf3my.png)


然后自定义声明文件，一般设置到 type 的文件中，为我自己编写的 JS 添加类型声明：

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/17606854810007su4hl.png)


后面在 TS 文件里面使用，就可以找到 JS 里面编写的内容，并且有对应的类型：

```ts
console.log(wName)//coderwhy
wFoo()//wFoo
const student = new Person('axl', 18)
console.log(student)//{name: 'axl', age: 18}
```


# 二、declare声明模块

可以声明模块，比如 lodash 模块默认不能使用的情况，可以**自己来声明这个模块**：

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1760687105000qe9g07.png)

自己配置一个 lodash 的声明文件：在 XLlodash.d.ts 里面配置，这样就对 lodash 模块进行了声明

```ts
declare module "lodash" { }
```

但是 lodash 里面，还有一些方法，比如 join 方法，编写的时候，是会报错的。

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/17606874700000jpngm.png)


这个时候也，需要对 lodash 模块里面的 join 方法也需要进行类型的声明并导出：

```ts
//对lodash模块进行类型声明
declare module "lodash" { 
  //对join函数进行类型声明
  export function join(args: any[]): any;
}
```

> [!tip]
> **声明模块的语法**: `declare module '模块名' {}`。
> 
> 在声明模块的内部，我们可以通过 export 导出对应库的类、函数等；
