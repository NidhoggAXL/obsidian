# 一、TS模块化

JavaScript 有一个很长的处理模块化代码的历史，TypeScript 从 2012 年开始跟进，现在已经实现支持了很多格式。但是随着时间流逝，社区和 JavaScript 规范已经使用为名为 [[05 ES Module用法详解|ES Module]] 的格式，这也就是我们所知的 import/export 语法。

 - ES 模块在 2015 年被添加到 JavaScript 规范中，到 2020 年，大部分的 web 浏览器和 JavaScript 运行环境都已经广泛支持。
- 所以在**TypeScript中最主要使用的模块化方案就是ES Module**；

```ts
export function sum(num1: number, num2: number): number {
  return num1 + num2
}
```

# 二、非模块

需要先理解 TypeScript 认为什么是一个模块。

- JavaScript 规范声明任何**没有 export 的 JavaScript 文件都应该被认为是一个脚本，而非一个模块**。
- 在一个脚本文件中，变量和类型会被声明在**共享的全局作用域**，将多个输入文件合并成一个输出文件，或者在 HTML使用多个 `<script>` 标签加载这些文件。

如果你有一个文件，现在没有任何 import 或者 export，但是你希望它被作为模块处理，添加这行代码：

```ts
export {}
```

> [!note]
> 这会**把文件改成一个没有导出任何内容的模块**，这个语法可以生效，无论你的模块目标是什么。

# 三、内置类型导入

TypeScript 4.5 也允许单独的导入

```ts
import { IPerson, TAnimal } from './utils/type'

const student: IPerson = {
  name: 'axl',
  age: 18,
}

const animal: TAnimal = 'dog'

console.log(student, animal)
//{ name: 'axl', age: 18 } dog
```

还有一种编写方式，**也是推荐的编写方式**，需要使用 type 前缀 ，表明被导入的是一个类型：不管是使用 interface 或者 type 定义的类型，都是使用 type 为前缀

```ts
//第一种编写方式，在导入的类型前面添加 type 关键字
// import { type IPerson, type TAnimal } from './utils/type'

//第二种编写方式，使用 import type 导入类型
//要求导入的全部都是类型
import type{ IPerson, TAnimal} from './utils/type'


const student: IPerson = {
  name: 'axl',
  age: 18,
}

const animal: TAnimal = 'dog'

console.log(student, animal)
//{ name: 'axl', age: 18 } dog
```

> [!tip] 那为什么推荐使用 type 关键字呢？
> 
> 可以让一个非 TypeScript 编译器比如 Babel、swc 或者 esbuild 知道什么样的导入可以被安全移除。
> 
> 因为最终 TS 会转化为 JS ，所以可以显示的告诉类似 Babel 这样的编译器，这个是导入的类型，可以安全的移除。
