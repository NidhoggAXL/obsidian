# 一、函数的类型

## 1.1 TS函数类型

在JavaScript开发中，函数是重要的组成部分，并且函数可以**作为一等公民**（可以作为参数，也可以作为返回值进行传递）。 

那么在使用函数的过程中，函数是否也可以有自己的类型呢？ 

我们可以**编写函数类型的表达式（Function Type Expressions），来表示函数类型**；

```ts
type CalcFunc = (numl: number, num2: number) => void;
function calc(fn: CalcFunc) {
  console.log(fn(20, 30));
}
function sum(num1: number, num2: number) {
  return num1 + num2;
}
function mul(num1: number, num2: number) {
  return num1 * num2;
}

calc(sum);
calc(mul);
//匿名函数,不需要指定类型，会更具上下文类型自行推导
calc(function (num1, num2) {
  return num1 - num2;
})
```

## 1.2 TS函数类型解析

在上面的语法中 `(num1: number, num2: number) => void`，代表的就是一个函数类型： 

- 接收两个参数的函数：num1和num2，并且都是number类型； 
- 并且这个函数是没有返回值的，所以是void；

> [!tip]
> 注意：在某些语言中，可能参数名称num1和num2是可以省略，但是TypeScript是不可以的：
> ![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1760152653000bogtif.png)

比如在C语言中：

```c
function foo(num1, num2) {}
//num1 和 num2 的参数的类型就是省略的
```

## 1.3 TS传入函数类型的参数个数

向看下面的代码？为什么**不报错**

```ts
type CaleType = (num1: number, num2: number) => number
function scal(scalFn: CaleType) {
  
}

scal((num1) => {
  return 123
})
```
定义了 CaleType 类型给 scalFn，但是我们调用scal的时候传入的函数只有一个参数，并不是 CaleType 规定的两个参数，那再来看下面的js内置的高阶函数代码

```ts
const name = ['abc', 'cba', 'nba']
name.forEach((item) => {
  console.log(item)
})
```

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1760255473000xwk0ru.png)

根据 TS 的提示可以看到，forEach 也是有一个回调函数，这个回调函数也是有 3 个参数的类型，但是我们并没有使用 3 个参数，只使用了 value: string 这个参数。

> [!tip]
> 为什么会这样呢？
> 
> 这样因为 TS 的内部规则，**TypeScript对于传入的函数类型的多余参数会被忽略**
> 
> 方便于程序员的编写，不然如果这些要检测的话，那么不管有没有用到后面的参数，都需要进行编写，让 TS 的使用变得不方便。

如果类似于下面代码的使用，那会反而变得麻烦

```ts
//只用到了 item，但是 index， ary 都必须编写
const name = ['abc', 'cba', 'nba']
name.forEach((item, index, arg) => {
  console.log(item)
})
```

官方文档说明：https://www.typescriptlang.org/docs/handbook/2/functions.html#optional-parameters-in-callbacks
# 二、函数签名(signatures)

## 2.1 调用签名

在 JavaScript 中，函数除了可以被调用，自己也是可以有属性值的。 

- 然而前面讲到的函数类型表达式**并不能支持声明属性**； 
- 如果我们想描述一个带**有属性的函数**，我们可以在一个对象类型中写一个**调用签名（call signature）**；
	- 就是把函数看作为一个对象，那么如何定义他的属性类型，和他函数调用的属性呢？
	- 函数类型表达式就不可以完全的表达，这个时候就有了**调用签名**

```ts
interface IScale {
  name: string,
  age: number,
  //函数可以调用，调用函数签名
  (action: string): void
}

function scale(scaleFn: IScale) {
  console.log(scaleFn.name)
  scaleFn('eat')
}
```

注意这个语法跟函数类型表达式稍有不同，在参数列表和返回的类型之间用的是 `:` 而不是  =>

>[!tip] 开发中如何选择:
>1. 如果只是描述函数类型本身(函数可以被调用)，使用函数类型表达式(Function Type Expressions)
>2. 如果在描述函数作为对象可以被调用,同时也有其他属性时，使用函数调用签名(Call·Signatures)

## 2.2 构造签名

JavaScript 函数也可以使用 new 操作符调用，当被调用的时候，TypeScript 会认为这是一个构造函数(constructors)，因为 他们会产生一个新对象。 

可以写一个**构造签名（ Construct Signatures ）**，方法是在**调用签名前面加一个 new 关键词**；

```ts
class Person { }

interface PersonConstructor {
  new (): Person
}
function createPerson(person: PersonConstructor) {
  const f = new person()
  //const f: Person 
}

createPerson(Person)
```
