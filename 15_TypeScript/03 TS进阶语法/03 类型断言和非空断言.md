# 一、类型断言as

有时候TypeScript无法获取具体的类型信息，这个我们需要使用类型断言（Type Assertions）。 

比如我们通过 document.querySelector('.box')，TypeScript只知道该函数会返回 Element ，但并不知道它具体的类型：

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1760101859000h81lxk.png)

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1760102041000ep8btr.png)


TypeScript只允许类型断言转换为 **更具体** 或者 **不太具体** 的类型版本，此规则可防止不可能的强制转换：

```ts
//下面代码ts检测来说正确，但是代码本身不太正确
const messag: string = 'axl'

const message_1 = messag as any//断言为不太具体

const message_2 = message_1 as number//断言为更具体

console.log(typeof message_2)//string
```

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/17601023220004zuejg.png)

> [!tip]
> 虽然说是进行了断言，但是一旦运行的时候就会回到 string 类型。
> 
> 通过上面的例子可以知道，断言as只是一种辅助编写代码的作用，可以让IDE提供更加好的提示
> 
> <mark class="hltr-orange">最好是在[[01 TS变量声明#三、变量的类型推导(推断)|推导]]不是自己明确知道的那个类型的时候在进行断言</mark>

# 二、非空类型断言!

我们编写下面的代码时，在执行ts的编译阶段会报错：  这是因为传入的 friends 有可能是为undefined的，这个时候是不能执行方法的；

```ts
interface Person {
  name: string
  age: number
  friends?: {
    name: string
  }
}

const person: Person = {
  name: 'why',
  age: 18,
}

console.log(person.friends?.name) // 可选链

person.friends?.name = 'kobe'
//报错：赋值表达式的左侧不能是可选属性访问

prison.friends!.name = 'koube'
//不会报错，强制告诉ts编译，我的friends不为空
```

> [!tip]
> 非空断言使用的是 ! ，表示可以确定某个标识符是有值的，**跳过ts在编译阶段对它的检测**；

使用场景：

```ts
function printMessage(message?: string) {
  console.log(message!.length)
}
printMessage("axl")
```
