# 一、any类型

在某些情况下，我们确实**无法确定一个变量的类型，并且可能它会发生一些变化**，这个时候我们可以使用any类型（类似于Dart 语言中的dynamic类型）。 

any类型有点像一种讨巧的TypeScript手段： 

- 我们可以对any类型的变量**进行任何的操作**，包括获取不存在的属性、方法； 
- 我们给一个any类型的**变量赋值任何的值**，比如数字、字符串的值；

```ts
let a: any = "axl";
a = 123;
a = true;

const array: any[] = ["axl", 18, 1.88]
```

> [!tip]
> 如果对于某些情况的处理过于繁琐不希望添加规定的类型注解（类似网络请求获取到的数据），或者在引入一些第三方库时，缺失了类型注解，这个时候我们可以使用any： 
> 
> 包括在Vue源码中，也会使用到any来进行某些类型的适配；

> [!warning] 
> 虽然 any 类型可以进行任意操作，但是这个操作只是在编译的时候不会报错，如果运行时可能会报错

```ts
let anyValue: any = "hello";

// 不会报错 - 完全放弃类型检查
anyValue.foo.bar(); // 运行时可能出错
anyValue(); // 运行时可能出错
anyValue[0]; // 运行时可能出错
```

# 二、unknown类型

unknown是TypeScript中比较特殊的一种类型，它用于描述类型不确定的变量。 

和any类型有点类似，但是unknown类型的<mark class="hltr-orange">值上</mark>**做任何事情都是不合法**的；

> [!tip]
> 那些是不合法的，请看本节[[03 TS数据类型#三、any和unknown对比|any和unknown对比]]


```ts
const flag = true
let result: unknown

function foo(): string {
  return 'foo'
}
function bar(): number {
  return 123
}

if (flag) {
  result = foo()
} else {
  result = bar()
}

//类型检测(缩小)
if (typeof result === 'string') {
  console.log(result.length)//3
}
```

# 三、any和unknown对比

## 3.1 类型安全性

any类型：

```ts
let anyValue: any = "hello";

// 不会报错 - 完全放弃类型检查
anyValue.foo.bar(); // 运行时可能出错
anyValue(); // 运行时可能出错
anyValue[0]; // 运行时可能出错
```
unknown 类型：

```ts
let unknownValue: unknown = "hello";

// 以下所有操作都会在编译时报错
// unknownValue.foo.bar(); // Error
// unknownValue(); // Error
// unknownValue[0]; // Error
```

## 3.2 赋值规则

any类型：

```ts
let anyVar: any;

// 可以赋值给任意类型
let str: string = anyVar; // ✅ 允许
let num: number = anyVar; // ✅ 允许
let bool: boolean = anyVar; // ✅ 允许
```

unknown类型：

```ts
let unknownVar: unknown;

// 不能赋值给其他类型（除了 any 和 unknown）
// let str: string = unknownVar; // ❌ 错误
// let num: number = unknownVar; // ❌ 错误

// 只能赋值给 any 或 unknown
let anyVar: any = unknownVar; // ✅ 允许
let anotherUnknown: unknown = unknownVar; // ✅ 允许
```

## 3.3 使用前类型检查

unknown 必须进行类型收窄：

```ts
function processValue(value: unknown) {
    // 直接使用会报错
    // value.toUpperCase(); // ❌ 错误
    
    // 需要类型检查
    if (typeof value === "string") {
        value.toUpperCase(); // ✅ 允许
    }
    
    // 或类型断言
    (value as string).toUpperCase(); // ✅ 允许（但要小心）
}
```

any 不需要类型检查：

```ts
function processAnyValue(value: any) {
    value.toUpperCase(); // ✅ 编译通过（但可能运行时出错）
    value.nonExistentMethod(); // ✅ 编译通过
}
```

## 3.4 最佳实践

推荐使用 unknown：

```ts
// 安全的做法
function safeParse(jsonString: string): unknown {
    return JSON.parse(jsonString);
}

const result = safeParse('{"name": "John"}');
if (result && typeof result === "object" && "name" in result) {
    console.log(result.name); // ✅ 安全
}
```

避免使用 any：

```ts
// 不安全的做法
function unsafeParse(jsonString: string): any {
    return JSON.parse(jsonString);
}

const result = unsafeParse('{"name": "John"}');
console.log(result.name); // ❌ 可能运行时出错
console.log(result.nonExistent); // ❌ 编译通过，但运行时出错
```

## 3.5 总结

|特性|any|unknown|
|---|---|---|
|类型安全性|无|高|
|可赋值给其他类型|✅|❌|
|可直接调用方法|✅|❌|
|需要类型收窄|❌|✅|
|推荐使用|尽量避免|推荐用于不确定的类型|

**核心原则**：优先使用 `unknown`，它提供了类型安全性，强制你在使用前进行适当的类型检查。

# 四、void类型

void通常用来指定一个函数是**没有返回值（也可以说返回undefined）** 的，那么它的返回值就是void类型（通过[[01 TS变量声明#三、变量的类型推导(推断)|TS类型推导]]）：


![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/176008482400098fn8p.png)

这个函数我们没有写任何类型，那么它默认返回值的类型就是void的，我们也可以显示的来指定返回值是void：

```ts
function sum(num1: number, num2: number): void {
  console.log(num1 + num2)
}
```

这里还有一个注意事项：  我们可以将undefined赋值给void类型，也就是函数可以返回undefined

```ts
function sum(num1: number, num2: number): void {
  console.log(num1 + num2)
  return undefined
}
```

> [!warning]
> 当**基于上下文的类型推导（Contextual Typing）** 推导出返回类型为 **void** 的时候，并**不会强制**函数一定不能返回内容。

# 五、never类型

never 表示永远不会发生值的类型，比如一个函数： 

- 如果一个函数中是一个死循环或者抛出一个异常，那么这个函数会**返回任何东西**吗？ 
- **不会（意思就是 undefined 都不会返回）**，那么写void类型或者其他类型作为返回值类型都不合适，我们就可以使用never类型；

```ts
function bar(): never {
  while (true) {
    console.log("bar")
  }
}

function foo(): never {
  throw new Error("error")
}
```

那推断 never 类型还是 void类型：

1. 推断为 `never` 类型的情况

- **函数内部所有分支都不会有正常返回**，即：
    - 只包含 `throw` 抛出异常；
    - 或者是无限递归/死循环且不会跳出（并且声明返回类型为 `never`）。
- 例子

```ts
function error(): never {
  throw new Error("error");
}
```

2. 推断为 `void` 类型的情况

- **函数没有返回值**，但函数体可以正常执行到末尾（即使最后是 `throw`，TypeScript 也可能推断为 `void`，尤其未显式声明时）。
- 例子：

```ts
function log(): void {
  console.log("hello");
}
function foo() {
  throw new Error("error");
}
```

> [!note] 总结
> - **所有分支都不会返回时**，推断为 `never`。
> - **没有返回值或可能正常结束时**，推断为 `void`。
> - **建议：需要 `never` 时显式声明返回类型。**

never有什么样的应用场景呢？这里我们举一个例子，但是它用到了联合类型，后面我们会讲到：

```ts
function handerMessage(message: string | number) {
  switch (typeof message) {
    case "string":
      console.log("string")
      break;
    case "number":
      console.log("number")
      break;
    default:
      const check: never = message
  }
}
```

# 六、tuple类型

tuple是元组类型，很多语言中也有这种数据类型，比如Python、Swift等。

```ts
const info: [string, number] = ['why', 18]
const item1 = info[0]//why,并且知道类型是 string
const item2 = info[1]//18，并且知道类型是 number
```

那么tuple和数组有什么区别呢？ 

- 首先，数组中通常建议存放相同类型的元素，不同类型的元素是不推荐放在数组中。（可以放在对象或者元组中） 
- 其次，元组中**每个元素都有自己特殊的类型，根据索引值获取到的值可以确定对应的类型**；

```ts
const info: (string | number)[] = ['why', 18]
const item1 = info[0]//不确定类型

const tinfo: [string, number, boolean] = ['why', 18, true]
const item2 = tinfo[0]//why,并且知道类型是 string
```

那么tuple在什么地方使用的是最多的呢？ **tuple通常可以作为返回的值，在使用的时候会非常的方便**；

- 比如下面代码，就可以明确的知道 count 是一个number类型，setCount 是一个函数类型

```ts
function useState(initialValue: number): [number, (newValue: number) => void] {
  let stateValue = initialValue
  function setValue(newValue: number) {
    stateValue = newValue
  }
  return [stateValue, setValue]
}

const [count, setCount] = useState(100)
console.log(count)//100
setCount(200)
```
