# 一、异步函数-async function
async关键字用于声明一个异步函数： 

* async是asynchronous单词的缩写，异步、非同步； 
* sync是synchronous单词的缩写，同步、同时； 

async异步函数可以有很多中写法：

```js
<script>
  async function foo1() {}
  
  const bar1 = async function foo2() {}
  
  const bar2 = async () => {}
  
  class Person {
    async foo3() {}
  }
</script>
```


# 二、异步函数的执行流程
异步函数的内部代码执行过程和普通的函数是一致的，默认情况下也是会被同步执行。

```js
<script>
  async function foo() {
    console.log("001")
    console.log("002")
    console.log("003")
  }

  foo()
  //001
  //002
  //003
</script>
```

异步函数有返回值时，和普通函数会有区别： 

* **情况一**：异步函数也可以有返回值，但是异步函数的返回值相当于被包裹到Promise.resolve中； 

```js
<script>
  async function foo() {
    //相当于 return new Promise(123)
    return 123
  }
  
  foo().then((res) => {
    console.log("res:", res)
    //res: 123
  })
</script>
```

* **情况二**：如果我们的异步函数的返回值是Promise，状态由会由Promise决定； 

```js
<script>
  async function foo() {
    return new Promise((resolve, reject) => {
      resolve("aaa")
    })
  }
  
  foo().then((res) => {
    console.log("res:", res)
    //res: aaa
  })
</script>
```

* **情况三**：如果我们的异步函数的返回值是一个对象并且实现了thenable，那么会由对象的then方法来决定；

```js
<script>
  async function foo() {
    return {
      then: (resolve, reject) => {
        resolve("bbb")
      }
    }
  }
  
  foo().then((res) => {
    console.log("res:", res)
    //res: bbb
  })
</script>
```

**如果我们在async中抛出了异常，那么程序它并不会像普通函数一样报错，而是会作为Promise的reject来传递；**

```js
<script>
  async function foo() {
    console.log("111")
    throw new Error("my is error")
    console.log("222")
  }
  
  foo().then((res) => {
    console.log("res:", res)
  }).catch((rej) => {
    console.log("rej", rej)
  })

  //111
  /*
  rej Error: my is error
  at foo (03 异步函数返回值.html:52:11)
  at 03 异步函数返回值.html:56:3
  */
</script>
```


# 三、await关键字
 async函数另外一个特殊之处就是可以在它内部使用await关键字，而**普通函数中是不可以**的。 

```js
<script>
  function foo() {
    await 123
  }
  
  foo()
  //Uncaught SyntaxError: await is only valid in async functions and the top level bodies of modules
  //未发现的语法错误:await仅在async函数和模块的顶级主体中有效
</script>
```
 
 await关键字有什么特点呢？ 
 
 * 通常使用await是后面会跟上**一个表达式**，这个表达式会**返回一个Promise**； 
 * 那么await会**等到Promise的状态变成fulfilled状态**，之后继续**执行异步函数**；

```js
<script>
  function requesData(url) {
    return new Promise((resolve, reject) => {
      setTimeout(() => {
        resolve("requesData执行")
      }, 3000);
    })
  }
  
  async function foo() {
    console.log("---------")

    const res1 = await requesData()
    console.log("res1", res1)
  
    const res2 = await requesData("my" + res1)
    console.log("res2", res2)

    console.log("+++++++++")
  }
  
  foo()
  /*
  ---------
  await关键字.html:31 res1 requesData执行
  await关键字.html:34 res2 requesData执行
  await关键字.html:35 +++++++++
  */
</script>
```

如果await后面是一个**普通的值**，那么会直接**返回这个值**； 

```js
<script>
  async function foo() {
    const res1 = await 123
    console.log("res1", res1)
  }
  
  foo()
  //res1 123
</script>
```

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1732872409000xuxzk6.png)


如果await后面是一个**thenable的对象**，那么会**根据对象的then方法调用来决定后续的值**；

```js
<script>
  function thenable() {
    return {
      then: ((reslove, reject) => {
        setTimeout(() => {
          reslove("thenable执行")
        }, 2000);
      })
    }
  }
  
  async function foo() {
    const res1 = await thenable()
    console.log("res1", res1)
  }
  
  foo()
  //res1 thenable执行
</script>
```

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2024/1732872646000g3ndxl.png)


如果await后面的表达式，**返回的Promise是reject的状态**，那么会将这个reject结果直接作为函数的Promise的reject值；

```js
<script>
  function requesData() {
    return new Promise((resolve, reject) => {
      setTimeout(() => {
        reject("requesData error")
      }, 3000);
    })
  }
  
  async function foo() {
    const res1 = await requesData()
    console.log(res1)
  }
  
  foo().catch((rej) => {
    console.log("rej", rej)
    //rej requesData error
  })
</script>
```
