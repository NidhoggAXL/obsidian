**我们先来看一个需求**：有一个对象，我们希望监听这个对象中的属性被设置或获取的过程 

* 通过我们前面所学的知识，能不能做到这一点呢？ 
* 其实是可以的，我们可以通过之前的**属性描述符中的存储属性描述符**[[01 Object.defineProperty]]来做到；

```html
<body>
  <h1 class="box"></h1>
  <script>
    const obj = {
      name: "axl",
      age: 18
    }
  
    //获取h1的元素
    const h1El = document.querySelector(".box")
  
    const keys = Object.keys(obj)
    console.log(keys)//['name', 'age']

    for(const key of keys) {
      let value = obj[key]
      Object.defineProperty(obj, key, {
        set: function(newValue) {
          console.log(`监听：${key}改变为${newValue}`)
          value = newValue
        },
        get: function() {
          console.log(`监听：获取${key}的值`)
          return value
        }
      })
    }
  
    console.log(obj.name)
    //监听：获取name的值
    //axl

    obj.name = "mba"
    //监听：name改变为mba
  </script>
</body>
```

**但是这样做有什么缺点呢？** 

* 首先，Object.defineProperty设计的初衷，不是为了去监听截止一个对象中 所有的属性的。 
	* 我们在定义某些属性的时候，**初衷其实是定义普通的属性**，但是后面我们强 行将它变成了数据属性描述符。 
	* 其次，如果我们想监听更加丰富的操作，比如**新增属性、删除属性，那么 Object.defineProperty是无能为力的。** 
* 所以我们要知道，存储数据描述符设计的初衷并不是为了去监听一个完整的对象。
