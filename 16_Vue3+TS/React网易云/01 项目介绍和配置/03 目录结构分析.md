## 📁 目录结构解析

### 1. **核心配置文件**


#### `package.json`

- **作用**：项目清单文件，定义项目信息、依赖和脚本
    
- **包含内容**：
    
    - 项目名称、版本、描述
        
    - 依赖包列表（dependencies 和 devDependencies）
        
    - 运行脚本：`dev`、`build`、`preview` 等
        

#### `vite.config.ts`

- **作用**：Vite 构建工具的核心配置文件
    
- **功能**：
    
    - 开发服务器配置（端口、代理等）
        
    - 插件配置（React、ESLint 等）
        
    - 构建优化设置
        
    - 路径别名配置
        

#### `tsconfig.json`

- **作用**：TypeScript 主配置文件，定义全局编译选项
    
- **功能**：
    
    - 指定编译目标（ESNext、CommonJS 等）
        
    - 配置模块解析策略
        
    - 启用严格类型检查
        

#### `tsconfig.app.json` & `tsconfig.node.json`

- **作用**：TypeScript 项目引用配置，将应用代码和构建工具代码分离
    
- **区别**：
    
    - `tsconfig.app.json`：用于浏览器端 React 应用代码
        
    - `tsconfig.node.json`：用于 Node.js 端工具代码（如 Vite 配置）
        

### 2. **源代码目录**

#### `src/` 目录

- **作用**：存放所有源代码文件
    
- **典型结构**：

```text
src/
├── main.tsx          # 应用入口点
├── App.tsx           # 根组件
├── components/       # 可复用组件
├── pages/            # 页面组件
├── hooks/            # 自定义 Hooks
├── utils/            # 工具函数
└── assets/           # 静态资源（图片、样式等）
```

#### `public/` 目录

- **作用**：存放不经过 Vite 处理的静态资源
    
- **特点**：
    
    - 不会被编译或优化
        
    - 通过 `/` 根路径直接访问
        
    - 适合放 favicon.ico、robots.txt 等文件
        

### 3. **开发工具配置**

#### `eslint.config.js`

- **作用**：代码质量和风格检查工具配置
    
- **功能**：
    
    - 定义代码规范规则
        
    - 配置 TypeScript 和 React 特定规则
        
    - 集成 Prettier 进行代码格式化
        

#### `.gitignore`

- **作用**：指定 Git 版本控制忽略的文件和目录
    
- **通常忽略**：
    
    - `node_modules/`
        
    - 构建输出目录（如 `dist/`）
        
    - 环境变量文件（如 `.env.local`）
        
    - 日志文件和缓存
        

### 4. **入口文件**

#### `index.html`

- **作用**：应用的 HTML 入口模板
    
- **Vite 特色**：

```html
<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
  </head>
  <body>
    <div id="root"></div>
    <!-- Vite 会自动注入模块脚本 -->
    <script type="module" src="/src/main.tsx"></script>
  </body>
</html>
```

### 5. **其他文件**

#### `README.md`

- **作用**：项目说明文档
    
- **内容**：安装、运行、构建等使用说明
    

#### `package-lock.json`

- **作用**：锁定依赖版本，确保安装一致性
