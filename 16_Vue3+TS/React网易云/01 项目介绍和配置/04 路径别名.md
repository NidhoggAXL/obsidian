
需要在vite.config.ts中添加alias配置，并同时在tsconfig.json中配置路径映射，这样TypeScript才能正确识别这些别名。

1. 首先修改 vite.config.ts:

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import path from 'path'

// https://vite.dev/config/
export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src')
    }
  }
})
```

2. 然后需要在 tsconfig.json 中添加路径映射：

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  }
}
```

配置完成后，你就可以在项目中使用 `@` 来代替 `src` 目录了，例如：

```typescript
import Component from '@/components/Component'
```

这样配置后：

- Vite 在构建时会正确解析 `@` 别名
- TypeScript 在类型检查时也能正确识别路径
- IDE（如VS Code）能够提供正确的代码提示和跳转功能
