# 二、动态管理路由的方式

## 2.1 根据角色

基于角色(Role)的动态路由管理

```ts
const roles = {
 "super": 部分路由,
 "admin": 一部分路由,
 "sever": 少部分路由
}
```

> [!warning] 弊端
> 
> - 每增加一个角色，都要增加 `key: value`
> - 后端返回路由的 json 对象的时候，不好组织 json 对象的结构


## 2.2 根据菜单(menu)

根据在前端的时候，会更具不同的角色返回不同的menu，后面根据不同的菜单对路由进行注册，这样就可以正常时候路由了。

> [!tip] 好处
> 
> - 后端只需要返回一个菜单列表，不需要对注册组件进行抽象为 json
> - 菜单只需要放回路由的路径，路由的组件前端进行路由的注册
> - 有新角色的时候，只是返回一个不同的菜单列表

# 三、创建路由对象和对应组件

#coderwhy/coderhy自动化工具 

当需要使用动态路由管理的时候，需要对每一个路由抽取为一个对象，当菜单有这个这个路由的时候，才对路由进行一个注册。

抽取的对象：（这其实就是一个路由对象）

```ts
const chat = () => import('@/views/main/story/chat/chat.vue')
export default {
  path: '/main/story/chat',
  name: 'chat',
  component: chat,
  children: []
}

```

那么有了路由对象，那么就需要根据创建对应的Vue页面就可以啦。

但是如果存在多个路由对应的页面，在创建的时候就会比较麻烦，需要手动的编写路由对象和对应的页面，那么可以使用 coderwhy 的自动化插件，来自动生成路由对象和对应页面。

> [!abstract]
> 
> 需要保持页面和路由的文件结构基本一致。
> 
> ![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1762327376000rluu6m.png)

安装自动化工具：

```bash
npm install coderwhy
```

如何使用：

```bash
 coderwhy add3page_setup 文件名 -d src/views/main/文件夹名/文件名
```

例如：

```bash
coderwhy add3page_setup department -d src/views/main/system/department
```

> [!ps]
> 如果router和view的main文件下面不存在system文件夹，会创建文件夹，
> - 在router的system文件夹下面创建一个department文件夹里面包含**department.ts**文件![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/17623278200006bxygf.png)
> - 在view的system文件夹下面创建一个department文件夹里面包含**department.vue**文件![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/176232786000095wv4j.png)

# 四、获取所有router对象

通过上面的操作，把所有的路由对象都放到了一个 .ts 的文件里面，那如何获取到所有的router对象呢？

获取了所有 router 对象后，并保存到一个数组中，这样就可以进行路由的注册：

```ts
const router = createRouter({
  history: createWebHashHistory(),
  routes: [所有的路由对象]
})
```

#vite/import/meta/glob

获取所有的路由对象，就需要对文件进行一个动态加载，在vite中需要使用 `import.meta.glob` 来动态加载文件。

```ts
//动态获取所有文件中的路由对象
const locarRoutes: RouteRecordRaw[] = []
//获取router文件下面的所有ts文件，**是包括子文件
const files = import.meta.glob("../../router/main/**/*.ts")
console.log(files)
```

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1762330078000726rwb.png)

获取到的是一个对象，以key=value的形式，但是会发现 value 是一个函数，观察知道这是一个[[03 路由懒加载和属性|懒加载(import)]]的函数。如果希望直接获得数据，可以在第二个参数编写 `{ eager: trur}` (渴望的)

```ts
//动态获取所有文件中的路由对象
const locarRoutes: RouteRecordRaw[] = []
//获取router文件下面的所有ts文件，**是包括子文件
const files = import.meta.glob("../../router/main/**/*.ts", { eager: true})
console.log(files)
```

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1762330416000zjbyfh.png)

这样获取到的 value 就是一个模块，获取到的内容保存到 default 属性中。

在保存的时候会报错：

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1762330655000h17a2t.png)

这是因为 files 的类型有问题：推断中存在 unknown 类型，所有要消除这个 unknown 累心，为什么会是这个呢？模块的话ts无法自行推导，所以显示编写为 any类型。

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/17623306820004u8sd8.png)

```ts
//5
//动态获取所有文件中的路由对象
//locarRoutes的类型是根据createRouter中的routes编写的
const locarRoutes: RouteRecordRaw[] = []
//获取router文件下面的所有ts文件，**是包括子文件
const files:Record<string, any> = import.meta.glob("../../router/main/**/*.ts", { eager: true})
for (const key in files) {
const module = files[key]
locarRoutes.push(module.default)
}
console.log(locarRoutes)
```

# 六、刷新路由错误

当路由注册成功并进入到路由后，如果刷新会发现错误，这是为什么？

因为菜单是在登录的时候获取到的，虽然保存到了本地缓存之中，但是刷新的时候并没有进行登录，如果把路由注册放入到登录的流程中，就会路由错误（刷新并没有登录）。

这个时候就需要把路由的注册冲登录的流程之中抽离出来：

```ts
const useLoginStore = defineStore("login", {
  state: (): ILoginState => ({
    userInfo: {},
    token:  "",
    userMenus: []
  }),
  actions: {
    async loginAccountAction(account: IAcount) {
      //1
      //coderwhy老师的服务器到期，所以自我模拟数据
      //帐号登录，获取token等信息
      // const loginResult = await loginAcountRequest(account)
      //模拟数据保存到login-store.json
      // 对模拟数据进行判断,进行不同帐号模拟登录
      this.token = loginStore.token
      const id = loginStore.id

      //2
      //进行本地缓存
      localCache.setItem(TOKEN, this.token)

      //3
      //根据登录账户的信息，获取角色等信息
      // const userInfoResult = await getUserInfoRequest(id)
      //模拟数据保存到login-authority-super.json和login-authority-admin.json
      if (account.name === "coderwhy") {
        this.userInfo = authoritySuper.data
        //本地缓存
        localCache.setItem("userInfo", authoritySuper.data)
      } else if (account.name === "coderdemo") {
        this.userInfo = authorityAdmin.data
        //本地缓存
        localCache.setItem("userInfo", authorityAdmin.data)
      }

      //4
      //根据角色请求用户的权限(莱单menus)
      //模拟两个用户角色
      // const userMenusResult = await getUserMuneRequest(id)
      //模拟数据保存到login-menum-super.json和login-menum-admin.json
      //对模拟数据进行判断
      if (this.userInfo.name === "coderwhy") {
        this.userMenus = menuSuper.data
        //本地缓存
        localCache.setItem("userMenus", menuSuper.data)
      } else if (this.userInfo.name === "coderdemo") {
        this.userMenus = menuAdmin.data
        //本地缓存
        localCache.setItem("userMenus", menuAdmin.data)
      }
      
      //5根据菜单注册路由
	  const routes = menuToRoutes(this.userMenus)
      routes.forEach((route) => router.addRoute("main", route))
        
      //6
      //页面跳转到main
      router.push("/main")
    },
    loadingCacheAction() {
      const userInfo = localCache.getItem("userInfo")
      const token = localCache.getItem(TOKEN)
      const userMenus = localCache.getItem("userMenus")
      if (userInfo && token && userMenus) {
        this.userInfo = userInfo
        this.token = token
        this.userMenus = userMenus
        //5(重要)根据菜单动态添加路由
        const routes = menuToRoutes(this.userMenus)
        routes.forEach((route) => router.addRoute("main", route))
      }
    }
  }
})
```

是抽离了出来，但是动态路由的注册还是在 Store 中，页面刷新的时候，还是错误，页面刷新并没有去执行 loadingCacheAction 这个方法，还是没有注册路由。

遇到这种页面刷新的就需要使用到 [[04 Vue中安装插件的方式|Vue插件]] 的使用了

```ts
import type { App } from "vue"
import { createPinia } from "pinia"
import useLoginStore from "./login/loginStore"

const pinia = createPinia()

function registerPinia(app: App<Element>) {
  //注册pinia插件
  app.use(pinia)
  //本地cache的加载和路由注册
  const loginStore = useLoginStore()
  loginStore.loadingCacheAction()
}

export default registerPinia

```

在 mian.ts 中也需要注意插件的注册顺序，pinia的流程中路由还没有注册的时候，最好不要使用 router，等pinia流程中的路由注册完成的时候在使用 router 会更好。

```ts
import { createApp } from "vue"
//引入自定义插件
import RegisterIcons from '@/global/register-icons'


import "normalize.css"
import "./styles/index.less"

import router from "./router/index"
import App from "./App.vue"
import registerPinia from "./stores"

const app = createApp(App)
app.use(RegisterIcons)
app.use(registerPinia)
app.use(router)
app.mount("#app")

```

# 七、默认路由

当进入登录操作的时候，是会默认进入一级路由 main ：

```ts
import { TOKEN } from "@/global/constant"
import { localCache } from "@/utils/cache"
import { createRouter, createWebHashHistory } from "vue-router"

const router = createRouter({
  history: createWebHashHistory(),
  routes: [
    {
      path: '/',
      redirect: '/main'
    },
    {
      path: "/main",
      name: "main",
      component: () => import("@/views/main/main.vue")
    },
    {
      path: "/login",
      name: "login",
      component: () => import("@/views/login/login.vue")
    },
    {
      path: "/:pathMatch(.*)*",
      name: "NotFound",
      component: () => import("@/ui/NotFount/NotFount.vue")
    }
  ]
})

router.beforeEach((to) => {
  const token = localCache.getItem(TOKEN)
  //只要是以main开头的都需要对token进行检测
  if (to.path.startsWith("/main") && !token) {
    return "/login"
  }
})

export default router

```


但是前面已经根据菜单匹配注册正确的路由了，那么默认进入 mian 的路由将是不在正确，需要重新修改路由。

第一种方式：给 main 的路由也重定向到注册路由的第一个路由，但是根据不同角色得到的路由是不同的，就需要动态的重定向，`redirect: 动态路劲` 是非常不好编写的。

第二种方式：在路由注册的时候，记录第一个路由对象，里面会保存了路由的路径，在导航守卫里面拦截路由到 main ，并在这里根据保存的路由对象里面的路径重定向。
