# 二、Vue2情况下

如果**关闭 Vue(Official) 这个插件**，那么会发现 `.vue` 文件的导入是错误的。

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/17616581830009wdmwh.png)

> [!info]
> 
> 通过上面图片可以知道，Vue(official) 这个插件时会默认添加 `.vue` 文件的类型声明的，如果不想使用插件默认的 `.vue` 类型(`import Demo`)，则可以通过下面的方式，自我声明类型。

以前在Vue2的情况下（**不使用 setup 函数**）使用TS的时候，在进行组件的导出的时候，不是直接默认导出对象，而是通过Vue内置的一个函数 defineComponent 导出，这样可以有更加好的提示。

```js
//默认组件的导出方式
export default {
  props: {},
  data: () => ({})
}

//使用 defineComponents 导出
import { defineComponent } from 'vue'

export default defineComponent({
  props: {},
  data: () => {

  }
})
```

因为 defineComponent 导出的时候，Vue对函数有进行TS类型的编写，这样在编写组件的属性的时候，VSCoder可以有更加好的提示。

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1761655973000fs82qc.png)

通过 TS 的类型推导，可以知道 definedComponent 函数的返回值类型是一个 DefinedComponent 的类型，那么就可以[[06 编写自定义声明文件|自定义声明文件]]，这样可以导入一个组件的时候可以知道更加具体的类型。

在 [[03 类型声明文件|env.d.ts]] 的 TS 配置文件中编写自定义声明 `.vue` 的文件：

```ts
/// <reference types="vite/client" />

// 自己声明 .vue 文件的类型
declare module "*.vue" {
  import { DefineComponent } from 'vue'
  const component: DefineComponent
  export default component
}
```

后面在使用组件的时候，就可以明确的知道是一个组件类型了:

![gh](https://raw.githubusercontent.com/AXLflechazoPN/Obsidian/main/2025/1761656949000h1n6nw.png)
