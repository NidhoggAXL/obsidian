# 二、基本作用

## VSCode 相关配置

|配置文件|作用|
|---|---|
|**`.vscode/extensions.json`**|推荐项目开发所需的VSCode扩展，团队协作时保持开发环境一致|
|**`.vscode/settings.json`**|工作区特定设置，配置编辑器行为如保存时格式化、代码检查等|

## 包管理和依赖

|配置文件|作用|
|---|---|
|**`package.json`**|项目依赖管理、脚本命令定义、项目元数据|
|**`pnpm-lock.yaml`**|pnpm包管理器的依赖版本锁定文件，确保安装一致性|
|**`pnpm-workspace.yaml`**|pnpm工作区配置，用于monorepo项目管理多个包|

## 代码质量和格式化

| 配置文件                   | 作用                           |
| ---------------------- | ---------------------------- |
| **`.editorconfig`**    | 统一不同编辑器和IDE的基本代码风格设置         |
| **`.prettierrc.json`** | Prettier代码格式化规则配置（缩进、引号、分号等） |
| **`eslint.config.ts`** | ESLint代码质量检查规则（新扁平化配置格式）     |

## TypeScript 配置

|配置文件|作用|
|---|---|
|**`tsconfig.json`**|TypeScript基础配置，其他tsconfig文件继承此配置|
|**`tsconfig.app.json`**|应用程序代码的TypeScript编译配置|
|**`tsconfig.node.json`**|Node.js环境（如Vite配置）的TypeScript配置|
|**`env.d.ts`**|Vite环境变量的TypeScript类型定义|

## 构建工具配置

|配置文件|作用|
|---|---|
|**`vite.config.ts`**|Vite构建工具配置（开发服务器、插件、构建选项等）|

## Git 相关

|配置文件|作用|
|---|---|
|**`.gitignore`**|指定Git版本控制忽略的文件和目录|
|**`.gitattributes`**|定义Git属性，如行结束符、文件差异比较方式等|

## 项目文档和入口

|配置文件|作用|
|---|---|
|**`README.md`**|项目说明文档，包含安装、使用、开发指南等|
|**`index.html`**|应用入口HTML文件|

## 🔧 关键配置说明

### ESLint 新配置格式 (`eslint.config.ts`)

这是ESLint的新扁平化配置格式，替代传统的 `.eslintrc.*` 文件：

```ts
// eslint.config.ts 示例
import js from '@eslint/js';
import { defineConfig } from 'eslint/config';

export default defineConfig([
  {
    files: ['**/*.ts', '**/*.vue'],
    rules: {
      // 你的规则配置
    }
  }
]);
```

### Prettier 配置 (`prettierrc.json`)

```ts
{
  "semi": true,
  "singleQuote": true,
  "printWidth": 80,
  "trailingComma": "none"
}
```
